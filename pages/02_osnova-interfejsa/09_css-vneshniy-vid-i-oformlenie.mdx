# CSS: внешний вид и оформление

## Коротко

CSS отвечает за внешний вид веб-страницы. С его помощью браузер понимает, какого размера должен быть текст, где расположить элементы, какие использовать цвета, отступы, рамки и фон.

HTML создаёт структуру страницы, а CSS превращает эту структуру в оформленный интерфейс.

Понимание CSS помогает тебе точнее объяснять ИИ-ассистенту, как должна выглядеть страница, как должны располагаться её блоки и как интерфейс должен изменяться на разных экранах.

## Что такое CSS

CSS расшифровывается как **Cascading Style Sheets** — каскадные таблицы стилей.

CSS — это язык описания внешнего вида HTML-элементов.

С его помощью можно определить:

* цвет текста и фона;
* размер и начертание шрифта;
* ширину и высоту элементов;
* расстояние между блоками;
* расположение элементов на странице;
* рамки и скругления;
* тени;
* оформление кнопок и ссылок;
* внешний вид страницы на разных экранах;
* простые анимации и визуальные эффекты.

CSS не создаёт смысловую структуру страницы и обычно не отвечает за её основную логику. Он сообщает браузеру, как должны выглядеть уже существующие элементы.

## Для чего используется CSS

Без CSS большинство страниц выглядели бы как простой документ: текст, ссылки, изображения и поля формы располагались бы друг за другом со стандартным оформлением браузера.

CSS позволяет превратить этот документ в полноценный интерфейс.

Например, с его помощью можно:

* расположить меню в верхней части сайта;
* выстроить карточки услуг в несколько колонок;
* выделить главную кнопку;
* оформить форму записи;
* сделать одинаковыми заголовки на разных страницах;
* показать мобильную версию меню на небольшом экране;
* скрыть или изменить некоторые элементы в зависимости от размера экрана;
* создать визуальное состояние кнопки при наведении или нажатии.

CSS помогает поддерживать единый стиль всего проекта. Если одинаковые правила используются на разных страницах, изменение одного правила может обновить внешний вид сразу во многих местах.

## Как CSS связан с HTML

HTML и CSS работают вместе.

HTML описывает, какие элементы существуют на странице:

* заголовок;
* абзац;
* изображение;
* кнопка;
* меню;
* карточка;
* форма.

CSS выбирает эти элементы и применяет к ним правила оформления.

Упрощённо процесс выглядит так:

1. Браузер получает HTML и определяет структуру страницы.
2. Браузер загружает CSS.
3. CSS-правила связываются с подходящими HTML-элементами.
4. Браузер рассчитывает размеры и расположение блоков.
5. Пользователь видит готовую страницу.

Один HTML-элемент может одновременно подходить под несколько CSS-правил. Браузер определяет, какие из них нужно применить и какое правило имеет приоритет.

## Селекторы и свойства

Чтобы оформить элемент, CSS должен сначала указать, к какой части страницы относится правило.

Для этого используются **селекторы**.

Селектор может выбирать:

* определённый тип элемента;
* элемент с конкретным классом;
* отдельный элемент с идентификатором;
* элементы внутри конкретного блока;
* элементы в определённом состоянии;
* несколько похожих элементов одновременно.

После выбора элемента задаются его **свойства**.

Свойство описывает, что именно нужно изменить: цвет, размер, отступ, положение, фон или другой параметр.

Значение свойства уточняет, каким должен быть результат.

Тебе не обязательно сразу запоминать все селекторы и свойства. Для работы с ИИ важнее понимать общий принцип: CSS находит нужный элемент и применяет к нему набор правил оформления.

## Что означает каскад

Слово «каскад» в названии CSS связано с тем, что на один элемент могут воздействовать разные правила.

Например, кнопка может одновременно получать:

* общие стили для всех кнопок;
* стили конкретного раздела;
* стили для мобильного экрана;
* стили при наведении;
* отдельные стили, заданные именно для этой кнопки.

Если правила противоречат друг другу, браузер определяет победителя по нескольким критериям:

* насколько точно правило указывает на элемент;
* где правило находится;
* в каком порядке оно было записано;
* не указано ли более важное правило.

Новичок может изменить цвет или размер элемента, но не увидеть результата, потому что другое CSS-правило имеет более высокий приоритет.

Поэтому при работе с ИИ полезно сообщать не только о желаемом изменении, но и о том, что существующие стили уже могут влиять на элемент.

## Размеры, отступы и блочная модель

Браузер воспринимает большинство элементов как прямоугольные области.

У такой области есть:

* содержимое;
* внутренний отступ;
* рамка;
* внешний отступ.

Эта система называется **блочной моделью**.

Например, у карточки услуги может быть текст внутри, свободное пространство вокруг текста, рамка и расстояние до соседней карточки.

Понимание блочной модели помогает разбираться в частых проблемах:

* блок получился шире, чем ожидалось;
* текст слишком близко расположен к краям;
* между элементами нет нужного расстояния;
* карточки имеют разную высоту;
* часть содержимого выходит за границы контейнера.

В промте для ИИ лучше различать внутренние и внешние отступы, а не писать только «добавь больше пространства».

## Расположение элементов

CSS определяет не только цвет и шрифт, но и расположение элементов.

В современных интерфейсах часто используются два основных подхода:

* **Flexbox** — удобен для расположения элементов в строку или колонку;
* **Grid** — удобен для создания сеток из строк и колонок.

Например, Flexbox можно использовать для меню, ряда кнопок или блока с изображением и текстом.

Grid подходит для каталога, галереи, таблицы тарифов или сетки карточек.

Тебе не обязательно сразу изучать все настройки этих систем. На базовом уровне важно понимать, что расположение блоков должно быть организовано осознанно, а не с помощью случайных отступов.

При постановке задачи ИИ можно прямо указать:

* выстроить карточки сеткой;
* расположить изображение и текст рядом;
* выровнять кнопки по центру;
* прижать один элемент к левому краю, а другой — к правому;
* на мобильном экране перестроить элементы в одну колонку.

## Адаптивное оформление

Пользователи открывают сайты на разных устройствах:

* смартфонах;
* планшетах;
* ноутбуках;
* больших мониторах.

Один и тот же интерфейс не всегда может выглядеть одинаково на всех экранах.

CSS позволяет изменять оформление в зависимости от ширины экрана. Такие изменения часто задаются с помощью **медиазапросов**.

На небольшом экране можно:

* перестроить несколько колонок в одну;
* уменьшить отступы;
* изменить размер текста;
* скрыть второстепенные элементы;
* заменить обычное меню мобильным;
* сделать кнопки шире;
* изменить порядок блоков.

Адаптивность не означает простое уменьшение всей страницы. Интерфейс должен оставаться понятным и удобным на каждом устройстве.

## Состояния элементов

Элементы интерфейса могут выглядеть по-разному в зависимости от своего состояния.

Например, у кнопки могут быть состояния:

* обычное;
* при наведении;
* при нажатии;
* недоступное;
* активное;
* выбранное.

У поля формы могут быть состояния:

* пустое;
* заполненное;
* в фокусе;
* с ошибкой;
* успешно проверенное.

CSS помогает визуально показывать эти состояния пользователю.

Это важно при работе с ИИ: просьбы «сделай красивую кнопку» часто недостаточно. Лучше уточнить, как кнопка должна выглядеть в обычном состоянии, при наведении и когда действие недоступно.

## CSS в современных проектах

В небольшом проекте стили могут храниться в одном или нескольких CSS-файлах.

В более крупных проектах используются разные способы организации:

* отдельные файлы для страниц;
* отдельные стили для компонентов;
* общие стили проекта;
* переменные с цветами и размерами;
* готовые CSS-библиотеки;
* системы дизайна;
* стили, связанные с конкретным фреймворком.

Например, у кнопки, карточки и формы могут быть собственные стили, а цвета и шрифты будут храниться в общих настройках.

Конкретный подход зависит от размера проекта и используемых технологий. При работе с ИИ важно сначала сообщить, как уже организованы стили, чтобы ассистент не создал ещё один несовместимый способ оформления.

## Почему это важно при работе с ИИ-ассистентом

ИИ может быстро написать CSS, но формулировка «сделай красиво» почти всегда оставляет слишком много неопределённости.

Красивый интерфейс для разных людей означает разное. Поэтому в промте полезно описывать:

* общую стилистику;
* основные цвета;
* размеры и типографику;
* расположение блоков;
* ширину основного содержимого;
* отступы;
* оформление кнопок;
* поведение на мобильных устройствах;
* состояния интерактивных элементов;
* существующую систему стилей проекта.

Например, вместо фразы «оформи карточки» можно попросить:

«Оформи карточки услуг в спокойном минималистичном стиле. На большом экране размести их по три в ряд, на планшете — по две, а на смартфоне — по одной. Сделай одинаковые внутренние отступы и расположи кнопки внизу карточек».

Такой запрос даёт ИИ понятные критерии результата.

При проверке созданных стилей стоит обратить внимание:

* не ломается ли страница на разных экранах;
* не дублируются ли одинаковые правила;
* не используются ли случайные размеры без общей системы;
* не конфликтуют ли новые стили со старыми;
* удобно ли читать текст;
* достаточно ли заметны кнопки и ссылки;
* понятны ли состояния элементов;
* не изменяет ли CSS другие страницы без необходимости.

## Простая аналогия

Представь веб-страницу как помещение.

HTML определяет, что в нём есть стены, двери, окна, столы и стулья.

CSS определяет цвет стен, размер мебели, расстояние между предметами, освещение и общий стиль интерьера.

При этом красивое оформление не исправит неправильную планировку. Поэтому структура HTML и оформление CSS должны работать вместе.

## Частые заблуждения новичков

### CSS нужен только для цветов и шрифтов

CSS также управляет размерами, расположением элементов, адаптивностью, состояниями, анимациями и общей компоновкой страницы.

### Можно исправить любую проблему с помощью дополнительных отступов

Случайное увеличение отступов может временно изменить внешний вид, но часто создаёт новые проблемы на других экранах. Сначала нужно понять, как организовано расположение элементов.

### Если стиль не применился, CSS не работает

Чаще всего элемент выбран неправильным селектором или другое правило имеет более высокий приоритет.

### Чем больше эффектов, тем лучше дизайн

Тени, анимации, градиенты и яркие цвета не делают интерфейс автоматически удобным. Оформление должно помогать пользователю понимать страницу.

### Достаточно проверить страницу только на компьютере

Стили могут хорошо выглядеть на широком экране и ломаться на смартфоне. Адаптивность нужно учитывать с самого начала.

### ИИ сам выберет подходящий дизайн

ИИ может предложить оформление, но не знает фирменный стиль, аудиторию и цели проекта, если ты не передал ему этот контекст.

### CSS может исправить неправильную HTML-структуру

CSS способен визуально переставлять элементы, но не всегда исправляет логические ошибки разметки. Сначала должна быть понятная структура, затем оформление.

## Что важно запомнить

* CSS отвечает за внешний вид и расположение элементов страницы.
* HTML создаёт структуру, а CSS оформляет её.
* Селекторы указывают, какие элементы нужно изменить.
* Свойства описывают цвет, размер, отступы, расположение и другие параметры.
* На один элемент могут влиять несколько правил.
* Блочная модель помогает понимать размеры и расстояния между элементами.
* Flexbox и Grid используются для организации расположения блоков.
* Адаптивные стили помогают странице работать на разных экранах.
* При работе с ИИ нужно описывать дизайн конкретно, а не ограничиваться просьбой «сделай красиво».
* Человек определяет требования к интерфейсу и проверяет, насколько результат им соответствует.

## Промты для самостоятельного уточнения

* «Объясни мне простыми словами, как CSS связан с HTML и что происходит, когда браузер загружает стили».

* «Покажи простой пример оформления HTML-карточки с помощью CSS и объясни назначение каждого правила».

* «Объясни блочную модель CSS на примере карточки товара или услуги».

* «Расскажи, чем Flexbox отличается от Grid и в каких случаях лучше использовать каждый из них».

* «Объясни, почему CSS-правило может не применяться к элементу и как найти конфликтующие стили».

* «Помоги составить CSS для адаптивной страницы, которая должна хорошо выглядеть на компьютере, планшете и смартфоне».

* «Проверь стили моего проекта и найди дублирующиеся правила, случайные размеры и возможные конфликты».

* «Составь промт для ИИ-ассистента, чтобы он оформил мою страницу в едином стиле и не изменил другие части проекта».
