# Терминал

## Что такое терминал

**Терминал** — это программа, через которую человек общается с компьютером с помощью текстовых команд.

Обычно мы управляем компьютером через графический интерфейс: окна, кнопки, меню, папки. Терминал делает то же самое, но не через клики, а через команды.

Например, вместо того чтобы открыть папку мышкой, можно написать команду:

```bash
cd Documents
```

А вместо создания новой папки через меню можно написать:

```bash
mkdir my-project
```

Терминал часто используют разработчики, системные администраторы, DevOps-инженеры и все, кто работает с кодом, серверами и инструментами разработки.

---

## Для чего нужен терминал веб-кодеру

Веб-кодеру терминал нужен не для красоты, а как рабочий инструмент. Через него часто выполняются задачи, которые неудобно или невозможно делать только мышкой.

### 1. Работа с файлами и папками

Через терминал можно:

- переходить между папками;
- создавать файлы и директории;
- удалять и копировать файлы;
- смотреть содержимое проекта.

Пример:

```bash
mkdir website
cd website
```

Эти команды создают папку `website` и переходят внутрь неё.

---

### 2. Запуск проектов

Современные веб-проекты часто запускаются через терминал.

Например, если проект написан на JavaScript и использует Node.js, запуск может выглядеть так:

```bash
npm run dev
```

Эта команда может запустить локальный сервер разработки. После этого сайт открывается в браузере, например по адресу:

```text
http://localhost:3000
```

---

### 3. Установка библиотек и пакетов

В веб-разработке часто используются готовые библиотеки: например, React, Vue, Vite, Tailwind CSS и другие.

Они обычно устанавливаются через терминал.

Пример:

```bash
npm install
```

Эта команда устанавливает зависимости проекта, то есть нужные библиотеки, указанные в настройках проекта.

Другой пример:

```bash
npm install axios
```

Так можно установить библиотеку `axios`, которую используют для HTTP-запросов.

---

### 4. Работа с Git

Git — это система контроля версий. Она почти всегда используется через терминал, хотя есть и графические программы.

Примеры команд:

```bash
git status
```

Показывает, какие файлы изменились.

```bash
git add .
```

Добавляет изменения в подготовку к сохранению.

```bash
git commit -m "Add main page"
```

Сохраняет изменения в истории проекта.

```bash
git push
```

Отправляет изменения на GitHub или другой удалённый репозиторий.

---

### 5. Создание новых проектов

Многие современные инструменты создают проект через команду в терминале.

Например:

```bash
npm create vite@latest
```

Эта команда запускает создание нового проекта с помощью Vite.

После этого терминал может задать вопросы: как назвать проект, какой фреймворк выбрать, использовать ли TypeScript и так далее.

---

### 6. Работа с сервером

Когда сайт размещают не только на компьютере разработчика, но и на настоящем сервере, терминал становится особенно важным.

Через терминал можно:

- подключаться к серверу;
- запускать приложения;
- смотреть логи;
- настраивать окружение;
- обновлять проект;
- проверять ошибки.

Например, подключение к серверу может выглядеть так:

```bash
ssh user@example.com
```

Команда `ssh` позволяет управлять удалённым сервером через терминал.

## Терминал, консоль и командная строка — это одно и то же?

В обычной речи эти слова часто используют как синонимы, но технически между ними есть разница.

| Термин | Что означает |
|---|---|
| **Терминал** | Программа-окно, где вводятся команды |
| **Командная строка** | Интерфейс для ввода команд |
| **Консоль** | Более общее название текстового интерфейса |
| **Shell / оболочка** | Программа, которая понимает и выполняет команды |

Например, на macOS есть приложение **Terminal**, на Windows — **Command Prompt**, **PowerShell** и **Windows Terminal**, а на Linux — разные терминальные программы.

---

## Что такое shell

**Shell** — это командная оболочка. Она принимает команды, понимает их и передаёт операционной системе.

Популярные оболочки:

| Shell | Где встречается |
|---|---|
| **Bash** | Linux, macOS, серверы |
| **Zsh** | macOS, Linux |
| **PowerShell** | Windows |
| **CMD** | Старый вариант командной строки Windows |

Когда ты пишешь команду в терминале, на самом деле её обрабатывает shell.

---

## Как выглядит работа в терминале

Обычно в терминале есть строка приглашения, например:

```bash
user@computer:~/project$
```

Это может означать:

- `user` — имя пользователя;
- `computer` — имя компьютера;
- `~/project` — текущая папка;
- `$` — приглашение ввести команду.

Ты вводишь команду:

```bash
ls
```

И получаешь ответ, например:

```text
index.html
style.css
script.js
```

Так терминал показывает содержимое текущей папки.

## Базовые команды терминала

Команды могут немного отличаться в Windows, macOS и Linux, но общая идея похожа.

### Навигация по папкам

```bash
pwd
```

Показывает, в какой папке ты сейчас находишься.

```bash
ls
```

Показывает файлы и папки внутри текущей папки.

```bash
cd folder-name
```

Переходит в папку `folder-name`.

```bash
cd ..
```

Возвращает на один уровень выше.

---

### Работа с папками и файлами

```bash
mkdir new-folder
```

Создаёт новую папку.

```bash
touch index.html
```

Создаёт новый файл.  
На Windows в PowerShell чаще используют:

```powershell
New-Item index.html
```

```bash
rm file.txt
```

Удаляет файл.

```bash
cp file.txt copy.txt
```

Копирует файл.

```bash
mv old.txt new.txt
```

Переименовывает или перемещает файл.

---

## Важная особенность терминала

Терминал выполняет команды буквально.

Если в графическом интерфейсе часто можно отменить действие, то в терминале некоторые команды могут сразу удалить или изменить данные.

Например:

```bash
rm file.txt
```

удаляет файл.

А команда вроде:

```bash
rm -rf folder
```

может удалить целую папку со всем содержимым.

Поэтому новичку важно:

- не копировать непонятные команды вслепую;
- внимательно смотреть, в какой папке ты находишься;
- осторожно использовать команды удаления;
- сначала понимать смысл команды, а потом запускать её.

---

## Почему терминал кажется сложным

Терминал может пугать, потому что в нём нет привычных кнопок. Но на практике новичку не нужно знать сотни команд.

Для начала достаточно понимать:

- как перейти в нужную папку;
- как посмотреть файлы;
- как запустить проект;
- как установить зависимости;
- как выполнить команды Git.

Большая часть повседневной работы веб-кодера сводится к повторяющимся командам.

---

## Пример типичного сценария веб-кодера

Допустим, у тебя есть проект сайта. Работа может выглядеть так:

```bash
cd my-website
npm install
npm run dev
```

Что происходит:

1. `cd my-website` — переход в папку проекта.
2. `npm install` — установка нужных библиотек.
3. `npm run dev` — запуск проекта в режиме разработки.

После этого сайт можно открыть в браузере и работать с ним.

## Как терминал связан с VS Code

В редакторе **Visual Studio Code** есть встроенный терминал.

Обычно его можно открыть через меню:

```text
Terminal → New Terminal
```

Или горячими клавишами:

```text
Ctrl + `
```

Это удобно, потому что можно писать код и сразу запускать команды в одном окне.

Например, слева открыт файл `index.html`, а снизу терминал, где запущен проект.

## Что нужно знать новичку в первую очередь

Для старта веб-кодеру полезно освоить:

| Навык | Зачем нужен |
|---|---|
| Переход между папками | Чтобы открыть нужный проект |
| Просмотр файлов | Чтобы понимать структуру проекта |
| Запуск команд `npm` | Для работы с JavaScript-проектами |
| Базовые команды Git | Для сохранения и отправки изменений |
| Осторожность с удалением | Чтобы не потерять файлы |
| Понимание ошибок | Чтобы читать сообщения терминала |

| Команда | Для чего используется |

| `ls` | Посмотреть файлы |

## Простая аналогия

Терминал можно представить как **пульт управления компьютером через текст**.

Графический интерфейс говорит:

> Нажми сюда, потом сюда, потом выбери пункт меню.

Терминал говорит:

```bash
npm run dev
```

Одна команда может заменить несколько действий мышкой.

---

## Главное

Терминал — это один из базовых инструментов веб-разработчика. Он нужен для работы с проектами, установки библиотек, запуска серверов разработки, работы с Git и управления файлами.

Новичку не нужно сразу становиться экспертом в командной строке. Достаточно постепенно привыкнуть к базовым командам и понимать, что именно они делают. Со временем терминал становится не страшным «чёрным окном», а обычной рабочей панелью управления проектом.
