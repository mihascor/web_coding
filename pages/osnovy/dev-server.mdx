# Dev сервер

## Что такое dev server

**Dev server** — это локальный сервер для разработки. Он запускается на твоём компьютере и показывает сайт или веб-приложение в браузере во время работы над кодом.

Обычно ты запускаешь его командой вроде:

```bash
npm run dev
```

После этого в терминале появляется адрес, например:

```text
http://localhost:5173
```

Ты открываешь этот адрес в браузере и видишь свой проект.

---

## Зачем он нужен

Dev server нужен, чтобы **разрабатывать сайт в условиях, похожих на настоящую работу в браузере**, но быстрее и удобнее.

Он умеет:

1. **Отдавать файлы браузеру**
    
    Браузер запрашивает HTML, CSS, JavaScript, картинки и шрифты — dev server их отдаёт.
    
2. **Автоматически обновлять страницу**
    
    Когда ты меняешь код, браузер может сам обновиться.
    
3. **Обновлять часть приложения без полной перезагрузки**
    
    Это называется **HMR — Hot Module Replacement**. Например, ты изменила компонент React, и он обновился прямо на странице без сброса состояния формы. Webpack описывает HMR как замену, добавление или удаление модулей во время работы приложения без полной перезагрузки страницы. ([webpack](https://webpack.js.org/concepts/hot-module-replacement/?utm_source=chatgpt.com "Hot Module Replacement"))
    
4. **Обрабатывать современный JavaScript**
    
    Современный код часто нужно преобразовать перед запуском в браузере: TypeScript, JSX, SCSS, импорты модулей, переменные окружения и так далее.
    
5. **Показывать ошибки разработки**
    
    Например, если в коде синтаксическая ошибка, dev server часто выводит её прямо в терминале или браузере.

## Почему нельзя просто открыть HTML-файл

На самых первых шагах можно открыть файл напрямую:

```text
file:///Users/me/project/index.html
```

Но это быстро становится неудобно.

Некоторые вещи в браузере работают корректно только через HTTP-сервер: загрузка модулей, запросы к API, маршрутизация, относительные пути, некоторые политики безопасности браузера. MDN отдельно объясняет, что локальный тестовый сервер нужен, чтобы проверять сайт не как набор файлов, а как веб-страницы, отдаваемые сервером. ([MDN Web Docs](https://developer.mozilla.org/en-US/docs/Learn_web_development/Howto/Tools_and_setup/set_up_a_local_testing_server?utm_source=chatgpt.com "How do you set up a local testing server? - MDN Web Docs"))

## Dev server — это не production server

Важно различать:

|Тип сервера|Для чего нужен|
|---|---|
|**Dev server**|Для разработки на своём компьютере|
|**Production server**|Для настоящих пользователей в интернете|

Например, в Next.js есть разные команды: `next dev` запускает сервер разработки, `next build` собирает приложение для продакшена, а `next start` запускает production-сервер. ([nextjs.org](https://nextjs.org/docs/app/getting-started/installation?utm_source=chatgpt.com "Getting Started: Installation"))

Dev server обычно:

- не оптимизирован для максимальной скорости у пользователей;
    
- может показывать подробные ошибки;
    
- может пересобирать код «на лету»;
    
- не предназначен для публикации сайта в интернет как финальная версия.

## Пример на практике

Допустим, у тебя проект на Vite.

Ты пишешь:

```bash
npm create vite@latest
cd my-app
npm install
npm run dev
```

После запуска открываешь адрес вроде:

```text
http://localhost:5173
```

Теперь ты меняешь файл, например:

```jsx
function App() {
  return <h1>Привет!</h1>
}
```

Сохраняешь — и браузер сразу показывает изменение. Vite как раз делает акцент на быстром dev server и HMR: изменения отражаются почти сразу при сохранении файлов. ([vitejs](https://vite.dev/?utm_source=chatgpt.com "Vite | Next Generation Frontend Tooling"))

---

## Что значит `localhost`

`localhost` — это адрес твоего собственного компьютера.

То есть:

```text
http://localhost:5173
```

означает:

> Открой сайт, который запущен на моём компьютере, на порту 5173.

**Порт** — это как «дверь», через которую конкретная программа принимает запросы. Один dev server может работать на `5173`, другой — на `3000`, третий — на `8080`.

## Популярные dev servers и инструменты

|Инструмент|Где часто используется|
|---|---|
|**Vite dev server**|React, Vue, Svelte, Vanilla JS|
|**Next.js dev server**|Next.js-приложения|
|**webpack-dev-server**|Проекты на Webpack|
|**Angular dev server**|Angular-приложения|
|**Live Server**|Простые HTML/CSS/JS-проекты в VS Code|

Для начинающего веб-кодера чаще всего встречаются:

```bash
npm run dev
npm start
```

Обе команды часто запускают локальный сервер, но точное поведение зависит от проекта.

## Простая аналогия

Можно представить так:

**Dev server — это тренировочная сцена.**

Ты ещё не показываешь сайт настоящим пользователям, но уже можешь видеть, как он работает в браузере почти как настоящий сайт.

А **production server** — это уже сцена перед зрителями.

---

## Главное

Dev server — это локальный сервер для удобной разработки. Он запускает проект на твоём компьютере, показывает его в браузере, автоматически обновляет изменения, помогает находить ошибки и обрабатывать современный код.

Для начинающего важно запомнить:

```bash
npm run dev
```

обычно означает:

> Запусти проект в режиме разработки.
