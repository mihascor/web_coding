# JSON: универсальный формат передачи данных

## Коротко

Веб-приложения постоянно передают данные между своими частями. Браузер отправляет информацию серверу, сервер возвращает результаты, а разные сервисы обмениваются сообщениями через API.

Чтобы все участники этого обмена одинаково понимали структуру информации, данные передают в определённом формате. Один из самых распространённых форматов в веб-разработке — JSON.

Тебе не обязательно вручную создавать сложные JSON-документы. Но важно понимать, как выглядит JSON, какие данные он может содержать и почему ИИ-ассистент часто показывает его в ответах.

## Что такое JSON

JSON — это текстовый формат для хранения и передачи структурированных данных.

Название расшифровывается как JavaScript Object Notation — объектная запись JavaScript. Однако JSON используется не только в JavaScript. С ним могут работать практически любые современные языки программирования и веб-технологии.

JSON позволяет представить данные в понятной структуре: указать названия полей и значения, которые им соответствуют.

Например, данные пользователя можно условно представить так:

```json
{
  "name": "Анна",
  "age": 32,
  "isClient": true
}
```

Здесь видно, что у пользователя есть имя, возраст и признак того, является ли он клиентом.

Это не полноценная программа и не команда для компьютера. Это описание данных в согласованном формате.

## Из чего состоит JSON

В JSON данные обычно организованы в виде пар:

* название поля;
* значение поля.

Например, в записи `"name": "Анна"`:

* `name` — название поля;
* `Анна` — значение этого поля.

JSON может содержать разные виды значений:

* текст;
* числа;
* значения `true` и `false`;
* пустое значение;
* списки;
* вложенные объекты.

Например, информация о заказе может включать данные клиента, список товаров, общую сумму и статус оплаты. Все эти сведения можно объединить в одной JSON-структуре.

### Объекты

Объект объединяет несколько связанных характеристик.

Например, объект пользователя может содержать имя, email и роль.

Объекты записываются внутри фигурных скобок:

```json
{
  "name": "Анна",
  "role": "client"
}
```

### Массивы

Массив — это список значений или объектов.

Например, сервер может вернуть список записей на приём:

```json
[
  {
    "date": "2026-07-15",
    "service": "Консультация"
  },
  {
    "date": "2026-07-20",
    "service": "Процедура"
  }
]
```

Такой ответ содержит два отдельных элемента в одном списке.

## Для чего JSON используется

JSON применяется там, где одной части системы нужно передать данные другой части.

Чаще всего он используется:

* в ответах API;
* в запросах от браузера к серверу;
* при обмене данными между сервисами;
* в настройках проектов;
* при хранении небольших структурированных наборов данных;
* в интеграциях с CRM, платёжными системами и другими платформами.

Например, пользователь заполняет форму записи на сайте. Браузер может собрать введённые данные и отправить их серверу в формате JSON:

```json
{
  "name": "Анна",
  "phone": "+49...",
  "serviceId": 12
}
```

Сервер получает эти данные, проверяет их и создаёт запись. После этого он также может вернуть результат в формате JSON:

```json
{
  "success": true,
  "appointmentId": 458
}
```

Таким образом, JSON выступает общим языком обмена данными.

## Как JSON работает в общей картине

Рассмотрим упрощённую последовательность работы веб-приложения.

1. Пользователь выполняет действие на сайте.
2. JavaScript собирает необходимые данные.
3. Браузер отправляет запрос к API.
4. Данные запроса могут быть представлены в формате JSON.
5. Сервер читает полученную структуру.
6. Backend выполняет нужную операцию.
7. Сервер формирует ответ.
8. Браузер получает JSON и использует его для обновления интерфейса.

Например, пользователь открывает каталог услуг. Frontend отправляет запрос серверу. Сервер находит услуги в базе данных и возвращает список:

```json
{
  "services": [
    {
      "id": 1,
      "name": "Консультация",
      "price": 100
    },
    {
      "id": 2,
      "name": "Процедура",
      "price": 250
    }
  ]
}
```

Frontend получает этот JSON и превращает данные в карточки, которые видит пользователь.

Сам JSON не определяет внешний вид карточек. Он только передаёт информацию. За отображение отвечают HTML, CSS, JavaScript и компоненты интерфейса.

## JSON и API

JSON тесно связан с API, но это не одно и то же.

API определяет:

* куда можно отправлять запросы;
* какие действия доступны;
* какие данные нужно передать;
* какой ответ можно получить.

JSON определяет формат, в котором эти данные могут быть записаны.

Можно представить, что API — это служба доставки с определёнными правилами работы, а JSON — стандартная упаковка, в которой передаётся содержимое.

Один API может принимать и возвращать JSON. При этом JSON может использоваться и в других ситуациях, не связанных напрямую с API.

## JSON и база данных

JSON также не является базой данных.

База данных отвечает за длительное хранение и поиск информации. JSON — это формат представления данных.

Сервер может получить информацию из базы данных, преобразовать её в JSON и отправить браузеру. Но это не означает, что данные обязательно хранятся в базе в таком же виде.

Например, в базе клиенты, услуги и записи могут находиться в разных таблицах. Backend собирает нужную информацию и формирует из неё удобный JSON-ответ для frontend.

Поэтому структура JSON-ответа и структура базы данных могут отличаться.

## JSON и JavaScript

Внешне JSON похож на объект JavaScript, но между ними есть различия.

JSON — это текст. Чтобы использовать полученные данные в программе, приложение обычно преобразует JSON-текст в структуру, понятную языку программирования.

При отправке данных происходит обратный процесс: программная структура преобразуется в JSON-текст.

Обычно эти преобразования выполняются автоматически или с помощью стандартных инструментов. Для общего понимания тебе важно различать:

* данные внутри программы;
* текстовый JSON, который передаётся между системами.

ИИ-ассистент может использовать выражения «преобразовать в JSON», «распарсить JSON» или «сериализовать данные». Обычно речь идёт о преобразовании между текстовым форматом и внутренней структурой программы.

## Почему структура JSON важна

Системам недостаточно получить просто набор значений. Они должны понимать назначение каждого значения.

Например, запись:

```json
{
  "name": "Анна",
  "phone": "+49..."
}
```

понятнее, чем список:

```json
[
  "Анна",
  "+49..."
]
```

В первом случае видно, где имя, а где телефон. Во втором случае значение каждого элемента приходится определять по его позиции.

Понятная структура JSON помогает:

* передавать данные без двусмысленности;
* добавлять новые поля;
* находить ошибки;
* документировать API;
* разделять frontend и backend;
* объяснять ИИ, какие данные ожидаются.

Названия полей особенно важны. Поля `userName`, `serviceName` и `appointmentDate` дают больше информации, чем неопределённые названия вроде `value1` или `data2`.

## Почему это важно при работе с ИИ-ассистентом

ИИ-ассистент часто предлагает структуру JSON при создании API, форм, интеграций и серверной логики.

Понимание JSON помогает тебе оценить:

* какие данные отправляются;
* какие поля обязательны;
* какие значения возвращает сервер;
* где находится список, а где отдельный объект;
* правильно ли названы поля;
* не передаются ли лишние или конфиденциальные данные;
* соответствует ли ответ API потребностям интерфейса.

Вместо общего запроса:

> Сделай получение данных с сервера.

можно сформулировать задачу точнее:

> Создай запрос к API, который получает список услуг в формате JSON. Для каждой услуги нужны идентификатор, название, цена и длительность.

Во втором случае ИИ лучше понимает ожидаемую структуру результата.

Также можно передать ИИ пример JSON и попросить создать под него интерфейс:

> Вот пример ответа API. Объясни его структуру и предложи, какие компоненты интерфейса нужны для отображения этих данных.

Так ты задаёшь ИИ конкретную модель данных, а не оставляешь структуру полностью на его усмотрение.

## Простая аналогия

JSON можно сравнить с заполненной анкетой.

В анкете каждое значение имеет подпись:

* имя;
* телефон;
* дата рождения;
* адрес.

Если передать только набор ответов без названий полей, другому человеку будет трудно определить, к какому вопросу относится каждый ответ.

JSON работает похожим образом. Он передаёт не только значения, но и названия, которые объясняют их назначение.

Вложенные объекты можно представить как дополнительные разделы анкеты, а массивы — как повторяющиеся списки, например список детей, заказов или записей.

## Частые заблуждения новичков

### JSON — это язык программирования

Нет. JSON не выполняет действия и не содержит программную логику. Он только описывает данные.

### JSON — это API

Нет. API определяет правила взаимодействия между системами. JSON может использоваться как формат данных внутри этого взаимодействия.

### JSON — это база данных

Нет. JSON может храниться в файле или базе данных, но сам по себе он не предоставляет полноценную систему хранения, поиска и управления данными.

### Любой объект JavaScript является правильным JSON

Нет. Несмотря на внешнее сходство, у JSON есть строгие правила оформления. Например, названия полей и текстовые значения записываются в двойных кавычках.

### Frontend должен получать все данные из базы

Обычно сервер должен передавать только ту информацию, которая действительно нужна интерфейсу. Лишние данные увеличивают объём ответа и могут создавать риски безопасности.

### JSON определяет внешний вид сайта

Нет. JSON содержит данные. За оформление и расположение элементов отвечает интерфейс.

### Структуру JSON можно менять без последствий

Если frontend ожидает поле `price`, а backend внезапно начинает возвращать поле `cost`, интерфейс может перестать работать. Изменения структуры данных должны быть согласованы между частями приложения.

## Что важно запомнить

* JSON — это текстовый формат хранения и передачи структурированных данных.
* Он часто используется при работе с API.
* JSON может содержать объекты, списки, текст, числа, логические и пустые значения.
* JSON не является языком программирования, API или базой данных.
* Backend часто формирует JSON на основе данных из базы, а frontend использует его для построения интерфейса.
* Названия и структура полей должны быть понятными и согласованными.
* Понимание JSON помогает точнее объяснять ИИ, какие данные приложение должно отправлять и получать.
* Человек должен проверять структуру данных, даже когда её предлагает ИИ-ассистент.
