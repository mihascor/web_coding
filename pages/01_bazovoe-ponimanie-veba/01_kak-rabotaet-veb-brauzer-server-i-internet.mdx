# Как работает веб: браузер, сервер и интернет

## Коротко

Когда ты открываешь сайт, браузер не показывает его «из ниоткуда». Он отправляет запрос через интернет, находит нужный сервер, получает от него данные и превращает их в страницу на экране.

Чтобы осознанно работать с ИИ-ассистентом в веб-кодинге, важно понимать роли трех основных участников:

* браузер показывает сайт и взаимодействует с пользователем;
* сервер хранит данные и выполняет серверную логику;
* интернет передает запросы и ответы между ними.

Это базовая схема, на которой строится почти вся веб-разработка.

## Что это такое

### Браузер

Браузер — это программа, в которой пользователь открывает сайты. Например, Chrome, Safari, Firefox или Edge.

Он умеет:

* отправлять запросы к сайтам;
* получать файлы и данные;
* читать HTML, CSS и JavaScript;
* собирать из них страницу;
* реагировать на действия пользователя.

Браузер можно назвать клиентом. Клиентом в веб-разработке называют программу или устройство, которое обращается к серверу за информацией.

### Сервер

Сервер — это компьютер или программа, которая принимает запросы и отправляет ответы.

На сервере могут храниться:

* файлы сайта;
* изображения;
* данные пользователей;
* товары и заказы;
* записи на прием;
* настройки приложения.

Сервер также может выполнять логику. Например, проверять пароль, рассчитывать стоимость заказа, сохранять форму или определять права пользователя.

Слово «сервер» может означать как физический компьютер, так и программу, которая обрабатывает запросы. На базовом уровне достаточно понимать сервер как сторону, которая получает запрос и готовит ответ.

### Интернет

Интернет — это глобальная сеть, которая связывает устройства между собой.

Он позволяет браузеру обратиться к серверу, даже если сервер находится в другой стране. Через интернет передаются запросы, страницы, изображения, видео, сообщения и другие данные.

Сам интернет не создает сайт и не хранит его содержимое. Он выступает средой передачи данных между браузером и сервером.

## Для чего это используется

Связка «браузер — интернет — сервер» позволяет пользователю работать с сайтом или веб-приложением.

Например, когда пользователь открывает страницу клиники:

1. Браузер запрашивает страницу у сервера.
2. Сервер отправляет файлы сайта.
3. Браузер отображает страницу.
4. Пользователь нажимает кнопку записи.
5. Браузер отправляет введенные данные на сервер.
6. Сервер проверяет данные и сохраняет запись.
7. Браузер получает ответ и показывает сообщение об успешной записи.

Такая схема используется в интернет-магазинах, личных кабинетах, социальных сетях, CRM-системах, сервисах бронирования и практически любых современных веб-проектах.

## Как это работает в общей картине

Рассмотрим упрощенный путь от ввода адреса сайта до появления страницы.

### 1. Пользователь вводит адрес

Пользователь вводит в браузере адрес сайта, например `example.com`.

Этот адрес называется доменом. Он помогает найти сервер, на котором работает сайт.

### 2. Браузер находит сервер

Браузеру нужно определить, к какому серверу обращаться. Для этого доменное имя связывается с IP-адресом.

IP-адрес — это сетевой адрес устройства. Его можно сравнить с адресом здания, по которому интернет понимает, куда доставить запрос.

### 3. Браузер отправляет запрос

Браузер отправляет серверу сообщение: какую страницу или какие данные он хочет получить.

Такое сообщение называется HTTP-запросом.

Например, браузер может попросить:

* главную страницу;
* список товаров;
* данные пользователя;
* изображение;
* результат поиска.

### 4. Сервер обрабатывает запрос

Сервер определяет, что именно нужно сделать.

Он может:

* найти готовый файл;
* обратиться к базе данных;
* проверить права пользователя;
* выполнить расчет;
* сформировать результат.

### 5. Сервер отправляет ответ

Сервер возвращает браузеру HTTP-ответ.

В ответе могут находиться:

* HTML-разметка;
* CSS-стили;
* JavaScript-код;
* изображения;
* данные в формате JSON;
* сообщение об ошибке.

### 6. Браузер показывает результат

Браузер получает ответ и обрабатывает его.

HTML задает структуру страницы, CSS отвечает за оформление, а JavaScript добавляет поведение и интерактивность.

В результате пользователь видит не набор файлов, а готовый интерфейс.

## Статический и динамический обмен данными

Не все сайты работают одинаково.

### Статическая страница

Сервер отправляет заранее подготовленные файлы. Содержимое страницы почти не меняется в зависимости от пользователя.

Так могут работать простые сайты-визитки, справочные страницы или лендинги.

### Динамическое веб-приложение

Сервер формирует ответ с учетом данных, пользователя и текущей ситуации.

Например:

* личный кабинет показывает данные конкретного человека;
* интернет-магазин отображает актуальные остатки;
* сервис записи показывает свободное время;
* CRM загружает список клиентов из базы данных.

Во многих современных проектах браузер сначала получает основной интерфейс, а затем отдельно запрашивает данные у сервера.

## Где находится база данных

База данных — это отдельная система для хранения информации.

Обычно браузер не обращается к ней напрямую. Запрос проходит через сервер:

**браузер → сервер → база данных → сервер → браузер**

Это важно для безопасности. Сервер решает:

* какие данные можно показать;
* кто имеет право их изменить;
* правильно ли заполнена форма;
* можно ли выполнить операцию.

Например, пользователь не должен иметь прямой доступ ко всей таблице клиентов. Браузер отправляет запрос серверу, а сервер возвращает только разрешенную информацию.

## Почему браузер и сервер разделены

Браузер работает на устройстве пользователя, а сервер — на стороне владельца проекта или облачного сервиса.

Такое разделение позволяет:

* хранить важные данные централизованно;
* управлять доступом;
* обновлять приложение для всех пользователей;
* выполнять защищенные операции;
* использовать один сервер для разных устройств;
* синхронизировать данные между пользователями.

Браузер отвечает в основном за интерфейс и взаимодействие. Сервер отвечает за данные, правила и защищенные операции.

Это упрощенная модель, но она помогает правильно понимать устройство веб-проектов.

## Почему это важно при работе с ИИ-ассистентом

ИИ-ассистент может написать код, но ему нужно понимать, для какой части системы предназначена задача.

Запрос «сделай форму записи» недостаточно точен. Такая задача может включать несколько частей:

* создать форму в браузере;
* проверить заполнение полей;
* отправить данные на сервер;
* сохранить запись в базе;
* вернуть сообщение об успехе;
* обработать ошибку.

Если ты понимаешь роли браузера и сервера, то можешь точнее сформулировать задачу:

> Создай форму записи на frontend, добавь проверку обязательных полей и отправку данных на backend. На сервере проверь данные, сохрани их в базе и верни понятный ответ.

Такой промт дает ИИ гораздо больше полезного контекста.

Понимание общей схемы также помогает проверять результат. Например, ты можешь заметить, что ИИ:

* разместил секретный ключ в браузере;
* забыл добавить серверную проверку данных;
* создал интерфейс, но не реализовал сохранение;
* обращается к базе данных напрямую из браузера;
* не обрабатывает ошибки сервера.

Человек остается архитектором задачи: определяет цель, разделяет систему на части и проверяет, правильно ли ИИ связал их между собой.

## Простая аналогия

Работу веба можно сравнить с посещением ресторана.

* Пользователь — посетитель.
* Браузер — столик и меню, через которые посетитель делает заказ.
* Интернет — официант, который передает заказ.
* Сервер — кухня, где заказ обрабатывается.
* База данных — склад и учетная система.
* Ответ сервера — готовое блюдо или сообщение, что нужного блюда нет.

Посетитель обычно не идет на кухню и не берет продукты со склада самостоятельно. Он взаимодействует через официанта.

Так же и браузер обычно не работает с базой данных напрямую. Он отправляет запрос серверу, а сервер решает, что можно сделать и какой ответ вернуть.

## Частые заблуждения новичков

### «Сайт находится в браузере»

Браузер только получает, обрабатывает и показывает сайт. Основные файлы и данные обычно находятся на сервере или загружаются с других серверов.

### «Интернет и веб — одно и то же»

Интернет — это сеть. Веб — один из способов использовать эту сеть. Кроме сайтов, через интернет работают электронная почта, мессенджеры, видеозвонки и другие сервисы.

### «Сервер — это обязательно огромный компьютер»

Сервером может быть обычный компьютер, виртуальная машина, облачный сервис или специальная программа. Главное — он принимает запросы и отправляет ответы.

### «Весь код сайта выполняется на сервере»

Часть кода выполняется в браузере, а часть — на сервере. Такое разделение называют клиентской и серверной частью.

### «Если страница отображается, значит сервер больше не нужен»

После загрузки страницы браузер может продолжать обращаться к серверу: получать данные, сохранять формы, обновлять профиль, выполнять поиск или загружать новые сообщения.

### «Браузер может безопасно хранить любые секретные данные»

Код и данные, отправленные в браузер, потенциально доступны пользователю. Пароли от базы данных, секретные ключи и защищенная логика должны оставаться на серверной стороне.

### «ИИ сам поймет, где должен работать код»

ИИ может сделать неверное предположение. В промте лучше указывать, относится задача к браузеру, серверу, базе данных или сразу к нескольким частям.

## Что важно запомнить

* Браузер — это клиент, который показывает интерфейс и отправляет запросы.
* Сервер принимает запросы, выполняет логику и возвращает ответы.
* Интернет передает данные между браузером и сервером.
* База данных обычно доступна через сервер, а не напрямую из браузера.
* Один пользовательский сценарий может затрагивать frontend, backend и базу данных.
* При работе с ИИ важно указывать, для какой части веб-приложения нужно написать или изменить код.
* ИИ помогает реализовать решение, но человек должен понимать структуру задачи и проверять результат.

## Промты для самостоятельного уточнения

* «Объясни мне простыми словами, что происходит после ввода адреса сайта в браузере».
* «Покажи путь запроса от браузера к серверу и обратно на примере формы записи».
* «Объясни разницу между браузером, сервером, хостингом и базой данных».
* «Приведи простой пример задачи, в которой одновременно участвуют frontend, backend и база данных».
* «Объясни, какой код должен выполняться в браузере, а какой — на сервере».
* «Покажи типичные ошибки безопасности, которые возникают, когда серверную логику размещают в браузере».
* «Помоги разделить функцию моего сайта на задачи для браузера, сервера и базы данных».
* «Составь промт для ИИ-ассистента, чтобы он создал простую веб-форму с отправкой данных на сервер».
