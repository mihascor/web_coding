# 12. Адаптивная верстка: как сайт подстраивается под разные экраны

Пользователи открывают сайты на устройствах с разными размерами экранов: смартфонах, планшетах, ноутбуках и больших мониторах.

**Адаптивная верстка** — это подход, при котором структура и оформление страницы меняются в зависимости от ширины экрана.

Цель адаптивности — сохранить удобство использования сайта на любом устройстве.

## Что меняется на разных экранах

На большом экране элементы могут располагаться в несколько колонок, а на смартфоне — выстраиваться друг под другом.

Также могут изменяться:

* размеры текста;
* ширина блоков;
* расстояния между элементами;
* расположение меню;
* размеры изображений;
* количество колонок;
* видимость второстепенных элементов.

Например, на компьютере список услуг может отображаться в три колонки, а на телефоне — в одну.

## Почему нельзя просто уменьшить страницу

Если компьютерную версию сайта пропорционально уменьшить, текст и кнопки станут слишком мелкими.

Пользователю придётся:

* увеличивать страницу;
* прокручивать её по горизонтали;
* пытаться попасть пальцем по маленьким кнопкам;
* читать слишком мелкий текст.

Поэтому адаптивная верстка не просто уменьшает интерфейс, а перестраивает его под условия использования.

## Основные элементы адаптивности

Адаптивная верстка обычно строится на нескольких принципах.

### Гибкие размеры

Вместо строго фиксированной ширины часто используются относительные размеры.

Например:

```css
.container {
  width: 90%;
  max-width: 1200px;
}
```

Контейнер занимает 90% доступной ширины, но не становится шире 1200 пикселей.

### Гибкие изображения

Изображение не должно выходить за границы родительского блока:

```css
img {
  max-width: 100%;
  height: auto;
}
```

Так изображение уменьшается вместе с экраном и сохраняет пропорции.

### Перестроение элементов

Для расположения блоков часто используются Flexbox и Grid.

Например:

```css
.services {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}
```

Такой код создаёт три равные колонки.

На узком экране их можно заменить одной колонкой.

## Что такое media queries

**Media query** — это условие в CSS, которое применяет определённые стили только при подходящем размере экрана.

Пример:

```css
.services {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

@media (max-width: 768px) {
  .services {
    grid-template-columns: 1fr;
  }
}
```

Здесь:

* на широком экране отображаются три колонки;
* при ширине до 768 пикселей остаётся одна колонка.

Такие точки изменения интерфейса называют **breakpoints**, или контрольными точками.

## Mobile First

Один из распространённых подходов — **Mobile First**.

При нём сначала создают стили для небольшого экрана, а затем добавляют изменения для более широких устройств.

Пример:

```css
.services {
  display: grid;
  grid-template-columns: 1fr;
}

@media (min-width: 768px) {
  .services {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

Сначала интерфейс рассчитан на телефон. При достаточной ширине появляется несколько колонок.

Mobile First помогает начинать с самого ограниченного пространства и выделять действительно важные элементы.

## Как меняется навигация

На большом экране все пункты меню могут находиться в одной строке.

На смартфоне такое меню часто заменяют:

* кнопкой;
* выпадающим списком;
* боковой панелью;
* сокращённым набором пунктов.

CSS отвечает за расположение и видимость элементов, а JavaScript может управлять открытием и закрытием мобильного меню.

## Адаптивность и удобство

Страница может технически помещаться на экране, но оставаться неудобной.

При проверке адаптивности важно учитывать:

* легко ли читать текст;
* достаточно ли крупные кнопки;
* удобно ли нажимать элементы пальцем;
* не расположены ли кнопки слишком близко;
* помещаются ли формы;
* не обрезается ли важная информация;
* сохраняется ли понятная последовательность блоков.

Адаптивность связана не только с размерами, но и с удобством интерфейса.

## Как проверять адаптивность

Проверить страницу можно в инструментах разработчика браузера.

Обычно для этого:

1. открывают инструменты разработчика;
2. включают режим мобильного устройства;
3. выбирают разные размеры экранов;
4. уменьшают и увеличивают ширину страницы;
5. проверяют расположение элементов.

Однако эмуляция не всегда полностью заменяет проверку на реальном смартфоне. На устройстве могут иначе работать касания, клавиатура, прокрутка и некоторые элементы браузера.

## Частые проблемы

При адаптации страницы часто встречаются следующие ошибки:

* появляется горизонтальная прокрутка;
* текст выходит за границы блока;
* изображение шире экрана;
* кнопки становятся слишком маленькими;
* колонки не перестраиваются;
* меню перекрывает содержимое;
* фиксированная высота обрезает текст;
* элементы формы не помещаются по ширине.

Причиной часто становятся фиксированные размеры или стили, рассчитанные только на один экран.

## Что важно описывать ИИ-ассистенту

Запрос:

> Сделай сайт адаптивным.

слишком общий.

Полезнее указать:

* какие страницы или блоки нужно адаптировать;
* как они выглядят на широком экране;
* как должны перестраиваться на телефоне;
* какие элементы нужно скрыть или переместить;
* какие проблемы уже наблюдаются.

Например:

> Сделай блок услуг адаптивным. На широком экране должно быть три карточки в строке, на планшете — две, на смартфоне — одна. Карточки не должны выходить за ширину экрана.

Ещё более полезно приложить HTML и CSS нужного блока.

## Как проверять код от ИИ

После изменений проверьте:

* нет ли горизонтальной прокрутки;
* перестраиваются ли колонки;
* читается ли текст;
* помещаются ли кнопки и поля формы;
* не перекрываются ли элементы;
* работают ли меню и модальные окна;
* не сломалась ли компьютерная версия;
* корректно ли выглядит страница между основными контрольными точками.

Не стоит проверять только один смартфон и один компьютер. Проблема может появиться на промежуточной ширине.

## Главное

Адаптивная верстка позволяет одной странице работать на экранах разных размеров.

Основная логика выглядит так:

**размер экрана меняется → CSS применяет подходящие правила → элементы перестраиваются**

Для адаптивности используются:

* гибкие размеры;
* Flexbox и Grid;
* адаптивные изображения;
* media queries;
* контрольные точки.

При работе с ИИ важно описывать не просто требование «сделать адаптивно», а конкретное поведение каждого блока на разных экранах.

### Промты для самостоятельного уточнения

> Объясни этот media query простыми словами.

> Покажи, как перестроить три колонки в одну на мобильном экране.

> Проверь этот CSS и найди причину горизонтальной прокрутки.

> Предложи адаптивное поведение для этого блока на компьютере, планшете и смартфоне.

> Сделай этот HTML- и CSS-код адаптивным, не меняя внешний вид компьютерной версии.

> Проверь, какие фиксированные размеры мешают странице подстраиваться под экран.
