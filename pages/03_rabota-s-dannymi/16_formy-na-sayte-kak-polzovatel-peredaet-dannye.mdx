# Формы на сайте: как пользователь передаёт данные

## Коротко

Формы — один из главных способов, с помощью которых пользователь передаёт информацию веб-приложению.

Через форму человек может зарегистрироваться, войти в аккаунт, оформить заказ, записаться на услугу, отправить сообщение или настроить фильтры. Понимание форм помогает тебе точнее объяснять ИИ-ассистенту, какие данные должен вводить пользователь, как их нужно проверять и что должно происходить после отправки.

## Что такое форма

Форма — это часть интерфейса, предназначенная для ввода и отправки данных.

Она может содержать разные элементы:

* текстовые поля;
* поле для электронной почты;
* поле для пароля;
* выбор даты и времени;
* переключатели;
* флажки;
* выпадающие списки;
* загрузку файлов;
* кнопку отправки.

Например, форма записи в клинику может содержать имя клиента, номер телефона, выбранную услугу, специалиста, дату и время визита.

Сама форма только собирает информацию. Чтобы данные действительно использовались приложением, их обычно нужно проверить, отправить на сервер и обработать.

## Для чего используются формы

Формы позволяют пользователю не только просматривать информацию, но и взаимодействовать с системой.

С их помощью можно:

* зарегистрировать аккаунт;
* авторизоваться;
* оформить заказ;
* отправить заявку;
* оставить комментарий;
* записаться на услугу;
* выполнить поиск;
* настроить фильтры;
* изменить данные профиля;
* загрузить документ или изображение.

Даже строка поиска технически является простой формой: пользователь вводит запрос, а приложение получает его и показывает подходящие результаты.

## Из каких частей состоит форма

Обычно форма включает несколько основных элементов.

### Поля ввода

Это места, куда пользователь вводит или выбирает информацию.

Каждое поле должно соответствовать определённым данным. Например:

* имя;
* телефон;
* электронная почта;
* дата рождения;
* адрес;
* комментарий.

### Названия и подсказки

Пользователь должен понимать, что именно нужно указать в каждом поле.

Для этого используются подписи, примеры заполнения и короткие пояснения.

### Обязательные и необязательные поля

Некоторые данные нужны приложению обязательно, а другие пользователь может не указывать.

Например, при оформлении доставки адрес может быть обязательным, а комментарий курьеру — необязательным.

### Кнопка действия

Она запускает обработку формы.

Текст кнопки должен объяснять результат действия:

* «Войти»;
* «Зарегистрироваться»;
* «Отправить заявку»;
* «Оформить заказ»;
* «Сохранить изменения».

### Сообщения о результате

После отправки пользователь должен понять, что произошло.

Приложение может показать:

* сообщение об успешной отправке;
* предупреждение о незаполненном поле;
* описание ошибки;
* индикатор загрузки;
* подтверждение сохранения данных.

## Как форма работает в общей картине

Работа формы обычно проходит в несколько этапов.

1. Пользователь вводит данные.
2. Браузер сохраняет введённые значения во временном состоянии страницы.
3. Приложение проверяет, заполнены ли обязательные поля.
4. После нажатия кнопки данные отправляются на сервер.
5. Сервер повторно проверяет информацию.
6. Сервер выполняет нужное действие.
7. При необходимости данные сохраняются в базе данных.
8. Пользователь получает результат.

Например, при регистрации человек вводит имя, электронную почту и пароль. Браузер проверяет, что поля заполнены, после чего отправляет информацию серверу.

Сервер проверяет, существует ли уже пользователь с такой почтой, обрабатывает пароль, создаёт аккаунт в базе данных и возвращает ответ. После этого интерфейс сообщает об успешной регистрации или показывает ошибку.

## Какие данные форма передаёт

Форма передаёт значения, связанные с её полями.

Например, форма обратной связи может отправлять:

* имя пользователя;
* контактный адрес;
* тему обращения;
* текст сообщения;
* согласие на обработку данных.

Важно заранее определить, какие данные нужны приложению. Не следует добавлять поля только потому, что их можно добавить.

Каждое дополнительное поле усложняет заполнение формы и увеличивает объём информации, которую приложение должно хранить и защищать.

## Что происходит до и после отправки

До отправки формы приложение может:

* проверять заполнение полей;
* подсказывать правильный формат;
* блокировать кнопку до выполнения обязательных условий;
* показывать количество введённых символов;
* предварительно рассчитывать стоимость.

После отправки приложение может:

* создать новую запись;
* отправить письмо;
* оформить заказ;
* обновить профиль;
* перенаправить пользователя на другую страницу;
* показать подтверждение;
* сообщить об ошибке.

Поэтому при создании формы важно описывать не только внешний вид полей, но и весь сценарий взаимодействия.

## Формы и проверка данных

Пользователь может случайно или намеренно ввести неправильную информацию.

Например:

* оставить обязательное поле пустым;
* написать буквы вместо номера телефона;
* указать некорректный адрес электронной почты;
* загрузить неподдерживаемый файл;
* ввести слишком длинный текст.

Поэтому данные формы нужно проверять. Такая проверка называется валидацией.

Часть проверок выполняется в браузере, чтобы быстро подсказать пользователю об ошибке. Но основные проверки должны повторяться на сервере, потому что данным, поступающим из браузера, нельзя полностью доверять.

Подробно валидация будет рассмотрена в следующей статье.

## Как формы связаны с другими частями веб-приложения

Форма объединяет несколько уровней веб-разработки.

**HTML** задаёт поля, подписи и кнопки.

**CSS** отвечает за внешний вид формы и сообщений об ошибках.

**JavaScript** управляет поведением: получает введённые значения, запускает проверки и отправляет данные.

**API** передаёт информацию между браузером и сервером.

**Backend** обрабатывает данные и применяет правила приложения.

**База данных** сохраняет информацию, если она должна храниться постоянно.

Например, форма записи на услугу может выглядеть правильно в интерфейсе, но без сервера и базы данных запись не будет сохранена.

## Почему это важно при работе с ИИ-ассистентом

Запрос «Сделай форму заявки» слишком общий.

ИИ-ассистенту нужно объяснить:

* какие поля должны быть в форме;
* какие из них обязательные;
* какие данные может вводить пользователь;
* как нужно проверять значения;
* куда отправлять информацию;
* что происходит после успешной отправки;
* какие ошибки нужно показывать;
* должна ли форма очищаться после отправки.

Более точный запрос может выглядеть так:

> Создай форму записи на консультацию с полями «Имя», «Телефон», «Услуга», «Дата» и «Комментарий». Имя и телефон обязательны. После отправки данные должны передаваться на сервер, а пользователь должен увидеть сообщение об успешной записи.

Так ИИ понимает не только внешний вид формы, но и её назначение, данные и поведение.

## Простая аналогия

Форму можно сравнить с бумажной анкетой.

В анкете есть вопросы, поля для ответов и правила заполнения. После заполнения человек передаёт её сотруднику, который проверяет информацию и выполняет нужное действие.

В веб-приложении роль анкеты выполняет интерфейс, а роль сотрудника — сервер.

Если в анкете нет понятных подписей или неизвестно, кому её передавать, процесс не сработает. То же самое происходит и с плохо спроектированной веб-формой.

## Частые заблуждения новичков

### Форма — это только набор полей

Кроме полей, нужно продумать проверки, отправку, обработку ошибок и результат действия.

### Достаточно проверить данные в браузере

Проверка в браузере полезна для пользователя, но сервер должен самостоятельно проверять полученную информацию.

### Чем больше полей, тем лучше

Лишние поля снижают вероятность заполнения формы и заставляют приложение собирать ненужные данные.

### После нажатия кнопки данные автоматически сохраняются

Кнопка только запускает действие. Для сохранения данных нужна обработка на стороне приложения и, как правило, база данных.

### ИИ сам поймёт, как должна работать форма

Без подробного описания ИИ может выбрать неподходящие поля, правила проверки или способ отправки данных.

### Сообщения об ошибках не важны

Если пользователь не понимает, что именно заполнено неправильно, он может просто закрыть страницу.

## Что важно запомнить

* Форма позволяет пользователю передавать данные веб-приложению.
* Она состоит не только из полей, но и из подписей, кнопок, проверок и сообщений о результате.
* Перед созданием формы нужно определить, какие данные действительно необходимы.
* Данные обычно проходят через интерфейс, JavaScript, API, сервер и базу данных.
* Проверки должны выполняться не только в браузере, но и на сервере.
* После отправки пользователь должен получить понятный результат.
* При работе с ИИ важно описывать поля, правила заполнения, способ отправки и последующее действие.
* Человек определяет сценарий работы формы, а ИИ помогает реализовать его в коде.

## Промты для самостоятельного уточнения

* «Объясни простыми словами, как данные из веб-формы проходят от браузера до базы данных.»

* «Помоги определить, какие поля нужны для формы онлайн-записи на услугу и какие из них должны быть обязательными.»

* «Покажи простой сценарий работы формы регистрации без подробного разбора кода.»

* «Объясни, чем проверка формы в браузере отличается от проверки данных на сервере.»

* «Помоги составить требования к форме обратной связи: поля, проверки, отправка и сообщения для пользователя.»

* «Какие ошибки проектирования форм чаще всего допускают новички?»

* «Проверь моё описание формы и скажи, какие данные, состояния или действия я не учёл.»

* «Составь промт для ИИ-ассистента на создание формы заявки, которая отправляет данные на сервер и показывает результат пользователю.»
