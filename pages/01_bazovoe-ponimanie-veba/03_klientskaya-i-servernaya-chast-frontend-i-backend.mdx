# Клиентская и серверная часть: frontend и backend

## Коротко

Большинство современных веб-проектов состоит из двух основных частей: клиентской и серверной.

Клиентская часть работает в браузере пользователя и отвечает за то, что он видит и с чем взаимодействует. Серверная часть работает на сервере, обрабатывает запросы, применяет правила приложения и взаимодействует с базой данных.

Эти части выполняют разные задачи, но обычно работают как единая система.

## Что такое frontend

Frontend — это клиентская часть веб-проекта, которая работает в браузере пользователя.

К frontend относятся:

* страницы сайта;
* меню;
* кнопки;
* формы;
* карточки товаров;
* таблицы;
* изображения;
* уведомления;
* анимации;
* элементы личного кабинета.

Frontend отвечает не только за внешний вид. Он также реагирует на действия пользователя.

Например, клиентская часть может:

* открыть меню;
* переключить вкладку;
* проверить, заполнено ли обязательное поле;
* показать сообщение об ошибке;
* отправить данные формы на сервер;
* получить данные с сервера и отобразить их;
* обновить часть страницы без полной перезагрузки.

Основными технологиями frontend являются HTML, CSS и JavaScript. В современных проектах также часто используются frontend-фреймворки, например React, Vue или Angular.

## Что такое backend

Backend — это серверная часть веб-проекта.

Пользователь обычно не видит её напрямую. Он видит только результат её работы.

Backend может:

* принимать запросы от браузера;
* проверять данные;
* выполнять правила приложения;
* работать с базой данных;
* создавать и изменять записи;
* проверять логин и пароль;
* определять права пользователя;
* отправлять электронные письма;
* обращаться к платёжным и другим внешним сервисам;
* возвращать браузеру результат или сообщение об ошибке.

Например, когда пользователь отправляет форму записи на приём, backend может проверить выбранное время, убедиться, что оно свободно, сохранить запись в базе данных и вернуть подтверждение.

Для backend могут использоваться разные языки и технологии: JavaScript, Python, PHP, Java, C# и другие.

## Как frontend и backend взаимодействуют

Frontend и backend обмениваются запросами и ответами.

Пользователь выполняет действие в браузере. Frontend формирует запрос и отправляет его серверу. Backend получает запрос, выполняет необходимую работу и возвращает ответ. После этого frontend показывает результат пользователю.

Например, при входе в личный кабинет происходит следующая последовательность:

1. Пользователь вводит email и пароль.
2. Frontend отправляет эти данные на сервер.
3. Backend проверяет данные.
4. Сервер определяет, существует ли пользователь и верен ли пароль.
5. Backend возвращает успешный ответ или сообщение об ошибке.
6. Frontend открывает личный кабинет или показывает предупреждение.

Frontend не должен самостоятельно решать, действительно ли пароль правильный. Такая проверка должна происходить на сервере.

## Что обычно относится к frontend

К клиентской части обычно относятся задачи, связанные с интерфейсом и действиями пользователя:

* расположение элементов;
* цвета, шрифты и размеры;
* адаптация под телефон и компьютер;
* открытие окон и меню;
* отображение данных;
* предварительная проверка форм;
* состояние кнопок;
* переходы между страницами;
* реакция на действия пользователя.

Frontend отвечает на вопрос: **что видит пользователь и как он взаимодействует с интерфейсом?**

## Что обычно относится к backend

К серверной части обычно относятся:

* обработка запросов;
* работа с базой данных;
* регистрация и вход;
* проверка прав доступа;
* расчёты;
* бизнес-правила;
* обработка заказов;
* хранение данных;
* интеграции с другими сервисами;
* защита закрытой информации.

Backend отвечает на вопрос: **что должна сделать система после действия пользователя?**

## Как это работает в общей картине

Рассмотрим пример формы онлайн-записи.

Frontend показывает пользователю список специалистов, календарь, доступное время и поля для контактных данных.

Когда пользователь выбирает время и нажимает кнопку записи, frontend отправляет запрос на backend.

Backend:

* проверяет правильность данных;
* уточняет, свободно ли выбранное время;
* создаёт запись;
* сохраняет её в базе данных;
* при необходимости отправляет уведомление;
* возвращает ответ.

Frontend получает ответ и показывает сообщение об успешной записи или объясняет, почему действие выполнить не удалось.

База данных в этой системе хранит информацию, backend управляет ею, а frontend показывает пользователю удобный интерфейс.

## Может ли сайт работать без backend

Да. Простой информационный сайт может состоять только из клиентской части.

Например, сайт-визитка с текстом, фотографиями и контактами может загружать готовые страницы без собственной серверной логики.

Однако backend обычно требуется, если проект должен:

* хранить пользовательские данные;
* поддерживать регистрацию;
* принимать заказы;
* обрабатывать оплату;
* предоставлять личный кабинет;
* изменять данные;
* проверять права доступа;
* выполнять сложные расчёты.

Иногда сайт использует готовые внешние сервисы вместо собственного backend. Например, форма может отправлять данные через сторонний сервис.

## Почему нельзя переносить всё во frontend

Код frontend загружается в браузер пользователя. Его можно просмотреть, изменить или попытаться обойти.

Поэтому в клиентской части нельзя надёжно хранить:

* пароли;
* секретные ключи;
* закрытые настройки;
* правила доступа;
* критически важные проверки;
* конфиденциальную логику.

Например, нельзя доверять frontend окончательное решение о том, имеет ли пользователь право удалить запись или получить чужие данные.

Frontend может скрыть кнопку, но backend всё равно обязан проверить права при получении запроса.

## Почему нельзя переносить всё в backend

Backend может выполнить большую часть логики, но пользователю всё равно нужен понятный и удобный интерфейс.

Если каждое простое действие требует полной загрузки новой страницы, проект может работать медленнее и ощущаться менее удобным.

Frontend позволяет сразу реагировать на действия пользователя, обновлять отдельные элементы страницы и показывать промежуточные состояния.

Задача не в том, чтобы выбрать только frontend или backend, а в том, чтобы правильно распределить ответственность между ними.

## Почему это важно при работе с ИИ-ассистентом

При постановке задачи ИИ важно указать, какую часть проекта нужно изменить.

Например, фраза «Добавь авторизацию» слишком общая. Авторизация может затрагивать:

* форму входа во frontend;
* отправку запроса;
* проверку пользователя в backend;
* работу с базой данных;
* хранение состояния входа;
* ограничение доступа к страницам.

Если попросить изменить только интерфейс, ИИ может создать красивую форму, которая не выполняет реальный вход.

Если попросить создать только серверный маршрут, в проекте может не появиться форма, через которую пользователь сможет войти.

Полезно описывать задачу через полный сценарий:

* что делает пользователь;
* что происходит в браузере;
* какие данные отправляются на сервер;
* что должен проверить backend;
* что сохраняется в базе;
* какой ответ получает frontend;
* что видит пользователь в результате.

Так ИИ лучше понимает границы задачи и реже создаёт неполное решение.

## Простая аналогия

Frontend можно сравнить с залом ресторана, а backend — с кухней.

В зале посетитель видит меню, общается с официантом и получает заказ. Это интерфейс взаимодействия.

На кухне принимают заказ, проверяют продукты, готовят блюдо и следуют внутренним правилам. Посетитель не видит весь процесс, но получает его результат.

Красивый зал без работающей кухни не сможет обслужить посетителя. Хорошая кухня без понятного меню и обслуживания тоже не создаст удобный сервис.

## Частые заблуждения новичков

### Frontend — это только дизайн

Frontend включает не только внешний вид, но и поведение интерфейса, обработку действий и обмен данными с сервером.

### Backend — это база данных

База данных хранит информацию. Backend получает, проверяет и изменяет её по правилам приложения.

### Любую проверку можно выполнить в браузере

Frontend может проверять форму для удобства пользователя, но важные проверки должны повторяться на сервере.

### Пользователь не видит код frontend

Код клиентской части загружается в браузер, поэтому его нельзя считать полностью закрытым.

### Frontend и backend всегда находятся в разных проектах

Они могут храниться отдельно или внутри одного проекта. Это зависит от выбранной архитектуры.

### Для небольшого сайта обязательно нужен сложный backend

Простой информационный сайт может работать без собственной серверной части.

## Что важно запомнить

* Frontend работает в браузере и отвечает за интерфейс.
* Backend работает на сервере и обрабатывает данные и правила приложения.
* Frontend отправляет запросы, а backend возвращает ответы.
* База данных хранит информацию, но обычно управляет ею backend.
* Важные проверки безопасности должны выполняться на сервере.
* Один пользовательский сценарий может затрагивать сразу несколько частей проекта.
* При работе с ИИ нужно уточнять, требуется ли изменение frontend, backend или всей цепочки.
* Человек определяет логику и границы системы, а ИИ помогает реализовать отдельные части.

* «Помоги разделить функции моего проекта на клиентскую и серверную части.»

* «Покажи простую схему взаимодействия frontend, backend и базы данных без сложного кода.»

* «Объясни, какие проверки можно выполнять во frontend, а какие обязательно должны выполняться на сервере.»

* «Разбери по шагам, что происходит на frontend и backend при регистрации пользователя.»

* «Помоги определить, нужен ли моему сайту собственный backend или можно использовать готовые внешние сервисы.»

* «Покажи пример хорошего промта для ИИ на создание функции, которая затрагивает frontend, backend и базу данных.»

* «Проверь описание моей функции и скажи, какие части нужно реализовать на frontend, а какие — на backend.»
