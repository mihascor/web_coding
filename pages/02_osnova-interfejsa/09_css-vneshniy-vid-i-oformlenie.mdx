# 9. CSS: внешний вид и оформление

HTML определяет структуру страницы: заголовки, тексты, изображения, кнопки и формы.

**CSS** отвечает за то, как эти элементы выглядят и располагаются на экране.

CSS расшифровывается как **Cascading Style Sheets** — каскадные таблицы стилей.

С помощью CSS можно управлять:

* цветами;
* шрифтами;
* размерами элементов;
* отступами;
* рамками;
* фоном;
* расположением блоков;
* адаптацией под разные экраны.

Простая модель:

* **HTML** — что находится на странице;
* **CSS** — как это выглядит;
* **JavaScript** — как это ведёт себя.

## Как выглядит CSS-правило

CSS состоит из правил.

Например:

```css
h1 {
  color: darkblue;
  font-size: 32px;
}
```

Здесь:

* `h1` — элемент, который нужно оформить;
* `color` — свойство;
* `darkblue` — значение свойства;
* `font-size` — размер текста;
* `32px` — значение размера.

Такое правило означает: все заголовки `h1` должны быть тёмно-синими и иметь размер 32 пикселя.

Общая структура CSS:

```css
селектор {
  свойство: значение;
}
```

## Что такое селектор

**Селектор** указывает, к каким HTML-элементам применяются стили.

Можно выбрать элемент по названию тега:

```css
p {
  color: gray;
}
```

Это правило изменит все абзацы `p`.

Чаще элементы оформляют с помощью классов.

HTML:

```html
<button class="booking-button">
  Записаться
</button>
```

CSS:

```css
.booking-button {
  background-color: black;
  color: white;
}
```

Точка перед названием означает, что используется CSS-класс.

Классы помогают применять одинаковое оформление к нескольким элементам и давать отдельным блокам понятные названия.

## Основные CSS-свойства

Для текста часто используются:

```css
.title {
  color: #222222;
  font-size: 36px;
  font-weight: 700;
  text-align: center;
}
```

Для фона и рамки:

```css
.card {
  background-color: white;
  border: 1px solid #dddddd;
  border-radius: 12px;
}
```

Для размеров:

```css
.card {
  width: 300px;
  min-height: 200px;
}
```

Для расстояний:

```css
.card {
  margin: 20px;
  padding: 24px;
}
```

Необязательно сразу запоминать все свойства. Важно понимать, что каждое правило отвечает за конкретную характеристику элемента.

## Отступы: margin и padding

У элементов есть два основных вида отступов.

**Padding** — внутреннее расстояние между содержимым и границей элемента.

**Margin** — внешнее расстояние между элементом и соседними блоками.

Например:

```css
.card {
  padding: 24px;
  margin-bottom: 20px;
}
```

Здесь:

* содержимое карточки отступает от её краёв на 24 пикселя;
* после карточки остаётся внешнее расстояние 20 пикселей.

Путаница между `margin` и `padding` часто становится причиной неправильного расположения элементов.

## Блочная модель

Браузер воспринимает каждый элемент как прямоугольный блок.

Он состоит из:

1. содержимого;
2. внутреннего отступа;
3. рамки;
4. внешнего отступа.

Эта структура называется **блочной моделью**, или **box model**.

Например, фактическая ширина блока может зависеть не только от свойства `width`, но также от внутренних отступов и рамок.

Чтобы расчёт размеров был предсказуемее, часто используют:

```css
* {
  box-sizing: border-box;
}
```

При таком правиле заданная ширина включает содержимое, внутренние отступы и рамку.

## Как CSS располагает элементы

По умолчанию HTML-элементы располагаются по обычному порядку документа.

Для более сложного расположения часто используются **Flexbox** и **Grid**.

Пример с Flexbox:

```css
.header {
  display: flex;
  justify-content: space-between;
  align-items: center;
}
```

Так можно расположить элементы шапки в одной строке и распределить свободное пространство между ними.

Пример с Grid:

```css
.services {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}
```

Так создаётся сетка из трёх равных колонок с расстоянием между карточками.

## Где хранится CSS

CSS можно подключить к HTML отдельным файлом:

```html
<link rel="stylesheet" href="styles.css">
```

Обычно это предпочтительный способ, потому что оформление хранится отдельно от структуры страницы.

Пример структуры проекта:

```text
project
├── index.html
└── styles.css
```

CSS также можно записать внутри HTML, но в реальных проектах отдельные файлы обычно удобнее поддерживать.

## Что означает каскадность

Слово «каскадные» означает, что на один элемент могут одновременно влиять несколько CSS-правил.

Например:

```css
button {
  color: black;
}

.booking-button {
  color: white;
}
```

Оба правила относятся к кнопке, но правило класса обычно имеет больший приоритет, поэтому текст будет белым.

На итоговый стиль влияют:

* порядок правил;
* тип селектора;
* вложенность;
* наследование;
* приоритет отдельных правил.

Новичку не обязательно сразу изучать все правила приоритетов. Достаточно понимать: если стиль не применяется, возможно, его переопределяет другое CSS-правило.

## Псевдоклассы и состояния

CSS может менять оформление элемента в определённом состоянии.

Например:

```css
.booking-button:hover {
  opacity: 0.8;
}
```

Правило `:hover` применяется, когда пользователь наводит курсор на кнопку.

Другие распространённые состояния:

* `:focus` — элемент получил фокус;
* `:active` — элемент нажат;
* `:disabled` — элемент отключён;
* `:checked` — переключатель выбран.

Такие состояния помогают показать пользователю, что происходит с элементом.

## CSS и адаптивность

CSS используется для настройки интерфейса под разные размеры экранов.

Например:

```css
.services {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

@media (max-width: 768px) {
  .services {
    grid-template-columns: 1fr;
  }
}
```

На широком экране карточки расположены в три колонки, а на узком — в одну.

Подробно адаптивная верстка рассматривается в отдельной теме.

## Как посмотреть CSS в браузере

CSS можно изучать через инструменты разработчика.

Для этого обычно нужно:

1. нажать правой кнопкой мыши на элемент;
2. выбрать «Просмотреть код» или «Исследовать»;
3. открыть вкладку Elements;
4. посмотреть раздел Styles.

Там можно:

* увидеть применённые правила;
* найти переопределённые свойства;
* временно изменить цвет или размер;
* отключить отдельное правило;
* определить CSS-класс элемента.

Такие изменения исчезнут после перезагрузки страницы, если не сохранить их в файлах проекта.

## Частые проблемы с CSS

При работе со стилями часто встречаются следующие ошибки:

* класс в HTML не совпадает с классом в CSS;
* правило переопределяется другим стилем;
* отсутствует точка перед названием класса;
* забыта точка с запятой;
* элемент имеет фиксированную ширину и выходит за экран;
* отступ задан не тому элементу;
* изменён общий селектор вместо конкретного класса;
* CSS-файл не подключён к HTML.

Если оформление не изменилось, сначала нужно проверить, находит ли CSS нужный элемент.

## Что важно описывать ИИ-ассистенту

Запрос:

> Сделай блок красивым.

слишком субъективный.

Полезнее указать:

* какой элемент нужно изменить;
* желаемые цвета;
* размеры и отступы;
* расположение элементов;
* стиль кнопок и карточек;
* поведение на разных экранах;
* что нельзя менять.

Например:

> Оформи карточки услуг. На компьютере расположи их в три колонки с расстоянием 24 пикселя. Добавь светлый фон, скругление 12 пикселей и внутренний отступ 24 пикселя. HTML-структуру не изменяй.

Такой запрос даёт ИИ конкретные параметры и ограничения.

## Как проверять CSS-код от ИИ

После изменений проверьте:

* применились ли стили к нужным элементам;
* не изменились ли другие части страницы;
* читается ли текст на выбранном фоне;
* одинаковы ли отступы у похожих элементов;
* не выходят ли блоки за границы экрана;
* сохраняется ли понятная структура страницы;
* корректно ли выглядит интерфейс на телефоне;
* не появилось ли слишком много повторяющихся правил.

Если результат отличается от ожидаемого, полезно передать ИИ HTML, CSS и снимок текущего вида страницы.

## Главное

CSS отвечает за внешний вид и расположение элементов страницы.

Базовая модель работы выглядит так:

**CSS выбирает HTML-элемент → применяет свойства → браузер отображает оформление**

Для осознанной работы с CSS важно понимать:

* что такое селектор;
* чем свойство отличается от значения;
* как работают классы;
* чем внутренние отступы отличаются от внешних;
* почему одно правило может переопределять другое;
* как проверить стиль через инструменты разработчика.

### Промты для самостоятельного уточнения

> Объясни этот CSS-код построчно простыми словами.

> Найди, какое правило отвечает за цвет, размер и отступы этого элемента.

> Проверь, почему CSS-класс не применяется к элементу.

> Объясни разницу между `margin` и `padding` на простом примере.

> Сделай оформление этого блока аккуратнее, не меняя HTML.

> Найди повторяющиеся или конфликтующие CSS-правила.

> Объясни, почему один CSS-стиль переопределяет другой.
