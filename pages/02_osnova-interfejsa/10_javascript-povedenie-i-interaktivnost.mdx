# 10. JavaScript: поведение и интерактивность

HTML определяет, **что находится на странице**, CSS отвечает за **внешний вид**, а JavaScript управляет **поведением интерфейса**.

Благодаря JavaScript сайт может реагировать на действия пользователя, изменять содержимое страницы, проверять введённые данные и получать информацию с сервера без полной перезагрузки.

Простая аналогия:

* **HTML** — конструкция здания;
* **CSS** — его оформление;
* **JavaScript** — электричество, двери, лифты и другие работающие механизмы.

## Что делает JavaScript на сайте

JavaScript используется, когда страница должна не просто отображать информацию, а выполнять какие-либо действия.

Например, он может:

* открыть или закрыть меню;
* переключить вкладку;
* показать всплывающее окно;
* изменить изображение в галерее;
* проверить заполнение формы;
* добавить товар в корзину;
* пересчитать итоговую стоимость;
* загрузить новые данные с сервера;
* показать сообщение об успешном действии или ошибке.

Без JavaScript кнопка может присутствовать на странице и выглядеть активной, но сама по себе она ничего не сделает. JavaScript описывает, **что должно произойти после нажатия**.

## Как работает JavaScript

Обычно работа JavaScript строится по простой схеме:

1. На странице происходит событие.
2. JavaScript замечает его.
3. Выполняется заданное действие.
4. Интерфейс или данные изменяются.

Событием может быть:

* нажатие на кнопку;
* ввод текста;
* отправка формы;
* выбор пункта;
* прокрутка страницы;
* наведение курсора;
* загрузка страницы.

Например, на странице есть кнопка:

```html
<button id="show-message">Показать сообщение</button>
```

JavaScript может добавить ей поведение:

```javascript
const button = document.querySelector('#show-message');

button.addEventListener('click', function () {
  alert('Кнопка нажата');
});
```

Здесь код:

1. находит кнопку на странице;
2. ожидает нажатия;
3. показывает сообщение.

Необязательно запоминать синтаксис. Важно понимать общую логику: **найти элемент → дождаться события → выполнить действие**.

## JavaScript изменяет страницу

JavaScript может менять элементы уже после того, как страница открылась:

* заменять текст;
* добавлять или удалять блоки;
* менять классы оформления;
* скрывать и показывать элементы;
* обновлять список товаров;
* выводить результаты поиска.

Для этого JavaScript взаимодействует с представлением страницы внутри браузера — DOM. DOM будет рассмотрен отдельно в следующей теме.

Например, после нажатия кнопки JavaScript может изменить текст:

```javascript
const status = document.querySelector('#status');

status.textContent = 'Заявка отправлена';
```

Пользователь увидит новый текст без перезагрузки всей страницы.

## JavaScript и данные

JavaScript работает не только с кнопками и анимациями. Он также управляет данными, которые используются интерфейсом.

Например, при заполнении формы JavaScript может:

1. получить введённое имя и номер телефона;
2. проверить, заполнены ли обязательные поля;
3. отправить данные на сервер;
4. получить результат;
5. показать пользователю сообщение.

Важно понимать: проверка в браузере делает интерфейс удобнее, но не заменяет проверку на сервере. Пользовательский ввод должен проверяться и во frontend, и в backend.

## Где выполняется JavaScript

Чаще всего JavaScript выполняется прямо в браузере пользователя. Такой код относится к frontend.

Последовательность выглядит так:

1. браузер получает файлы сайта;
2. создаёт структуру страницы из HTML;
3. применяет стили CSS;
4. запускает JavaScript;
5. начинает реагировать на действия пользователя.

JavaScript также может выполняться на сервере, например с помощью Node.js. В этом случае он используется уже для backend-задач: обработки запросов, работы с базой данных и выполнения бизнес-логики.

Таким образом, один язык может применяться и в клиентской, и в серверной части проекта.

## JavaScript, библиотека и фреймворк — не одно и то же

JavaScript — это язык программирования.

React, Vue и другие frontend-инструменты работают на основе JavaScript и помогают создавать сложные интерфейсы. Они не заменяют сам язык, а предоставляют дополнительные правила и готовые возможности.

Поэтому фраза «проект написан на React» обычно означает, что в нём используется JavaScript или TypeScript вместе с библиотекой React.

## Что важно описывать ИИ-ассистенту

При работе с ИИ недостаточно написать:

> Добавь красивую кнопку.

Это описание относится в основном к внешнему виду. Чтобы ИИ создал поведение, нужно указать:

* что запускает действие;
* что должно произойти;
* при каких условиях;
* что увидит пользователь;
* что делать при ошибке.

Более точный запрос:

> Добавь кнопку «Записаться». После нажатия открой модальное окно с формой. Если обязательные поля не заполнены, покажи ошибки под соответствующими полями. После успешной отправки закрой форму и покажи сообщение «Заявка отправлена».

Такое описание содержит не только элемент интерфейса, но и его JavaScript-логику.

## Как проверять JavaScript-код от ИИ

После добавления интерактивности проверьте:

* происходит ли действие после нужного события;
* работает ли функция при повторном использовании;
* что происходит при пустых или неправильных данных;
* появляется ли понятное сообщение об ошибке;
* не перестали ли работать другие элементы страницы;
* сохраняется ли корректное поведение на мобильном устройстве.

Если функция не работает, полезно сообщить ИИ:

* что вы сделали;
* какой результат ожидали;
* что произошло на самом деле;
* есть ли ошибка в консоли браузера;
* какой код связан с проблемой.

## Главное

JavaScript отвечает за действия, реакции и изменения на странице.

Базовая модель работы выглядит так:

**событие → JavaScript-код → изменение интерфейса или данных**

Для осознанной работы с ИИ необязательно сразу знать весь синтаксис языка. Важнее уметь точно описывать поведение:

* что делает пользователь;
* как должна отреагировать система;
* какие существуют условия;
* каким должен быть успешный результат;
* как обрабатываются ошибки.
