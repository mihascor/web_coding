# API, JSON, fetch

## API, JSON и `fetch`: как это связано в веб-разработке

Когда сайт или приложение хочет получить данные не “вшитые” прямо в страницу, а взять их откуда-то с сервера, чаще всего используются **API**, формат данных **JSON** и JavaScript-инструмент **`fetch`**.

Например, сайт погоды показывает температуру не потому, что разработчик каждый час вручную меняет текст на странице. Сайт обращается к серверу погоды, получает свежие данные и показывает их пользователю.

Упрощённая схема такая:

```text
Браузер → запрос к API → сервер → ответ в JSON → браузер показывает данные
```

---

# 1. Что такое API

**API** — это способ, с помощью которого одна программа может обращаться к другой программе и получать от неё данные или выполнять действия.

Расшифровка: **Application Programming Interface**, то есть “интерфейс программирования приложений”.

Проще говоря, API — это **набор правил и адресов**, по которым можно обратиться к системе.

## Простая аналогия

Представь ресторан.

Ты не заходишь на кухню и не готовишь еду сам. Ты говоришь официанту:

> “Принесите мне кофе и салат”.

Официант передаёт заказ на кухню, кухня готовит, официант приносит результат.

В веб-разработке:

| Роль | В аналогии | В вебе |
|---|---|---|
| Пользователь | Гость ресторана | Человек на сайте |
| Сайт | Столик / меню | Интерфейс в браузере |
| API | Официант | Правила общения с сервером |
| Сервер | Кухня | Место, где хранятся и обрабатываются данные |
| Ответ | Блюдо | Данные, которые вернулись |

---

## Зачем нужен API

API позволяет сайту:

- получать список товаров из интернет-магазина;
- показывать профиль пользователя;
- отправлять форму заявки;
- загружать комментарии;
- получать прогноз погоды;
- авторизовать пользователя;
- добавлять товар в корзину;
- отправлять сообщение в чат.

То есть API нужен, когда сайт должен **обмениваться данными с сервером**.

---

## Пример API-адреса

Допустим, у интернет-магазина есть API:

```text
https://example-shop.com/api/products
```

Если сайт отправит запрос по этому адресу, сервер может вернуть список товаров.

Например:

```json
[
  {
    "id": 1,
    "title": "Крем для лица",
    "price": 2500
  },
  {
    "id": 2,
    "title": "Сыворотка",
    "price": 3900
  }
]
```

Это уже не HTML-страница, а данные, которые сайт может использовать.

# 2. Что такое endpoint

В контексте API часто встречается слово **endpoint**.

**Endpoint** — это конкретный адрес внутри API, к которому можно обратиться.

Например:

```text
/api/products
/api/users
/api/orders
/api/login
```

Каждый endpoint обычно отвечает за свою задачу.

| Endpoint | Что может делать |
|---|---|
| `/api/products` | Получить список товаров |
| `/api/products/15` | Получить товар с ID 15 |
| `/api/users` | Получить пользователей |
| `/api/login` | Отправить логин и пароль |
| `/api/orders` | Создать заказ |

Можно думать об endpoint как о “двери” на сервере. В одну дверь заходят за товарами, в другую — за пользователями, в третью — для авторизации.

# 3. Что такое JSON

**JSON** — это текстовый формат для передачи данных.

Расшифровка: **JavaScript Object Notation**.

Он похож на JavaScript-объекты, но используется как универсальный формат данных. Его понимают разные языки программирования: JavaScript, Python, PHP, Java, Ruby и многие другие.

---

## Пример JSON

```json
{
  "name": "Anna",
  "age": 28,
  "isClient": true
}
```

Здесь есть данные о человеке:

| Поле | Значение |
|---|---|
| `name` | `"Anna"` |
| `age` | `28` |
| `isClient` | `true` |

---

## JSON похож на объект JavaScript

В JavaScript объект может выглядеть так:

```js
const user = {
  name: "Anna",
  age: 28,
  isClient: true
};
```

JSON выглядит почти так же:

```json
{
  "name": "Anna",
  "age": 28,
  "isClient": true
}
```

Главное отличие: в JSON ключи обязательно пишутся в двойных кавычках.

Правильно:

```json
{
  "name": "Anna"
}
```

Неправильно для JSON:

```json
{
  name: "Anna"
}
```

---

## Какие данные бывают в JSON

JSON может хранить:

### Строки

```json
{
  "title": "Крем для лица"
}
```

### Числа

```json
{
  "price": 2500
}
```

### Логические значения

```json
{
  "inStock": true
}
```

### Массивы

```json
{
  "categories": ["уход", "косметика", "лицо"]
}
```

### Вложенные объекты

```json
{
  "user": {
    "name": "Anna",
    "city": "Moscow"
  }
}
```

### Массив объектов

```json
{
  "products": [
    {
      "id": 1,
      "title": "Крем",
      "price": 2500
    },
    {
      "id": 2,
      "title": "Сыворотка",
      "price": 3900
    }
  ]
}
```

# 4. Зачем JSON нужен в вебе

JSON часто используется как “язык общения” между браузером и сервером.

Например:

1. Пользователь открывает страницу интернет-магазина.
2. JavaScript делает запрос к API.
3. Сервер возвращает данные в JSON.
4. JavaScript превращает эти данные в карточки товаров на странице.

Сервер может вернуть такой JSON:

```json
{
  "products": [
    {
      "id": 1,
      "title": "Крем для лица",
      "price": 2500
    },
    {
      "id": 2,
      "title": "Сыворотка",
      "price": 3900
    }
  ]
}
```

А сайт уже сам решает, как это красиво показать пользователю.

# 5. Что такое `fetch`

**`fetch`** — это встроенный в JavaScript способ отправлять HTTP-запросы из браузера.

С его помощью можно обратиться к API и получить данные.

Например:

```js
fetch("https://example.com/api/products")
```

Эта строка отправляет запрос на сервер.

Но одного запроса мало. Нужно ещё дождаться ответа, преобразовать данные и использовать их.

---

## Простой пример `fetch`

```js
fetch("https://example.com/api/products")
  .then(response => response.json())
  .then(data => {
    console.log(data);
  });
```

Что здесь происходит:

```js
fetch("https://example.com/api/products")
```

Отправляем запрос к API.

```js
.then(response => response.json())
```

Когда сервер ответил, преобразуем ответ из JSON в обычные JavaScript-данные.

```js
.then(data => {
  console.log(data);
});
```

Получаем данные и выводим их в консоль.

---

# 6. `fetch` с `async/await`

В современной разработке часто используют `async/await`. Такой код обычно читается проще.

```js
async function loadProducts() {
  const response = await fetch("https://example.com/api/products");
  const data = await response.json();

  console.log(data);
}

loadProducts();
```

Здесь:

- `async` говорит, что функция работает с асинхронными действиями;
- `await` говорит: “подожди, пока операция завершится”;
- `fetch(...)` отправляет запрос;
- `response.json()` превращает JSON-ответ в данные JavaScript.

---

# 7. Почему `fetch` работает асинхронно

Запрос к серверу занимает время.

Сервер может ответить быстро, а может медленно. Интернет может быть нестабильным. Поэтому JavaScript не должен “замораживать” всю страницу, пока ждёт ответ.

Асинхронность позволяет странице продолжать работать.

Например, пока загружаются товары:

- пользователь может видеть спиннер загрузки;
- меню остаётся кликабельным;
- страница не зависает.

# 8. Основные HTTP-методы

Когда сайт обращается к API, он обычно использует HTTP-методы. Метод показывает, что именно мы хотим сделать.

| Метод | Для чего используется | Пример |
|---|---|---|
| `GET` | Получить данные | Получить список товаров |
| `POST` | Отправить новые данные | Создать заявку |
| `PUT` | Полностью обновить данные | Обновить профиль |
| `PATCH` | Частично обновить данные | Изменить только имя |
| `DELETE` | Удалить данные | Удалить товар из корзины |

---

## Пример `GET`

`GET` используется для получения данных.

```js
async function getProducts() {
  const response = await fetch("https://example.com/api/products");
  const products = await response.json();

  console.log(products);
}
```

Здесь мы просто получаем данные.

---

## Пример `POST`

`POST` используется, когда мы хотим отправить данные на сервер.

Например, отправить форму заявки:

```js
async function sendRequest() {
  const response = await fetch("https://example.com/api/requests", {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      name: "Anna",
      phone: "+79990000000",
      service: "Consultation"
    })
  });

  const result = await response.json();

  console.log(result);
}
```

Что здесь важно:

```js
method: "POST"
```

Говорим, что отправляем данные.

```js
headers: {
  "Content-Type": "application/json"
}
```

Сообщаем серверу: “я отправляю JSON”.

```js
body: JSON.stringify(...)
```

Преобразуем JavaScript-объект в JSON-строку.

---

# 9. `JSON.stringify` и `response.json`

Важно не путать две вещи.

## `JSON.stringify`

Преобразует JavaScript-объект в JSON-строку.

```js
const user = {
  name: "Anna",
  age: 28
};

const json = JSON.stringify(user);

console.log(json);
```

Результат:

```json
{"name":"Anna","age":28}
```

Это нужно, когда мы **отправляем данные на сервер**.

---

## `response.json()`

Преобразует JSON-ответ сервера в JavaScript-данные.

```js
const response = await fetch("/api/user");
const user = await response.json();

console.log(user.name);
```

Это нужно, когда мы **получаем данные от сервера**.

---

# 10. Обработка ошибок

В реальной разработке важно учитывать, что запрос может завершиться ошибкой.

Например:

- сервер недоступен;
- адрес API неправильный;
- пользователь не авторизован;
- данных не найдено;
- интернет пропал.

Простой пример:

```js
async function loadUser() {
  try {
    const response = await fetch("https://example.com/api/user");

    if (!response.ok) {
      throw new Error("Ошибка при загрузке данных");
    }

    const user = await response.json();
    console.log(user);
  } catch (error) {
    console.error("Что-то пошло не так:", error);
  }
}
```

Здесь:

```js
try
```

Пробуем выполнить запрос.

```js
if (!response.ok)
```

Проверяем, успешный ли ответ.

```js
catch
```

Ловим ошибку, если что-то пошло не так.

# 11. Частые HTTP-статусы

Сервер отвечает не только данными, но и статусом.

| Статус | Значение |
|---|---|
| `200` | Всё хорошо |
| `201` | Создано |
| `400` | Ошибка в запросе |
| `401` | Не авторизован |
| `403` | Доступ запрещён |
| `404` | Не найдено |
| `500` | Ошибка сервера |

Например, если ты запрашиваешь товар с ID 999, которого нет, сервер может вернуть `404`.

# 12. Как данные попадают на страницу

Допустим, API вернул товары:

```js
const products = [
  {
    id: 1,
    title: "Крем для лица",
    price: 2500
  },
  {
    id: 2,
    title: "Сыворотка",
    price: 3900
  }
];
```

JavaScript может создать HTML:

```js
const container = document.querySelector(".products");

products.forEach(product => {
  const card = document.createElement("div");

  card.innerHTML = `
    <h3>${product.title}</h3>
    <p>${product.price} ₽</p>
  `;

  container.appendChild(card);
});
```

И на странице появятся карточки товаров.

То есть данные приходят отдельно, а внешний вид создаётся уже на стороне сайта.

# 14. API не всегда открытое

Некоторые API можно использовать свободно. Например, публичные API с погодой, курсами валют, тестовыми данными.

Но многие API закрытые.

Для доступа может понадобиться:

- логин и пароль;
- токен доступа;
- API key;
- права пользователя;
- оплата тарифа.

Например, админ клиники и обычный клиент могут обращаться к одному сайту, но видеть разные данные, потому что у них разные права.

# 15. Что такое токен

**Токен** — это специальная строка, которая подтверждает, что пользователь или приложение имеет право обращаться к API.

Примерно как временный пропуск.

Упрощённо:

1. Пользователь вводит логин и пароль.
2. Сервер проверяет их.
3. Сервер выдаёт токен.
4. Браузер использует токен в следующих запросах.

Пример заголовка с токеном:

```js
const response = await fetch("https://example.com/api/profile", {
  headers: {
    "Authorization": "Bearer тут_будет_токен"
  }
});
```

Так сервер понимает, кто именно делает запрос.

---

# 16. Что такое REST API

Часто можно встретить выражение **REST API**.

REST API — это популярный стиль организации API, где данные представлены как ресурсы.

Например:

```text
GET /products
GET /products/15
POST /products
PATCH /products/15
DELETE /products/15
```

Здесь `products` — ресурс.

А HTTP-методы показывают действие:

| Запрос | Значение |
|---|---|
| `GET /products` | Получить все товары |
| `GET /products/15` | Получить товар 15 |
| `POST /products` | Создать товар |
| `PATCH /products/15` | Изменить товар 15 |
| `DELETE /products/15` | Удалить товар 15 |

---

# 17. Что такое CRUD

Рядом с API часто встречается термин **CRUD**.

CRUD — это четыре базовых действия с данными:

| Буква | Значение | HTTP-метод |
|---|---|---|
| C | Create — создать | `POST` |
| R | Read — прочитать | `GET` |
| U | Update — обновить | `PUT` / `PATCH` |
| D | Delete — удалить | `DELETE` |

Например, в админке услуг клиники:

- создать услугу;
- посмотреть список услуг;
- изменить цену;
- удалить устаревшую услугу.

Это и есть CRUD.

# 18. Что важно понимать новичку

На старте достаточно понять главное:

**API** — это способ общения сайта с сервером.

**JSON** — это формат, в котором часто передаются данные.

**`fetch`** — это JavaScript-инструмент, с помощью которого браузер делает запросы к API.

Связка выглядит так:

```text
JavaScript использует fetch
        ↓
делает запрос к API
        ↓
сервер возвращает JSON
        ↓
JavaScript показывает данные на странице
```

# 19. Мини-пример целиком

HTML:

```html
<button id="loadBtn">Загрузить пользователя</button>
<div id="user"></div>
```

JavaScript:

```js
const button = document.querySelector("#loadBtn");
const userBlock = document.querySelector("#user");

button.addEventListener("click", async () => {
  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/users/1");

    if (!response.ok) {
      throw new Error("Не удалось загрузить пользователя");
    }

    const user = await response.json();

    userBlock.innerHTML = `
      <h2>${user.name}</h2>
      <p>Email: ${user.email}</p>
      <p>Город: ${user.address.city}</p>
    `;
  } catch (error) {
    userBlock.textContent = "Ошибка загрузки данных";
    console.error(error);
  }
});
```

Что делает этот пример:

1. На странице есть кнопка.
2. При клике JavaScript делает запрос к API.
3. API возвращает данные пользователя.
4. JavaScript вставляет имя, email и город в HTML.
5. Если что-то пошло не так, показывается ошибка.

# 20. Частые ошибки новичков

## Забыли `await`

Неправильно:

```js
const response = fetch("/api/products");
const data = response.json();
```

Правильно:

```js
const response = await fetch("/api/products");
const data = await response.json();
```

---

## Забыли преобразовать ответ в JSON

Неправильно:

```js
const response = await fetch("/api/products");
console.log(response);
```

Так ты увидишь объект ответа, но не сами данные.

Правильно:

```js
const response = await fetch("/api/products");
const data = await response.json();
console.log(data);
```

---

## Путают объект и JSON-строку

JavaScript-объект:

```js
const user = {
  name: "Anna"
};
```

JSON-строка:

```js
const json = JSON.stringify(user);
```

Они похожи, но это не одно и то же.

---

## Не проверяют ошибки

Новички часто сразу пишут:

```js
const response = await fetch("/api/products");
const data = await response.json();
```

Лучше добавить проверку:

```js
if (!response.ok) {
  throw new Error("Ошибка запроса");
}
```

# 21. Коротко

| Термин | Что означает |
|---|---|
| API | Способ общения сайта с сервером |
| Endpoint | Конкретный адрес внутри API |
| JSON | Формат передачи данных |
| `fetch` | JavaScript-функция для запросов к API |
| `GET` | Получить данные |
| `POST` | Отправить новые данные |
| `PUT` | Полностью обновить данные |
| `PATCH` | Частично обновить данные |
| `DELETE` | Удалить данные |

Главная мысль:

> API — это “куда обращаться”, JSON — это “в каком виде приходят данные”, а `fetch` — это “как обратиться из JavaScript”.
