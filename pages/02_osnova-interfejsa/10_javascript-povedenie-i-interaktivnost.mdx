# JavaScript: поведение и интерактивность

## Коротко

JavaScript отвечает за поведение веб-страницы. С его помощью сайт реагирует на действия пользователя, изменяет содержимое без полной перезагрузки, проверяет формы, открывает меню, загружает данные с сервера и обновляет интерфейс.

HTML создаёт структуру страницы, CSS отвечает за её внешний вид, а JavaScript добавляет действия и правила.

Например:

* HTML создаёт кнопку;
* CSS определяет её цвет и размер;
* JavaScript решает, что произойдёт после нажатия.

Понимание JavaScript особенно важно при работе с ИИ-ассистентом. Оно помогает не просто просить «оживить страницу», а точно описывать события, ожидаемое поведение, данные, состояния интерфейса и возможные ошибки.

## Что это такое

JavaScript — это язык программирования, который широко используется в веб-разработке.

В отличие от HTML и CSS, JavaScript может выполнять команды, проверять условия, работать с данными и принимать решения.

Например, с его помощью можно задать правило:

> Если пользователь нажал кнопку, открой форму записи.

Или:

> Если поле телефона не заполнено, покажи сообщение об ошибке и не отправляй форму.

Или:

> Получи список услуг с сервера и покажи его на странице.

JavaScript чаще всего выполняется в браузере пользователя. Браузеры содержат специальный механизм, который читает и запускает JavaScript-код.

Но JavaScript может использоваться и на сервере. Например, среда Node.js позволяет создавать на JavaScript серверную логику, API и другие части backend.

В этой статье основной фокус — JavaScript в браузере, который управляет интерфейсом страницы.

## Для чего это используется

JavaScript используется, когда страница должна не только показывать информацию, но и реагировать на действия пользователя или изменения данных.

С его помощью можно реализовать:

* открытие и закрытие меню;
* переключение вкладок;
* раскрывающиеся ответы на вопросы;
* слайдеры и галереи;
* всплывающие окна;
* проверку полей формы;
* отправку формы без перезагрузки страницы;
* поиск и фильтрацию;
* загрузку новых данных;
* обновление корзины;
* переключение темы оформления;
* отображение уведомлений;
* выбор даты и времени;
* работу личного кабинета;
* изменение интерфейса в зависимости от состояния пользователя.

Например, на странице записи в клинику JavaScript может:

1. Отследить выбор услуги.
2. Загрузить доступных специалистов.
3. Показать свободные даты.
4. Проверить заполнение контактных данных.
5. Отправить информацию на сервер.
6. Показать сообщение об успешной записи.

Без JavaScript такая страница могла бы оставаться обычным документом с текстом и ссылками либо требовала бы полной перезагрузки после каждого действия.

## JavaScript и интерактивность

**Интерактивность** — это способность интерфейса реагировать на пользователя.

Пользователь может:

* нажать кнопку;
* ввести текст;
* выбрать пункт;
* отправить форму;
* навести курсор;
* прокрутить страницу;
* переключить вкладку;
* загрузить файл.

Такие действия называются **событиями**.

JavaScript может отслеживать событие и выполнять нужную команду.

Простой пример:

```javascript
button.addEventListener('click', () => {
  form.classList.add('is-open');
});
```

Этот код условно означает:

> Когда пользователь нажмёт на кнопку, добавь форме класс, который показывает её на странице.

Для общего понимания необязательно подробно разбирать синтаксис. Здесь важен сам принцип:

1. На странице есть элемент.
2. Происходит событие.
3. JavaScript замечает событие.
4. Выполняется заданное действие.
5. Интерфейс меняется.

При постановке задачи ИИ полезно описывать именно эту последовательность.

Например:

> При нажатии на кнопку «Записаться» открой модальное окно с формой. По нажатию на крестик, затемнённую область или клавишу Escape окно должно закрываться.

Такой запрос точнее, чем фраза «сделай всплывающую форму».

## Как JavaScript связан с HTML

JavaScript работает с элементами, созданными в HTML.

Он может:

* находить нужные элементы;
* читать их содержимое;
* менять текст;
* добавлять и удалять классы;
* создавать новые элементы;
* скрывать или показывать блоки;
* получать значения полей;
* изменять атрибуты;
* реагировать на события.

Например, в HTML есть кнопка и текстовый блок:

```html
<button id="show-message">Показать сообщение</button>
<p id="message" hidden>Запись успешно создана.</p>
```

JavaScript может найти эти элементы и показать сообщение после нажатия:

```javascript
const button = document.querySelector('#show-message');
const message = document.querySelector('#message');

button.addEventListener('click', () => {
  message.hidden = false;
});
```

HTML определяет, какие элементы существуют.

JavaScript определяет, что с ними происходит.

Чтобы JavaScript мог надёжно находить элементы, в проекте могут использоваться:

* классы;
* идентификаторы;
* специальные атрибуты;
* ссылки на элементы внутри компонентов.

При работе с ИИ важно следить, чтобы названия, используемые в HTML и JavaScript, совпадали. Если код пытается найти элемент с классом, которого нет в HTML, нужное действие не сработает.

## Как JavaScript связан с CSS

JavaScript может напрямую менять стили, но чаще он добавляет или удаляет CSS-классы.

Например, CSS-класс описывает открытое меню:

```css
.menu.is-open {
  display: block;
}
```

JavaScript добавляет этот класс после нажатия:

```javascript
menu.classList.add('is-open');
```

Такое разделение удобно:

* JavaScript управляет состоянием;
* CSS управляет внешним видом этого состояния.

Например, JavaScript определяет, что меню открыто, а CSS решает:

* где оно находится;
* какого оно размера;
* какой у него фон;
* как оно появляется;
* как выглядят его пункты.

Это лучше, чем задавать в JavaScript большое количество отдельных визуальных свойств.

При работе с ИИ можно попросить:

> Используй JavaScript только для переключения состояния меню. Внешний вид открытого и закрытого состояния опиши через CSS-классы.

Так код будет проще читать и изменять.

## Как JavaScript работает с DOM

Когда браузер получает HTML, он создаёт внутреннее представление страницы — DOM.

DOM можно представить как дерево объектов, соответствующих элементам страницы.

JavaScript работает не с исходным HTML-файлом напрямую, а с DOM, который браузер построил на его основе.

Через DOM JavaScript может:

* найти кнопку;
* изменить заголовок;
* добавить карточку;
* удалить уведомление;
* получить текст из поля;
* переключить CSS-класс;
* изменить доступность элемента.

Например, после выбора категории JavaScript может очистить текущий список карточек и добавить в DOM только подходящие элементы.

Важно понимать разницу:

* HTML — исходное описание структуры;
* DOM — текущая структура страницы внутри браузера.

После работы JavaScript DOM может измениться, хотя исходный HTML-файл останется прежним.

Подробнее эта тема рассматривается в отдельной статье о DOM. На базовом уровне достаточно помнить: DOM является основным связующим звеном между JavaScript и страницей.

## Переменные, функции и условия

JavaScript-код состоит из команд и конструкций, которые позволяют хранить данные и описывать действия.

Для общего понимания полезно знать три базовых понятия.

### Переменные

Переменная хранит значение, которое понадобится программе.

Например:

* имя пользователя;
* выбранная услуга;
* стоимость заказа;
* состояние меню;
* количество товаров;
* текст сообщения.

Условный пример:

```javascript
const selectedService = 'Консультация';
```

Здесь переменная хранит выбранную услугу.

### Функции

Функция — это именованный набор действий, который можно вызывать при необходимости.

Например, функция может:

* открыть окно;
* закрыть меню;
* проверить форму;
* рассчитать стоимость;
* загрузить данные;
* показать ошибку.

Условный пример:

```javascript
function openAppointmentForm() {
  // Действия для открытия формы
}
```

Функции помогают не повторять одинаковый код и разделять сложное поведение на понятные части.

### Условия

Условие позволяет выполнить разные действия в зависимости от ситуации.

Например:

> Если пользователь авторизован, покажи личный кабинет. Иначе покажи кнопку входа.

Или:

> Если поле пустое, покажи ошибку. Иначе продолжи отправку формы.

Условный пример:

```javascript
if (phoneNumber === '') {
  showError();
} else {
  submitForm();
}
```

При работе с ИИ необязательно самостоятельно писать такие конструкции. Но понимание их назначения помогает описывать правила интерфейса обычным языком.

## Состояние интерфейса

Во время работы интерфейс может находиться в разных состояниях.

Например, форма может быть:

* пустой;
* заполненной;
* с ошибками;
* в процессе отправки;
* успешно отправленной;
* недоступной из-за ошибки сервера.

Меню может быть открытым или закрытым.

Пользователь может быть авторизован или не авторизован.

Список данных может:

* загружаться;
* успешно отобразиться;
* оказаться пустым;
* не загрузиться из-за ошибки.

JavaScript часто отвечает за хранение текущего состояния и обновление интерфейса в соответствии с ним.

Например, после отправки формы логика может быть такой:

1. Заблокировать кнопку повторной отправки.
2. Показать индикатор загрузки.
3. Отправить данные на сервер.
4. Получить ответ.
5. При успехе показать подтверждение.
6. При ошибке показать понятное сообщение.
7. Снова сделать кнопку доступной.

Если в промте описать только успешный результат, ИИ может не учесть загрузку, ошибку или повторное нажатие.

Поэтому полезно перечислять основные состояния заранее.

## JavaScript и данные

JavaScript умеет работать с разными видами данных.

Например:

* текстом;
* числами;
* логическими значениями «да» или «нет»;
* списками;
* объектами;
* датами;
* данными формы.

Объект может объединять несколько связанных значений.

Например, данные услуги могут включать:

* название;
* описание;
* стоимость;
* длительность;
* изображение;
* идентификатор.

Список может содержать несколько таких услуг.

JavaScript получает данные, преобразует их и показывает в интерфейсе.

Например, он может:

1. Получить список специалистов.
2. Оставить только специалистов выбранного филиала.
3. Отсортировать их по имени.
4. Создать карточки.
5. Добавить карточки на страницу.

Это одна из причин, почему современные веб-приложения могут обновляться без полной перезагрузки страницы.

## Как JavaScript общается с сервером

JavaScript в браузере может отправлять запросы на сервер и получать ответы.

Обычно это происходит через API.

Например, JavaScript может запросить:

* список услуг;
* данные пользователя;
* доступные даты;
* содержимое корзины;
* результаты поиска;
* статус оплаты.

Последовательность может выглядеть так:

1. Пользователь выбирает услугу.
2. JavaScript отправляет запрос серверу.
3. Сервер обрабатывает запрос.
4. При необходимости сервер обращается к базе данных.
5. Сервер возвращает ответ.
6. JavaScript обновляет интерфейс.

Важно понимать, что JavaScript в браузере не должен самостоятельно обращаться напрямую к базе данных.

Обычно схема выглядит так:

```text
Браузер → JavaScript → API → сервер → база данных
```

И в обратную сторону:

```text
База данных → сервер → API → JavaScript → интерфейс
```

JavaScript отвечает за взаимодействие пользователя с данными, но важные правила и защищённые операции должны проверяться сервером.

## Асинхронная работа

Обращение к серверу занимает время. Ответ может прийти через долю секунды или через несколько секунд.

При этом страница не должна полностью останавливаться.

JavaScript умеет выполнять такие операции асинхронно. Это означает, что программа может начать запрос и продолжить работу, пока ожидает ответ.

Например:

1. Пользователь нажимает «Найти».
2. Интерфейс показывает индикатор загрузки.
3. JavaScript отправляет запрос.
4. Пользователь всё ещё может видеть и прокручивать страницу.
5. После получения ответа JavaScript показывает результаты.

Асинхронность используется не только для запросов, но и для:

* таймеров;
* загрузки файлов;
* чтения некоторых данных браузера;
* ожидания разрешения пользователя;
* выполнения последовательности зависимых действий.

Для начинающего здесь важно не изучение синтаксиса, а понимание: некоторые действия завершаются не сразу, поэтому интерфейс должен учитывать ожидание, успех и ошибку.

## Проверка форм

Одна из распространённых задач JavaScript — проверка данных формы перед отправкой.

Например, можно проверить:

* заполнены ли обязательные поля;
* похож ли введённый текст на email;
* достаточно ли символов в пароле;
* выбрана ли услуга;
* принято ли согласие на обработку данных.

Такая проверка называется **клиентской валидацией**, потому что она выполняется в браузере пользователя.

Она помогает быстро сообщить об очевидной ошибке без обращения к серверу.

Однако клиентской проверки недостаточно.

Пользователь может отключить JavaScript, изменить запрос или обойти интерфейс. Поэтому сервер должен повторно проверять важные данные.

Правильная схема:

* JavaScript помогает пользователю исправить ошибку;
* сервер защищает систему и проверяет данные независимо.

Если ИИ создаёт форму, полезно попросить:

> Добавь понятную клиентскую валидацию, но не считай её заменой серверной проверки.

## Что JavaScript не должен делать в браузере

JavaScript в браузере доступен пользователю. Его код можно посмотреть, а его работу — изменить через инструменты разработчика.

Поэтому в клиентском JavaScript нельзя безопасно хранить:

* пароли;
* секретные ключи;
* закрытые токены сервисов;
* данные для доступа к базе;
* правила, которые должны быть скрыты;
* критически важные проверки прав доступа.

Например, нельзя считать надёжной такую логику:

> Если в интерфейсе указана роль администратора, показать административные функции.

Сервер всё равно обязан проверить, действительно ли пользователь имеет соответствующие права.

Нельзя защищать данные только тем, что кнопка скрыта через JavaScript. Пользователь может изменить страницу или самостоятельно отправить запрос.

JavaScript управляет интерфейсом, но безопасность должна поддерживаться серверной частью.

## Как JavaScript подключается к странице

JavaScript может быть подключён к HTML как отдельный файл:

```html
<script src="script.js"></script>
```

В небольших примерах код иногда размещают прямо внутри HTML в теге `<script>`.

Для реального проекта обычно удобнее хранить его в отдельных файлах или модулях.

Это помогает:

* отделить структуру от поведения;
* легче находить нужный код;
* повторно использовать функции;
* тестировать отдельные части;
* не превращать HTML в смешение разметки и логики.

В современных проектах JavaScript часто разбит на модули и компоненты.

Например, отдельно могут храниться:

* логика меню;
* работа формы;
* запросы к API;
* уведомления;
* фильтрация списка;
* общие вспомогательные функции.

ИИ стоит сообщать, в каком формате устроен проект. Код для простой HTML-страницы будет отличаться от кода для React, Vue, Next.js или другого фреймворка.

## Обычный JavaScript и фреймворки

JavaScript можно писать без дополнительных фреймворков. Такой подход часто называют **ванильным JavaScript**, или Vanilla JavaScript.

Для небольших задач его может быть достаточно:

* открыть меню;
* переключить вкладки;
* проверить простую форму;
* добавить фильтр;
* показать уведомление.

В больших интерфейсах часто используют библиотеки и фреймворки:

* React;
* Vue;
* Angular;
* Svelte;
* Next.js и другие.

Они помогают организовывать сложный интерфейс, повторно использовать компоненты и управлять большим количеством состояний.

Но фреймворк не заменяет JavaScript. Он строится поверх языка и использует его возможности.

Поэтому запрос «напиши JavaScript» недостаточно точен, если проект уже использует React или Vue.

Лучше указать контекст:

> Добавь переключение вкладок в существующий React-компонент. Не используй прямое изменение DOM через `document.querySelector`, а управляй активной вкладкой через состояние компонента.

Или:

> Реализуй переключение вкладок на обычном JavaScript без библиотек и фреймворков.

## Почему это важно при работе с ИИ-ассистентом

JavaScript описывает поведение системы, поэтому неточная постановка задачи может привести к неожиданному результату.

Фраза:

> Сделай форму рабочей.

не объясняет:

* какие поля обязательны;
* что считать корректными данными;
* куда отправлять форму;
* что показать во время отправки;
* что делать при успехе;
* что делать при ошибке;
* можно ли отправить форму повторно;
* нужно ли очищать поля;
* где отображать сообщения.

Более точный промт может выглядеть так:

> Добавь JavaScript-логику для формы записи. Проверь обязательные поля имени, телефона и согласия на обработку данных. Во время отправки заблокируй кнопку и покажи текст «Отправляем». При успешном ответе очисти форму и покажи подтверждение. При ошибке сохрани введённые данные и выведи понятное сообщение рядом с кнопкой.

Такой запрос описывает не только действие, но и состояния интерфейса.

При постановке задачи JavaScript полезно указывать:

* что запускает действие;
* какие элементы участвуют;
* какие данные используются;
* какие условия нужно проверить;
* как должен измениться интерфейс;
* требуется ли обращение к серверу;
* что считать успешным результатом;
* как обрабатывать ошибку;
* какие части существующего кода нельзя менять;
* какая технология используется в проекте.

## Как описывать поведение интерфейса

Удобный способ сформулировать задачу — описать её в формате:

> Когда происходит X, система должна сделать Y. Если возникает Z, нужно показать A.

Например:

> Когда пользователь выбирает город, загрузить список доступных клиник. Пока идёт запрос, показать индикатор загрузки. Если клиник нет, вывести сообщение «В этом городе пока нет филиалов». Если сервер недоступен, показать ошибку и кнопку повторной загрузки.

Такое описание легко преобразовать в программную логику.

Для более сложной функции можно перечислить сценарии.

### Основной сценарий

Что происходит при обычной работе.

### Пустое состояние

Что показывать, если данных нет.

### Загрузка

Что видит пользователь во время ожидания.

### Ошибка

Как интерфейс сообщает о проблеме.

### Повторное действие

Можно ли попробовать ещё раз.

### Ограничения

Что пользователь не должен иметь возможности сделать.

Такой подход помогает ИИ создать более полный и предсказуемый код.

## Как проверять JavaScript, созданный ИИ

Даже если код выглядит убедительно, его нужно проверить в реальном интерфейсе.

### Проверь основной сценарий

Выполняется ли действие, ради которого был написан код?

Например, открывается ли меню и отправляется ли форма.

### Проверь повторные действия

Что произойдёт, если пользователь:

* нажмёт кнопку несколько раз;
* повторно откроет окно;
* несколько раз отправит форму;
* быстро переключит варианты;
* вернётся к предыдущему состоянию?

Код может правильно сработать один раз, но сломаться при повторном использовании.

### Проверь ошибочные данные

Попробуй:

* оставить поле пустым;
* ввести неверный формат;
* вставить очень длинный текст;
* выбрать несовместимые параметры.

### Проверь ошибку сервера

Хороший интерфейс не должен бесконечно показывать загрузку или полностью ломаться при неудачном запросе.

### Проверь соответствие HTML

Все элементы, которые JavaScript пытается найти, должны существовать на странице.

### Проверь консоль браузера

Ошибки JavaScript часто отображаются в консоли инструментов разработчика.

Сообщение об ошибке можно передать ИИ вместе с:

* текстом ошибки;
* фрагментом кода;
* описанием ожидаемого результата;
* описанием фактического поведения.

### Проверь область изменений

ИИ может изменить больше файлов или функций, чем требовалось.

Если задача касается одного компонента, убедись, что остальной интерфейс продолжает работать.

### Проверь безопасность

Убедись, что в клиентский код не попали секретные ключи и что важные проверки не выполняются только в браузере.

### Попроси объяснение

Если код непонятен, не ограничивайся копированием.

Можно попросить:

> Объясни этот код по смысловым блокам: какие события он обрабатывает, какие данные хранит, какие элементы изменяет и где возможны ошибки.

## Обработка ошибок

Ошибка — это не только красное сообщение в консоли.

Для пользователя ошибка может означать:

* данные не загрузились;
* форма не отправилась;
* файл слишком большой;
* связь с сервером прервалась;
* пользователь не имеет нужных прав;
* запрашиваемая запись больше недоступна.

JavaScript должен не просто заметить ошибку, но и правильно отразить её в интерфейсе.

Хорошее сообщение сообщает:

* что произошло;
* что пользователь может сделать;
* сохранились ли введённые данные;
* можно ли повторить действие.

Например, сообщение «Error 500» мало помогает обычному пользователю.

Более понятный вариант:

> Не удалось отправить форму. Проверь подключение к интернету и попробуй ещё раз. Введённые данные сохранены.

При этом техническая информация может быть записана отдельно в консоль или систему логирования для разработчиков.

## Производительность и лишний JavaScript

Не каждое действие на странице требует JavaScript.

Например:

* переход по ссылке уже поддерживается HTML;
* простое оформление выполняется CSS;
* некоторые раскрывающиеся элементы можно реализовать стандартными HTML-возможностями;
* состояния наведения оформляются через CSS.

Лишний JavaScript может:

* увеличить объём загружаемых файлов;
* замедлить страницу;
* усложнить поддержку;
* добавить новые ошибки;
* ухудшить доступность.

Поэтому хороший вопрос к ИИ:

> Нужен ли здесь JavaScript, или задачу можно решить надёжнее с помощью HTML и CSS?

Это не означает, что JavaScript нужно избегать. Его стоит использовать там, где действительно нужна логика, работа с данными или сложная интерактивность.

## Доступность интерактивных элементов

JavaScript может сделать интерфейс визуально интерактивным, но не всегда удобным для всех пользователей.

Например, ИИ может создать кликабельный блок из обычного `<div>`. Мышью он будет работать, но пользователь клавиатуры может не суметь его выбрать и активировать.

Поэтому для действий обычно нужно использовать подходящие HTML-элементы:

* кнопку — для действия;
* ссылку — для перехода;
* поле формы — для ввода;
* стандартные элементы выбора — для вариантов.

JavaScript должен дополнять правильную HTML-структуру, а не заменять её.

Также важно учитывать:

* управление с клавиатуры;
* состояние фокуса;
* закрытие окон клавишей Escape;
* понятные подписи;
* сообщения об ошибках;
* информирование программ экранного доступа об изменениях.

В промте можно указать:

> Сохрани доступность интерфейса: используй семантические кнопки, добавь управление с клавиатуры и корректно управляй фокусом при открытии и закрытии модального окна.

## Простая аналогия

Веб-страницу можно сравнить с автомобилем.

HTML — это конструкция: кузов, сиденья, руль, педали и двери.

CSS — внешний вид: цвет, форма приборной панели, обивка и оформление кнопок.

JavaScript — механизмы, которые реагируют на действия водителя:

* нажатие педали запускает движение;
* поворот руля меняет направление;
* кнопка включает свет;
* датчик показывает уровень топлива;
* система сообщает об ошибке.

Красивой панели недостаточно. Нужно заранее определить, какая кнопка за что отвечает, в каких условиях она работает и что произойдёт при неисправности.

Так же устроен веб-интерфейс: JavaScript связывает действия пользователя с изменениями системы.

## Частые заблуждения новичков

### «JavaScript нужен для оформления страницы»

Основное оформление выполняется с помощью CSS. JavaScript может переключать классы и состояния, но не должен без необходимости заменять систему стилей.

### «Java и JavaScript — это одно и то же»

Это разные языки с разными областями применения. Похожее название не означает, что один является сокращённой версией другого.

### «JavaScript работает только в браузере»

JavaScript широко используется в браузерах, но также может работать на сервере и в других средах.

### «Любой кликабельный элемент нужно делать через JavaScript»

Ссылки, кнопки, поля и другие стандартные HTML-элементы уже имеют встроенное поведение. JavaScript стоит добавлять, когда требуется дополнительная логика.

### «Если проверка есть в JavaScript, серверная проверка не нужна»

Клиентский код нельзя считать надёжной защитой. Важные данные, права и ограничения должен повторно проверять сервер.

### «Скрытый элемент защищён от пользователя»

Скрытие кнопки или раздела не является системой безопасности. Пользователь может изменить интерфейс и отправить запрос другим способом.

### «ИИ достаточно попросить сделать элемент рабочим»

Понятие «рабочий» может включать много сценариев. Нужно описывать события, условия, данные, загрузку, успех и ошибки.

### «Если код не показывает ошибку, значит он работает правильно»

Некоторые логические ошибки не вызывают сообщения в консоли. Код может выполняться, но давать неправильный результат.

### «Чем больше JavaScript, тем современнее сайт»

Современный интерфейс не обязан содержать много клиентского кода. Важно использовать подходящий инструмент для конкретной задачи.

### «Можно вставить код ИИ в любое место проекта»

JavaScript зависит от структуры проекта, порядка загрузки файлов, используемого фреймворка и существующих функций. Код нужно адаптировать к контексту.

### «После успешного первого теста проверка закончена»

Нужно проверить повторные действия, неверные данные, медленное соединение, отсутствие данных и ошибки сервера.

### «JavaScript сам сохраняет данные навсегда»

Переменные в браузере обычно существуют только во время работы страницы. Для постоянного хранения данных используются сервер, база данных или специальные механизмы браузера.

## Что важно запомнить

* JavaScript — язык программирования, который добавляет странице поведение и интерактивность.
* HTML создаёт структуру, CSS определяет внешний вид, JavaScript управляет действиями и состояниями.
* JavaScript реагирует на события: клики, ввод, выбор, отправку формы и другие действия.
* Через DOM JavaScript находит и изменяет элементы страницы.
* JavaScript может работать с данными, проверять условия и выполнять повторно используемые функции.
* Для обращения к серверу JavaScript использует запросы к API.
* Интерфейс должен учитывать загрузку, успех, отсутствие данных и ошибку.
* Клиентская проверка формы не заменяет серверную проверку.
* Секретные данные и критически важные проверки нельзя хранить только в браузере.
* Не каждая задача требует JavaScript: иногда достаточно HTML или CSS.
* При работе с ИИ нужно описывать события, ожидаемые действия, условия и все основные состояния.
* Код, созданный ИИ, необходимо проверять в реальных сценариях, а не только читать.
* Человек определяет правила работы интерфейса, а ИИ помогает преобразовать эти правила в код.

## Промты для самостоятельного уточнения

* «Объясни мне простыми словами, как JavaScript используется в реальном веб-проекте и чем он отличается от HTML и CSS.»

* «Покажи короткий пример интерактивного элемента на JavaScript и объясни его по смысловым шагам без подробного разбора синтаксиса.»

* «Объясни, что такое событие в JavaScript, на примерах кнопки, формы и поля ввода.»

* «Расскажи, как JavaScript связан с DOM и как с его помощью изменяется содержимое страницы.»

* «Объясни простыми словами, что такое переменные, функции и условия в JavaScript и зачем они нужны интерфейсу.»

* «Покажи, как JavaScript получает данные с сервера и обновляет страницу. Не используй сложный код.»

* «Помоги описать логику формы: загрузку, проверку полей, успешную отправку и ошибку.»

* «Проверь мой JavaScript-код и объясни, какие события он обрабатывает, какие элементы изменяет и где могут возникнуть ошибки.»

* «Определи, действительно ли для этой задачи нужен JavaScript или её лучше решить с помощью HTML и CSS.»

* «Помоги составить точный промт для добавления JavaScript-функции в мой проект. Учти используемый фреймворк, существующие файлы и ограничения.»
