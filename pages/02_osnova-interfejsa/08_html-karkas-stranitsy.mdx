# HTML: каркас страницы

## Коротко

HTML — это основа практически любой веб-страницы. Он описывает, какие элементы находятся на странице и какую роль они выполняют: где расположен заголовок, где основной текст, где изображение, форма, кнопка или меню.

Когда ты работаешь с ИИ-ассистентом, понимание HTML помогает правильно объяснить, какую страницу нужно создать, из каких блоков она должна состоять и какие элементы должны находиться внутри каждого блока.

## Что такое HTML

HTML расшифровывается как **HyperText Markup Language** — язык гипертекстовой разметки.

HTML не отвечает за всю работу сайта. Его основная задача — описать структуру содержимого страницы.

Например, с помощью HTML браузер понимает, что на странице есть:

* заголовок;
* абзац текста;
* изображение;
* ссылка;
* список;
* кнопка;
* поле для ввода;
* форма обратной связи;
* меню навигации;
* основной раздел страницы;
* нижняя часть сайта.

Эти элементы записываются с помощью специальных обозначений — **тегов**. Каждый тег сообщает браузеру, какой тип содержимого перед ним.

HTML часто сравнивают с каркасом здания. Он определяет, какие помещения существуют и как они расположены, но не отвечает за цвет стен, мебель или работу электрических приборов.

## Для чего используется HTML

HTML используется для создания структуры веб-страниц.

С его помощью можно обозначить:

* шапку сайта;
* навигационное меню;
* основное содержимое;
* карточки товаров или услуг;
* статьи и новости;
* формы записи и регистрации;
* таблицы;
* изображения и видео;
* контактные данные;
* подвал сайта.

HTML также помогает браузерам, поисковым системам и вспомогательным технологиям понимать смысл содержимого.

Например, заголовок должен быть обозначен именно как заголовок, а навигация — как область с основными ссылками. Такая осмысленная разметка называется **семантической**.

Семантика важна не только для внешнего вида. Она делает страницу понятнее для поисковых систем, программ чтения с экрана и других инструментов.

## Как это работает в общей картине

Когда пользователь открывает сайт, браузер получает HTML-документ и начинает разбирать его структуру.

На основе HTML браузер формирует внутреннее представление страницы — DOM. В нём все элементы связаны между собой и образуют дерево: внутри страницы находятся разделы, внутри разделов — заголовки, тексты, изображения и другие элементы.

После этого к HTML подключаются другие технологии:

* **CSS** определяет внешний вид элементов;
* **JavaScript** добавляет поведение и интерактивность;
* **сервер** может подготавливать HTML или передавать данные для его формирования;
* **браузер** собирает все части и показывает пользователю готовую страницу.

Упрощённо их роли можно представить так:

* HTML отвечает за то, **что находится на странице**;
* CSS отвечает за то, **как это выглядит**;
* JavaScript отвечает за то, **как это ведёт себя**.

Например, HTML создаёт кнопку, CSS задаёт её цвет и размер, а JavaScript определяет, что произойдёт после нажатия.

## Структура HTML-страницы

Обычная HTML-страница включает служебную часть и видимую часть.

В служебной части указываются сведения, которые помогают браузеру правильно обработать страницу:

* название страницы;
* кодировка текста;
* настройки отображения на мобильных устройствах;
* подключение стилей;
* информация для поисковых систем.

В видимой части располагается всё, что пользователь видит в браузере:

* меню;
* тексты;
* изображения;
* кнопки;
* формы;
* карточки;
* таблицы;
* ссылки.

Внутри видимой части элементы обычно объединяются в логические блоки. Например, страница услуги может состоять из шапки, первого экрана, описания услуги, преимуществ, цен, формы записи и подвала.

Такая структура помогает разработчику и ИИ-ассистенту понимать, из каких частей должна быть собрана страница.

## Теги, атрибуты и вложенность

Основными строительными элементами HTML являются теги.

Тег сообщает браузеру назначение элемента. Например, один тег обозначает заголовок, другой — абзац, третий — изображение или ссылку.

У элементов могут быть **атрибуты** — дополнительные настройки. Они могут указывать:

* адрес ссылки;
* путь к изображению;
* название элемента;
* дополнительное описание;
* идентификатор;
* принадлежность к определённой группе;
* тип поля формы.

HTML-элементы могут находиться внутри других элементов. Это называется **вложенностью**.

Например, внутри карточки услуги могут находиться:

* изображение;
* название;
* описание;
* цена;
* кнопка записи.

Правильная вложенность важна, потому что она отражает логические связи между частями страницы.

## Семантическая разметка

HTML позволяет использовать универсальные контейнеры, но во многих случаях лучше выбирать элементы, которые объясняют назначение блока.

Например, можно отдельно обозначить:

* шапку сайта;
* навигацию;
* основное содержимое;
* самостоятельный раздел;
* статью;
* дополнительную информацию;
* подвал.

Такая разметка помогает другим системам понять страницу без анализа её внешнего вида.

Для человека страница может выглядеть одинаково при разной разметке. Но для браузера, поисковой системы или программы чтения с экрана разница может быть существенной.

Поэтому при постановке задачи ИИ полезно говорить не только «создай несколько блоков», но и уточнять их смысл: навигация, основная статья, список услуг, форма записи или контактный раздел.

## HTML и содержимое страницы

HTML содержит или подключает содержимое, которое видит пользователь.

К содержимому относятся:

* текст;
* изображения;
* ссылки;
* видео;
* таблицы;
* списки;
* элементы форм.

При этом HTML не должен использоваться только ради визуального расположения элементов. Структура должна отражать смысл страницы.

Например, крупный текст не обязательно является заголовком. Он должен быть обозначен как заголовок только тогда, когда действительно называет страницу или её раздел.

Это важно при работе с ИИ: ассистент может сделать страницу визуально похожей на макет, но использовать неподходящую структуру. Поэтому результат нужно оценивать не только по внешнему виду, но и по логике разметки.

## Почему это важно при работе с ИИ-ассистентом

ИИ может быстро создать HTML-структуру страницы, но качество результата зависит от того, насколько понятно описана задача.

Фраза «сделай страницу клиники» слишком общая. ИИ придётся самостоятельно угадывать её структуру.

Более точная задача может содержать:

* назначение страницы;
* список основных разделов;
* порядок блоков;
* содержимое каждого блока;
* наличие меню, кнопок, форм и ссылок;
* требования к семантической разметке;
* необходимость адаптации для дальнейшего подключения CSS и JavaScript.

Например, можно попросить ИИ создать структуру страницы услуги, в которой будут шапка, навигация, описание процедуры, преимущества, показания, цены, специалисты, ответы на вопросы и форма записи.

При проверке результата полезно убедиться, что:

* все нужные блоки присутствуют;
* заголовки расположены в логическом порядке;
* кнопки и ссылки имеют понятное назначение;
* поля формы правильно подписаны;
* элементы вложены логично;
* структура не состоит только из безымянных контейнеров;
* HTML не смешан без необходимости с большим количеством стилей и логики.

Понимание HTML позволяет обсуждать с ИИ не отдельные символы кода, а структуру страницы и назначение её элементов.

## Простая аналогия

Представь страницу как здание.

HTML — это архитектурный план: комнаты, этажи, двери и лестницы.

CSS — это отделка: цвета стен, материалы, освещение и мебель.

JavaScript — это работающие системы: автоматические двери, лифты, сигнализация и управление освещением.

Без HTML браузеру было бы непонятно, какие части страницы существуют и как они связаны между собой.

## Частые заблуждения новичков

### HTML — это полноценный язык программирования

HTML описывает структуру документа, но сам по себе не выполняет сложную логику и вычисления. Для интерактивного поведения обычно используется JavaScript, а для серверных операций — backend-технологии.

### HTML отвечает за дизайн страницы

HTML может влиять на базовое отображение, но полноценное оформление обычно создаётся с помощью CSS.

### Любую страницу достаточно собрать из одинаковых блоков

Технически страницу можно построить из универсальных контейнеров, но такая разметка хуже передаёт смысл содержимого. Лучше использовать подходящие семантические элементы.

### Если страница выглядит правильно, HTML тоже сделан правильно

Внешне одинаковые страницы могут иметь разное качество структуры. Ошибки HTML могут влиять на доступность, поисковую оптимизацию, поддержку проекта и работу JavaScript.

### ИИ сам определит идеальную структуру страницы

ИИ может предложить разумный вариант, но он не знает всех целей проекта. Человек должен определить состав страницы, назначение блоков и желаемое поведение.

### HTML-файл всегда является всей страницей целиком

В современных проектах страница может собираться из компонентов, шаблонов и данных. HTML может формироваться браузером, сервером или фреймворком во время работы приложения.

## Что важно запомнить

* HTML создаёт структуру веб-страницы.
* Страница состоит из элементов, которые обозначаются тегами.
* Атрибуты передают дополнительные настройки элементов.
* Вложенность показывает, какие элементы находятся внутри других.
* Семантическая разметка отражает назначение блоков.
* CSS отвечает за оформление, а JavaScript — за поведение.
* Хороший HTML должен быть понятен не только браузеру, но и человеку.
* При работе с ИИ нужно описывать назначение страницы, состав блоков и их порядок.
* ИИ создаёт разметку, но человек определяет структуру и проверяет результат.

## Промты для самостоятельного уточнения

* «Объясни мне простыми словами, из каких основных частей состоит HTML-документ и зачем нужна каждая часть».

* «Покажи простую HTML-структуру страницы услуги без сложных стилей и подробно объясни назначение каждого блока».

* «Объясни разницу между HTML, CSS и JavaScript на примере обычной страницы сайта».

* «Расскажи, что такое семантическая HTML-разметка и почему она важна для сайта».

* «Проверь HTML моей страницы и укажи, какие элементы имеют неправильную вложенность или не соответствуют своему назначению».

* «Предложи логичную HTML-структуру для страницы клиники с описанием услуги, ценами, специалистами и формой записи».

* «Объясни, как HTML связан с DOM и как JavaScript получает доступ к элементам страницы».

* «Создай HTML-каркас страницы по моему описанию. Не добавляй сложные стили и сначала перечисли предполагаемые разделы страницы».
