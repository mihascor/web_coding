# Как работает веб - браузер, сервер, HTTP

## Как работает веб: браузер, сервер, HTTP

Веб — это система, благодаря которой ты открываешь сайты, приложения, интернет-магазины, личные кабинеты, карты, почту и другие онлайн-сервисы.

Когда ты вводишь адрес сайта в браузере, например:

```text
https://example.com
```

происходит обмен данными между твоим устройством и удалённым компьютером, на котором хранится сайт. Этот обмен работает по определённым правилам, главное из которых называется **HTTP**.

---

# 1. Что такое браузер

**Браузер** — это программа, через которую человек открывает сайты.

Примеры браузеров:

|Браузер|Где используется|
|---|---|
|Google Chrome|Один из самых популярных браузеров|
|Safari|Часто используется на iPhone и Mac|
|Mozilla Firefox|Альтернативный браузер|
|Microsoft Edge|Браузер от Microsoft|
|Opera|Браузер с дополнительными функциями|

Браузер нужен, чтобы:

- открыть сайт по адресу;
    
- отправить запрос на сервер;
    
- получить HTML, CSS, JavaScript, изображения и другие файлы;
    
- собрать из этих файлов готовую страницу;
    
- показать её пользователю.
    

Проще говоря, браузер — это **программа, которая превращает код сайта в видимую страницу**.

---

# 2. Что такое сервер

**Сервер** — это компьютер или программа, которая хранит сайт и отдаёт данные по запросу.

Например, сайт клиники, интернет-магазина или онлайн-школы физически где-то размещён: на сервере хостинга, в облаке или в дата-центре.

Сервер может хранить:

- HTML-страницы;
    
- CSS-файлы;
    
- JavaScript-файлы;
    
- изображения;
    
- видео;
    
- данные пользователей;
    
- товары интернет-магазина;
    
- записи из базы данных;
    
- настройки сайта.
    

Когда браузер просит:  
**«Дай мне главную страницу сайта»**,  
сервер отвечает:  
**«Вот нужные файлы»**.

---

# 3. Что такое клиент и сервер

В веб-разработке часто говорят о модели **клиент — сервер**.

## Клиент

**Клиент** — это сторона, которая делает запрос.

Обычно клиентом является:

- браузер на компьютере;
    
- браузер на телефоне;
    
- мобильное приложение;
    
- иногда другая программа.
    

## Сервер

**Сервер** — это сторона, которая принимает запрос, обрабатывает его и отправляет ответ.

Пример:

```text
Браузер: Покажи страницу "Контакты"
Сервер: Вот HTML, CSS, JavaScript и данные для этой страницы
```

---

# 4. Что такое HTTP

**HTTP** — это протокол, то есть набор правил, по которым браузер и сервер общаются друг с другом.

HTTP расшифровывается как:

```text
HyperText Transfer Protocol
```

По-русски: **протокол передачи гипертекста**.

Но сегодня через HTTP передаётся не только текст, а почти всё:

- HTML;
    
- CSS;
    
- JavaScript;
    
- изображения;
    
- JSON-данные;
    
- видео;
    
- файлы;
    
- формы;
    
- ответы от API.
    

HTTP нужен, чтобы браузер и сервер понимали друг друга.

---

# 5. Как открывается сайт пошагово

Представим, что ты вводишь в браузере адрес:

```text
https://site.ru
```

Происходит примерно следующее:

## Шаг 1. Браузер получает адрес сайта

Ты вводишь URL в адресную строку.

URL — это адрес ресурса в интернете.

Пример:

```text
https://site.ru/about
```

В нём есть:

|Часть|Значение|
|---|---|
|`https`|Протокол|
|`site.ru`|Домен сайта|
|`/about`|Конкретная страница или путь|

---

## Шаг 2. Браузер ищет сервер

Браузеру нужно понять, где физически находится сайт.

Для этого используется система **DNS**.

DNS можно представить как телефонную книгу интернета:

```text
site.ru → IP-адрес сервера
```

Домен удобен человеку, а IP-адрес нужен компьютерам.

Пример IP-адреса:

```text
192.0.2.1
```

---

## Шаг 3. Браузер отправляет HTTP-запрос

Браузер отправляет серверу запрос:

```text
GET /about HTTP/1.1
Host: site.ru
```

В человеческом виде это значит:

```text
Пожалуйста, дай мне страницу /about сайта site.ru
```

---

## Шаг 4. Сервер обрабатывает запрос

Сервер смотрит, что именно запросил браузер.

Он может:

- найти готовый HTML-файл;
    
- обратиться к базе данных;
    
- проверить авторизацию пользователя;
    
- собрать страницу динамически;
    
- вернуть ошибку, если страница не найдена.
    

---

## Шаг 5. Сервер отправляет HTTP-ответ

Сервер возвращает ответ.

Например:

```text
HTTP/1.1 200 OK
Content-Type: text/html
```

А ниже отправляет содержимое страницы:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>О нас</title>
  </head>
  <body>
    <h1>О компании</h1>
  </body>
</html>
```

---

## Шаг 6. Браузер показывает страницу

Браузер получает HTML и начинает его читать.

Затем он загружает дополнительные файлы:

```html
<link rel="stylesheet" href="style.css">
<script src="script.js"></script>
<img src="photo.jpg">
```

То есть браузер дополнительно просит у сервера:

- CSS для оформления;
    
- JavaScript для интерактивности;
    
- картинки;
    
- шрифты;
    
- другие ресурсы.
    

После этого браузер собирает страницу и показывает её пользователю.

---

# 6. Что такое HTTPS

Ты часто видишь адреса, которые начинаются не с `http`, а с `https`.

```text
https://site.ru
```

**HTTPS** — это защищённая версия HTTP.

Буква **S** означает:

```text
Secure
```

То есть «безопасный».

HTTPS шифрует данные между браузером и сервером.

Это особенно важно для:

- паролей;
    
- банковских данных;
    
- личных кабинетов;
    
- медицинских данных;
    
- онлайн-оплат;
    
- форм обратной связи.
    

Сегодня нормальный современный сайт почти всегда должен работать через HTTPS.

---

# 7. Что такое HTTP-запрос

**HTTP-запрос** — это сообщение от клиента к серверу.

Например:

```text
GET /products
```

Это значит:

```text
Получи список товаров
```

Или:

```text
POST /login
```

Это значит:

```text
Отправь данные формы входа
```

Запрос обычно содержит:

- метод;
    
- адрес;
    
- заголовки;
    
- иногда тело запроса.
    

---

# 8. Основные HTTP-методы

HTTP-методы показывают, что именно клиент хочет сделать.

|Метод|Для чего используется|
|---|---|
|`GET`|Получить данные|
|`POST`|Отправить новые данные|
|`PUT`|Полностью обновить данные|
|`PATCH`|Частично обновить данные|
|`DELETE`|Удалить данные|

Примеры:

```text
GET /articles
```

Получить список статей.

```text
POST /orders
```

Создать новый заказ.

```text
DELETE /products/15
```

Удалить товар с ID 15.

---

# 9. Что такое HTTP-ответ

**HTTP-ответ** — это сообщение от сервера к клиенту.

Ответ содержит:

- статус;
    
- заголовки;
    
- тело ответа.
    

Пример:

```text
HTTP/1.1 200 OK
Content-Type: application/json
```

Тело ответа может быть HTML:

```html
<h1>Главная страница</h1>
```

Или JSON:

```json
{
  "name": "Anna",
  "role": "admin"
}
```

---

# 10. Что такое статус-коды HTTP

**Статус-код** показывает результат запроса.

Например, всё прошло успешно или возникла ошибка.

|Код|Значение|
|---|---|
|`200 OK`|Всё хорошо|
|`201 Created`|Объект создан|
|`301 Moved Permanently`|Страница переехала навсегда|
|`302 Found`|Временное перенаправление|
|`400 Bad Request`|Ошибка в запросе|
|`401 Unauthorized`|Нужно войти в аккаунт|
|`403 Forbidden`|Доступ запрещён|
|`404 Not Found`|Страница не найдена|
|`500 Internal Server Error`|Ошибка на сервере|

Самый известный код — **404**. Он означает, что сервер не нашёл запрошенную страницу.

---

# 11. Что такое фронтенд и бэкенд

Веб-разработка часто делится на две большие части: **фронтенд** и **бэкенд**.

## Фронтенд

**Фронтенд** — это всё, что пользователь видит и с чем взаимодействует в браузере.

Фронтенд включает:

- HTML;
    
- CSS;
    
- JavaScript;
    
- кнопки;
    
- формы;
    
- меню;
    
- анимации;
    
- интерфейс сайта.
    

Пример задачи фронтенда:

```text
Показать красивую карточку товара и кнопку "Купить"
```

---

## Бэкенд

**Бэкенд** — это серверная часть сайта.

Бэкенд отвечает за:

- хранение данных;
    
- регистрацию пользователей;
    
- авторизацию;
    
- обработку заказов;
    
- работу с базой данных;
    
- бизнес-логику;
    
- безопасность;
    
- отправку ответов браузеру.
    

Пример задачи бэкенда:

```text
Проверить, есть ли товар в наличии, создать заказ и сохранить его в базе данных
```

---

# 12. Простой пример: форма заявки

Представим сайт клиники с формой заявки:

```text
Имя
Телефон
Услуга
Кнопка "Отправить"
```

Когда пользователь нажимает кнопку:

1. Браузер собирает данные из формы.
    
2. JavaScript может проверить, заполнены ли поля.
    
3. Браузер отправляет HTTP-запрос на сервер.
    
4. Сервер принимает данные.
    
5. Сервер сохраняет заявку в базе данных.
    
6. Сервер возвращает ответ.
    
7. Браузер показывает сообщение:
    

```text
Спасибо, заявка отправлена
```

Это и есть типичный процесс взаимодействия браузера, сервера и HTTP.

---

# 13. Что такое API

**API** — это способ, с помощью которого разные программы общаются между собой.

В вебе часто используется API между фронтендом и бэкендом.

Например, фронтенд отправляет запрос:

```text
GET /api/doctors
```

Сервер отвечает JSON-данными:

```json
[
  {
    "name": "Иванова Мария",
    "specialization": "Косметолог"
  },
  {
    "name": "Петрова Анна",
    "specialization": "Дерматолог"
  }
]
```

После этого браузер красиво отображает список врачей на странице.

---

# 14. Что такое JSON

**JSON** — это формат передачи данных.

Он похож на объект JavaScript и часто используется в API.

Пример JSON:

```json
{
  "name": "Мария",
  "age": 32,
  "city": "Berlin"
}
```

JSON удобен, потому что его легко читать и людям, и программам.

# 15. Статические и динамические сайты

## Статический сайт

**Статический сайт** — это сайт, где сервер отдаёт уже готовые файлы.

Например:

- HTML;
    
- CSS;
    
- JavaScript;
    
- картинки.
    

Такой сайт подходит для:

- лендингов;
    
- портфолио;
    
- простых страниц;
    
- документации;
    
- презентационных сайтов.
    

---

## Динамический сайт

**Динамический сайт** создаёт страницы в зависимости от ситуации.

Например:

- кто вошёл в аккаунт;
    
- какие товары есть в базе;
    
- какой город выбран;
    
- какие записи доступны;
    
- какие права есть у пользователя.
    

Примеры динамических сайтов:

- интернет-магазины;
    
- личные кабинеты;
    
- CRM-системы;
    
- сайты клиник с онлайн-записью;
    
- социальные сети.
    

---

# 16. Зачем веб-кодеру понимать браузер, сервер и HTTP

Даже если ты начинаешь с HTML, CSS и JavaScript, важно понимать, как работает веб в целом.

Это помогает:

- понимать, куда уходят данные из формы;
    
- разбираться, почему страница не загружается;
    
- понимать ошибки `404`, `500`, `403`;
    
- работать с API;
    
- подключать фронтенд к бэкенду;
    
- читать вкладку **Network** в инструментах разработчика;
    
- понимать, где проблема: в браузере, сервере или запросе;
    
- лучше управлять ИИ-ассистентом при создании сайтов.

# 17. Простая схема работы веба

```text
Пользователь
    ↓
Браузер
    ↓ HTTP-запрос
Сервер
    ↓
База данных / файлы / логика
    ↓
Сервер
    ↓ HTTP-ответ
Браузер
    ↓
Готовая страница на экране
```

Ещё проще:

```text
Браузер просит → сервер отвечает
```

# 18. Мини-пример на человеческом языке

Представь ресторан.

**Браузер** — это посетитель.  
**HTTP-запрос** — это заказ официанту.  
**Сервер** — это кухня.  
**HTTP-ответ** — это готовое блюдо.  
**HTML, CSS, JavaScript** — это ингредиенты и оформление блюда.

Посетитель говорит:

```text
Хочу главную страницу сайта
```

Кухня отвечает:

```text
Вот HTML, CSS, JavaScript и картинки
```

Браузер всё собирает и показывает пользователю красивую страницу.

# Кратко

**Браузер** — программа, которая открывает сайты и показывает страницы пользователю.

**Сервер** — компьютер или программа, которая хранит сайт, обрабатывает запросы и отдаёт данные.

**HTTP** — протокол, по которому браузер и сервер общаются.

**HTTPS** — защищённая версия HTTP.

**Запрос** — сообщение от браузера к серверу.

**Ответ** — сообщение от сервера к браузеру.

**Фронтенд** — видимая часть сайта.

**Бэкенд** — серверная логика и работа с данными.

Главная идея веба очень простая:

```text
Пользователь открывает сайт → браузер отправляет запрос → сервер отвечает → браузер показывает страницу
```
