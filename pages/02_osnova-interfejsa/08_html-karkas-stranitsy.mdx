# HTML: каркас страницы

## Коротко

HTML — это основа практически любой веб-страницы. Он описывает, какие элементы находятся на странице: заголовки, текст, изображения, кнопки, ссылки, формы, списки и другие части интерфейса.

Когда ты просишь ИИ-ассистента создать страницу сайта, именно HTML определяет её первоначальную структуру. CSS отвечает за внешний вид элементов, а JavaScript — за их поведение. Поэтому понимание HTML помогает точнее объяснять ИИ, какие блоки должны быть на странице и как они связаны между собой.

## Что это такое

HTML расшифровывается как **HyperText Markup Language** — язык гипертекстовой разметки.

Это не язык программирования в привычном смысле. HTML не выполняет вычисления и не принимает решения. Его задача — описывать содержание и структуру страницы.

Для этого используются специальные обозначения — **теги**. Каждый тег сообщает браузеру, какой тип элемента нужно отобразить.

Например:

* заголовок;
* обычный абзац;
* ссылка;
* изображение;
* кнопка;
* список;
* поле формы;
* раздел страницы.

Условно HTML-страница может выглядеть так:

```html
<h1>Название страницы</h1>
<p>Короткое описание страницы.</p>
<button>Записаться</button>
```

Браузер понимает, что первая строка — главный заголовок, вторая — текстовый абзац, а третья — кнопка.

Теги могут находиться внутри других тегов. Благодаря этому страница превращается не в случайный набор элементов, а в понятную структуру.

Например, внутри общего блока карточки могут находиться:

* изображение;
* название услуги;
* описание;
* цена;
* кнопка записи.

Такое вложение помогает браузеру, разработчику и ИИ-ассистенту понимать, какие элементы относятся друг к другу.

## Для чего это используется

HTML используется для создания содержательной основы веб-страницы.

С его помощью можно описать:

* шапку сайта;
* меню навигации;
* главный экран;
* карточки товаров или услуг;
* тексты и заголовки;
* изображения и видео;
* формы обратной связи;
* таблицы;
* списки;
* подвал сайта.

HTML также помогает показать смысл каждого элемента.

Например, текст можно оформить просто как универсальный блок. Но лучше сообщить браузеру, что это именно заголовок, абзац, кнопка или навигационное меню.

Такая осмысленная разметка называется **семантической**.

Семантика означает, что HTML-элемент выбирается не только по внешнему виду, но и по его назначению.

Например:

* `<header>` — верхняя часть страницы или раздела;
* `<nav>` — навигация;
* `<main>` — основное содержание;
* `<section>` — отдельный смысловой раздел;
* `<article>` — самостоятельный материал;
* `<footer>` — нижняя часть страницы.

Семантическая структура делает страницу понятнее для разработчиков, поисковых систем и программ экранного доступа, которыми пользуются люди с нарушениями зрения.

## Как это работает в общей картине

Когда пользователь открывает сайт, браузер получает HTML-документ и начинает его разбирать.

На основе HTML браузер строит внутреннее представление страницы — **DOM**. DOM можно представить как дерево элементов, в котором видно, какие блоки находятся внутри других блоков.

Например:

* страница;

  * шапка;

    * логотип;
    * меню;
  * основная часть;

    * заголовок;
    * список услуг;

      * карточка услуги;
      * карточка услуги;
  * подвал.

После этого к HTML подключается CSS. Он определяет цвета, размеры, отступы, шрифты и расположение элементов.

Затем JavaScript может добавить интерактивность:

* открытие меню;
* переключение вкладок;
* отправку формы;
* появление всплывающего окна;
* загрузку данных с сервера.

Таким образом, три основные технологии интерфейса выполняют разные задачи:

* **HTML** описывает, что находится на странице;
* **CSS** определяет, как это выглядит;
* **JavaScript** управляет тем, как это работает и реагирует на действия пользователя.

HTML также может взаимодействовать с сервером. Например, форма записи содержит поля, созданные с помощью HTML. Пользователь вводит данные и нажимает кнопку. После этого JavaScript или сам браузер отправляет информацию на сервер.

HTML при этом не сохраняет запись и не обрабатывает оплату. Он только создаёт элементы интерфейса, через которые пользователь взаимодействует с системой.

## Основные части HTML-документа

У обычной HTML-страницы есть общая структура.

В ней можно выделить две основные части:

### Служебная часть

В ней находятся настройки страницы, которые обычно не отображаются как обычный контент.

Здесь могут быть указаны:

* название страницы для вкладки браузера;
* кодировка текста;
* настройки отображения на мобильных устройствах;
* подключение CSS;
* описание страницы для поисковых систем.

Эта часть находится внутри элемента `<head>`.

### Видимая часть

В ней располагается всё, что пользователь видит на странице:

* заголовки;
* текст;
* изображения;
* кнопки;
* формы;
* меню;
* карточки;
* таблицы.

Эта часть находится внутри элемента `<body>`.

Поэтому просьба «создай HTML-страницу» может означать разные вещи. Иногда нужен полный документ со служебной частью, а иногда — только отдельный блок, который будет встроен в существующий проект.

Это полезно уточнять в промте.

## Теги, атрибуты и содержимое

Большинство HTML-элементов состоит из трёх частей:

1. Открывающего тега.
2. Содержимого.
3. Закрывающего тега.

Например:

```html
<a href="/services">Посмотреть услуги</a>
```

Здесь:

* `<a>` — тег ссылки;
* `Посмотреть услуги` — видимый текст;
* `href="/services"` — атрибут, который указывает адрес перехода.

**Атрибуты** передают браузеру дополнительную информацию об элементе.

С их помощью можно указать:

* адрес ссылки;
* путь к изображению;
* тип поля формы;
* уникальный идентификатор;
* CSS-класс;
* доступность элемента;
* дополнительное поведение.

При работе с ИИ полезно разделять три вещи:

* какой элемент нужен;
* какое содержимое должно быть внутри;
* какие свойства или атрибуты ему необходимы.

Например, вместо просьбы «сделай кнопку» можно написать:

> Создай кнопку «Записаться», добавь ей понятное имя CSS-класса и укажи, что при нажатии должна открываться форма записи.

## Почему это важно при работе с ИИ-ассистентом

ИИ может быстро создать HTML-разметку, но качество результата зависит от того, насколько понятно описана структура страницы.

Фраза «сделай красивый сайт клиники» слишком общая. ИИ будет вынужден самостоятельно придумывать состав и порядок блоков.

Более точная задача может выглядеть так:

> Создай семантическую HTML-разметку страницы услуги. Добавь шапку с логотипом и меню, основной заголовок, описание процедуры, блок преимуществ, список показаний, форму записи и подвал. Не добавляй CSS и JavaScript.

В таком промте уже указаны:

* тип страницы;
* необходимые разделы;
* порядок элементов;
* требование использовать семантическую разметку;
* технологии, которые пока не нужны.

Понимание HTML также помогает проверять ответ ИИ.

Ты можешь обратить внимание:

* есть ли один понятный главный заголовок;
* логично ли расположены разделы;
* используются ли подходящие теги;
* не перепутаны ли ссылки и кнопки;
* есть ли подписи у полей формы;
* понятны ли названия CSS-классов;
* не вставлены ли стили прямо в HTML без необходимости;
* нет ли слишком большого количества бессмысленных универсальных блоков.

Важно помнить, что визуально похожие элементы могут иметь разное назначение.

**Ссылка** обычно переводит пользователя на другую страницу или раздел.

**Кнопка** выполняет действие: отправляет форму, открывает окно, запускает загрузку или подтверждает выбор.

ИИ иногда использует эти элементы неправильно, если задача сформулирована слишком общо.

## Простая аналогия

HTML можно сравнить с конструкцией здания.

В проекте здания указано:

* где находятся стены;
* где будут двери и окна;
* сколько этажей;
* какие помещения связаны между собой.

При этом проект ещё не определяет цвет стен, мебель и освещение.

В веб-странице HTML играет роль конструкции здания. CSS отвечает за оформление, а JavaScript — за работающие механизмы: двери, лифты, сигнализацию и освещение.

Если конструкция страницы построена неправильно, красивое оформление не решит проблему полностью.

## Частые заблуждения новичков

### «HTML отвечает за весь внешний вид страницы»

HTML создаёт элементы, но основное оформление выполняется с помощью CSS.

HTML может влиять на базовое отображение, однако цвета, сетки, размеры и адаптивность обычно описываются отдельно.

### «HTML — это язык программирования»

HTML является языком разметки. Он описывает структуру документа, но сам по себе не содержит полноценной программной логики.

### «Для всех элементов можно использовать один универсальный тег»

Технически многие блоки можно создать с помощью `<div>`, но такая страница будет хуже передавать смысл структуры.

Лучше использовать подходящие семантические элементы там, где это возможно.

### «Заголовки выбирают по размеру текста»

Уровни заголовков показывают структуру содержания, а не только визуальный размер.

Главный заголовок страницы обычно обозначается как `<h1>`, его подразделы — как `<h2>`, а вложенные темы — как `<h3>`.

Размер заголовка можно изменить с помощью CSS.

### «Кнопка и ссылка — одно и то же»

Они могут выглядеть одинаково, но выполняют разные задачи. Ссылка ведёт к другому адресу, а кнопка запускает действие.

### «Если страница выглядит нормально, HTML написан правильно»

Страница может выглядеть приемлемо даже при неудачной структуре.

Проблемы могут проявиться позже:

* при добавлении CSS;
* на мобильных устройствах;
* при подключении JavaScript;
* при работе с поисковыми системами;
* при использовании программ экранного доступа;
* при дальнейшем редактировании проекта.

### «ИИ всегда выберет правильную структуру сам»

ИИ может создать рабочую разметку, но он не знает всех требований проекта, пока ты их не сообщишь.

Именно человек определяет назначение страницы, состав блоков и ожидаемое поведение интерфейса.

## Что важно запомнить

* HTML описывает структуру и содержимое веб-страницы.
* HTML состоит из элементов, тегов, атрибутов и вложенного содержимого.
* CSS отвечает за оформление, а JavaScript — за интерактивность.
* Семантические теги помогают передать назначение частей страницы.
* Браузер превращает HTML в структуру DOM.
* Хорошая HTML-разметка должна быть логичной и понятной, даже без оформления.
* При работе с ИИ полезно заранее перечислять нужные разделы страницы.
* ИИ создаёт разметку, но человек определяет структуру задачи и проверяет результат.

## Промты для самостоятельного уточнения

* «Объясни мне простыми словами, как HTML используется в реальном веб-проекте.»

* «Покажи простую структуру HTML-страницы и объясни назначение каждой её части без подробного разбора синтаксиса.»

* «Объясни, чем HTML отличается от CSS и JavaScript, на примере одной страницы сайта.»

* «Расскажи, что такое семантическая HTML-разметка и почему она лучше большого количества тегов div.»

* «Покажи, как может быть устроена HTML-структура страницы услуги, но не добавляй CSS и JavaScript.»

* «Проверь мою HTML-разметку и укажи, где нарушена логика заголовков, вложенности или семантики.»

* «Объясни, чем кнопка отличается от ссылки и в каких случаях нужно использовать каждый элемент.»

* «Помоги составить промт для создания HTML-каркаса страницы моего сайта. Сначала выдели основные блоки страницы.»
