# Формы и валидация

## Формы и валидация в веб-разработке

**Формы** — это элементы сайта, через которые пользователь вводит и отправляет данные.

Например:

- имя;
- телефон;
- email;
- пароль;
- комментарий;
- адрес доставки;
- дату записи;
- файл;
- выбранный вариант из списка.

Формы используются почти на каждом сайте: при регистрации, авторизации, оформлении заказа, записи на услугу, отправке заявки, поиске, подписке на рассылку.

---

# 1. Что такое форма

В HTML форма создаётся с помощью тега:

```html
<form>
  ...
</form>
```

Внутри формы размещаются поля ввода, кнопки, выпадающие списки, чекбоксы и другие элементы.

Простой пример формы:

```html
<form>
  <label>
    Ваше имя:
    <input type="text" name="username">
  </label>

  <button type="submit">Отправить</button>
</form>
```

Здесь:

- `<form>` — контейнер формы;
- `<label>` — подпись к полю;
- `<input>` — поле ввода;
- `button type="submit"` — кнопка отправки формы.

---

# 2. Основные элементы форм

## `input`

Самый распространённый элемент формы.

```html
<input type="text">
```

У `input` есть разные типы.

| Тип | Для чего используется |
|---|---|
| `text` | Обычный текст |
| `email` | Email-адрес |
| `password` | Пароль |
| `number` | Число |
| `tel` | Телефон |
| `date` | Дата |
| `checkbox` | Галочка |
| `radio` | Выбор одного варианта |
| `file` | Загрузка файла |
| `submit` | Кнопка отправки |

Пример:

```html
<input type="email" placeholder="Введите email">
<input type="password" placeholder="Введите пароль">
```

---

## `textarea`

Используется для длинного текста.

Например, для комментария или сообщения:

```html
<textarea name="message"></textarea>
```

---

## `select`

Выпадающий список:

```html
<select name="city">
  <option value="moscow">Москва</option>
  <option value="spb">Санкт-Петербург</option>
  <option value="kazan">Казань</option>
</select>
```

---

## `button`

Кнопка:

```html
<button type="submit">Отправить</button>
```

Основные типы кнопок:

| Тип кнопки | Что делает |
|---|---|
| `submit` | Отправляет форму |
| `button` | Обычная кнопка для JavaScript |
| `reset` | Сбрасывает поля формы |

---

# 3. Атрибут `name`

У полей формы часто есть атрибут `name`.

```html
<input type="text" name="username">
```

Он нужен, чтобы при отправке формы было понятно, **какое значение к какому полю относится**.

Например:

```html
username=Anna
email=anna@example.com
```

Без `name` данные поля обычно не попадут в отправляемую форму.

---

# 4. Атрибут `placeholder`

`placeholder` показывает подсказку внутри поля:

```html
<input type="text" placeholder="Введите имя">
```

Важно: `placeholder` не заменяет полноценную подпись `<label>`. Он исчезает, когда пользователь начинает вводить текст.

Лучше писать так:

```html
<label for="name">Имя</label>
<input id="name" type="text" name="name" placeholder="Анна">
```

---

# 5. Что такое валидация

**Валидация** — это проверка данных, которые пользователь ввёл в форму.

Например:

- поле email действительно похоже на email;
- пароль не слишком короткий;
- телефон указан в нужном формате;
- обязательные поля заполнены;
- возраст больше 18;
- файл имеет допустимый размер;
- пользователь поставил галочку согласия с условиями.

Пример: пользователь нажимает «Отправить», но забыл указать email. Сайт показывает ошибку:  
**«Введите email»**.

---

# 6. Зачем нужна валидация

Валидация нужна, чтобы:

- не отправлять пустые или неправильные данные;
- помогать пользователю исправить ошибки;
- уменьшать количество мусорных заявок;
- защищать сайт от некорректного ввода;
- улучшать качество данных в базе;
- делать интерфейс понятнее и удобнее.

Например, если в поле «Телефон» пользователь ввёл только `123`, система может попросить его ввести полный номер.

---

# 7. HTML-валидация

В HTML уже есть простые встроенные инструменты для проверки формы.

## `required`

Поле обязательно для заполнения:

```html
<input type="text" name="name" required>
```

---

## `type="email"`

Браузер проверит, что введённый текст похож на email:

```html
<input type="email" name="email" required>
```

---

## `minlength` и `maxlength`

Минимальная и максимальная длина текста:

```html
<input type="password" minlength="8" maxlength="20">
```

---

## `min` и `max`

Минимальное и максимальное значение числа:

```html
<input type="number" min="18" max="100">
```

---

## `pattern`

Проверка по шаблону:

```html
<input type="text" pattern="[A-Za-z]{3,}">
```

Этот пример разрешает только латинские буквы, минимум 3 символа.

---

# 8. Пример формы с HTML-валидацией

```html
<form>
  <label for="name">Имя</label>
  <input id="name" type="text" name="name" required>

  <label for="email">Email</label>
  <input id="email" type="email" name="email" required>

  <label for="password">Пароль</label>
  <input id="password" type="password" name="password" minlength="8" required>

  <button type="submit">Зарегистрироваться</button>
</form>
```

Если пользователь не заполнит обязательные поля или введёт слишком короткий пароль, браузер сам покажет предупреждение.

---

# 9. Валидация через JavaScript

HTML-валидация полезна, но часто её недостаточно. Поэтому используют JavaScript.

JavaScript позволяет:

- показывать свои тексты ошибок;
- подсвечивать поля;
- проверять сложные условия;
- сравнивать поля, например пароль и повтор пароля;
- отправлять форму без перезагрузки страницы;
- блокировать кнопку, пока форма не заполнена правильно.

Простой пример:

```html
<form id="loginForm">
  <input id="email" type="email" placeholder="Email">
  <p id="error"></p>
  <button type="submit">Войти</button>
</form>

<script>
  const form = document.querySelector('#loginForm');
  const email = document.querySelector('#email');
  const error = document.querySelector('#error');

  form.addEventListener('submit', function (event) {
    if (email.value === '') {
      event.preventDefault();
      error.textContent = 'Введите email';
    }
  });
</script>
```

Что здесь происходит:

1. JavaScript находит форму и поле email.
2. Отслеживает отправку формы.
3. Если email пустой, останавливает отправку.
4. Показывает сообщение об ошибке.

# 10. Клиентская и серверная валидация

Валидация бывает двух основных типов.

## Клиентская валидация

Это проверка в браузере пользователя.

Например:

- HTML-атрибут `required`;
- проверка через JavaScript;
- подсветка ошибок прямо на странице.

Плюсы:

- работает быстро;
- помогает пользователю сразу увидеть ошибку;
- снижает количество неправильных отправок.

Минус: её можно обойти.

---

## Серверная валидация

Это проверка данных на сервере после отправки формы.

Например, сервер проверяет:

- существует ли такой пользователь;
- свободен ли email;
- правильный ли пароль;
- разрешён ли формат файла;
- не превышен ли лимит запроса.

Серверная валидация обязательна, потому что данные из браузера нельзя считать полностью безопасными.

Даже если на сайте есть JavaScript-проверка, злоумышленник может отправить запрос напрямую, минуя форму.

---

# 11. Пример: регистрация пользователя

Допустим, есть форма регистрации:

```html
<form>
  <input type="text" name="name" required>
  <input type="email" name="email" required>
  <input type="password" name="password" minlength="8" required>
  <button type="submit">Создать аккаунт</button>
</form>
```

Пользователь вводит данные.

На стороне браузера можно проверить:

- заполнено ли имя;
- похож ли email на email;
- не короче ли пароль 8 символов.

На стороне сервера нужно проверить:

- нет ли уже пользователя с таким email;
- корректные ли данные пришли;
- не содержит ли ввод опасные значения;
- можно ли создать пользователя;
- надёжно ли обработан пароль.

# 12. Отправка формы

Раньше формы часто отправлялись классическим способом: пользователь нажимает кнопку, браузер отправляет данные на сервер, страница перезагружается.

Пример:

```html
<form action="/signup" method="post">
  <input type="email" name="email">
  <button type="submit">Отправить</button>
</form>
```

Здесь:

- `action="/signup"` — куда отправить данные;
- `method="post"` — каким способом отправить.

---

## `GET` и `POST`

| Метод | Для чего обычно используется |
|---|---|
| `GET` | Получить данные, например поиск |
| `POST` | Отправить данные, например заявку или регистрацию |

Пример поиска:

```html
<form action="/search" method="get">
  <input type="text" name="q">
  <button type="submit">Найти</button>
</form>
```

После отправки URL может выглядеть так:

```text
/search?q=косметология
```

Пример заявки:

```html
<form action="/request" method="post">
  <input type="text" name="name">
  <input type="tel" name="phone">
  <button type="submit">Оставить заявку</button>
</form>
```

---

# 13. Отправка формы через `fetch`

В современных сайтах форму часто отправляют через JavaScript без перезагрузки страницы.

Упрощённый пример:

```html
<form id="requestForm">
  <input name="name" placeholder="Имя" required>
  <input name="phone" placeholder="Телефон" required>
  <button type="submit">Отправить</button>
</form>

<script>
  const form = document.querySelector('#requestForm');

  form.addEventListener('submit', async function (event) {
    event.preventDefault();

    const formData = new FormData(form);

    const response = await fetch('/api/request', {
      method: 'POST',
      body: formData
    });

    if (response.ok) {
      alert('Заявка отправлена');
      form.reset();
    } else {
      alert('Ошибка отправки');
    }
  });
</script>
```

Здесь форма не перезагружает страницу. JavaScript собирает данные и отправляет их на сервер.

---

# 14. Что такое `FormData`

`FormData` — это специальный объект JavaScript для сбора данных формы.

```js
const formData = new FormData(form);
```

Он автоматически берёт значения полей внутри формы, у которых есть `name`.

Например:

```html
<input name="email" value="test@example.com">
```

В `FormData` попадёт:

```text
email = test@example.com
```

`FormData` особенно удобен, когда нужно отправить файлы.

# 15. Ошибки в формах

Хорошая форма не просто запрещает отправку, а понятно объясняет, что нужно исправить.

Не очень хороший вариант:

```text
Ошибка
```

Лучше:

```text
Введите email
```

Ещё лучше:

```text
Введите email в формате name@example.com
```

Для пароля:

```text
Пароль должен быть не короче 8 символов
```

Для телефона:

```text
Введите номер телефона полностью
```

# 16. UX форм

**UX** — это удобство для пользователя.

Хорошая форма:

- имеет понятные подписи;
- показывает обязательные поля;
- не просит лишнего;
- объясняет ошибки рядом с нужным полем;
- сохраняет уже введённые данные после ошибки;
- использует подходящий тип поля;
- работает на телефоне;
- не заставляет пользователя угадывать формат.

Пример хорошей подписи:

```html
<label for="phone">Телефон для связи</label>
<input id="phone" type="tel" name="phone">
```

Пример плохой формы:

```html
<input placeholder="Введите данные">
```

Непонятно, какие именно данные нужны.

---

# 17. Доступность форм

Формы должны быть удобны не только визуально, но и технически.

Важно:

- использовать `<label>` для каждого поля;
- связывать `label` и `input` через `for` и `id`;
- не полагаться только на цвет ошибки;
- писать понятные тексты ошибок;
- делать элементы доступными с клавиатуры;
- не убирать видимый фокус без замены;
- помогать экранным дикторам понимать структуру формы.

Правильный пример:

```html
<label for="email">Email</label>
<input id="email" name="email" type="email">
```

Так браузер и вспомогательные технологии понимают, что поле связано с подписью «Email».

---

# 18. Частые ошибки новичков

## 1. Использовать только `placeholder` вместо `label`

Плохо:

```html
<input placeholder="Email">
```

Лучше:

```html
<label for="email">Email</label>
<input id="email" name="email" type="email" placeholder="name@example.com">
```

---

## 2. Не указывать `name`

```html
<input type="email">
```

Лучше:

```html
<input type="email" name="email">
```

Без `name` данные могут не отправиться.

---

## 3. Проверять форму только на клиенте

JavaScript-проверка полезна, но сервер всё равно должен проверять данные.

---

## 4. Делать слишком много обязательных полей

Чем длиннее форма, тем выше шанс, что пользователь её не заполнит.

---

## 5. Показывать непонятные ошибки

Плохо:

```text
Invalid value
```

Лучше:

```text
Введите корректный email
```

# 19. Мини-пример формы заявки

```html
<form id="leadForm">
  <label for="name">Имя</label>
  <input id="name" name="name" type="text" required>

  <label for="phone">Телефон</label>
  <input id="phone" name="phone" type="tel" required>

  <label for="service">Услуга</label>
  <select id="service" name="service" required>
    <option value="">Выберите услугу</option>
    <option value="consultation">Консультация</option>
    <option value="diagnostics">Диагностика</option>
    <option value="procedure">Процедура</option>
  </select>

  <label>
    <input type="checkbox" name="agreement" required>
    Я согласен на обработку персональных данных
  </label>

  <button type="submit">Отправить заявку</button>
</form>
```

Это типичная форма заявки: имя, телефон, выбор услуги и согласие.

# 20. Кратко

**Форма** — это способ получить данные от пользователя.

**Валидация** — это проверка этих данных.

Формы нужны для:

- регистрации;
- входа;
- заявок;
- заказов;
- поиска;
- обратной связи;
- анкет;
- загрузки файлов.

Валидация нужна, чтобы данные были корректными, понятными и безопасными.

Для начинающего веб-кодера важно запомнить:

- у формы есть поля и кнопка отправки;
- у полей должен быть `name`;
- подписи лучше делать через `<label>`;
- HTML умеет делать простую валидацию;
- JavaScript даёт больше контроля;
- серверная проверка всё равно обязательна;
- хорошие формы помогают пользователю, а не просто ругают его за ошибки.
