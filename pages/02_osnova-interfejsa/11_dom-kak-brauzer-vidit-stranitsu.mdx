# DOM: как браузер видит страницу

## Коротко

DOM — это внутреннее представление веб-страницы, которое браузер создаёт на основе HTML.

Когда браузер получает HTML-документ, он не просто показывает его как готовую картинку. Он разбирает разметку, распознаёт элементы и строит из них связанную структуру. В этой структуре заголовки, кнопки, формы, изображения и другие элементы становятся объектами, с которыми может работать JavaScript.

Благодаря DOM страница может изменяться прямо во время работы:

* открывается меню;
* меняется текст кнопки;
* появляется сообщение об ошибке;
* добавляются карточки;
* обновляется корзина;
* показываются полученные с сервера данные.

Понимание DOM помогает точнее объяснять ИИ-ассистенту, какой элемент нужно найти, что в нём изменить, какое событие обработать и почему интерактивность может не работать.

## Что это такое

DOM расшифровывается как **Document Object Model** — объектная модель документа.

Простыми словами, DOM — это модель страницы внутри браузера.

HTML описывает исходную структуру:

```html
<main>
  <h1>Наши услуги</h1>
  <button>Записаться</button>
</main>
```

После загрузки браузер превращает эту разметку в систему связанных объектов.

Условно она выглядит так:

```text
Документ
└── main
    ├── h1
    │   └── «Наши услуги»
    └── button
        └── «Записаться»
```

Такую структуру часто называют **деревом DOM**.

Она напоминает дерево, потому что начинается с общего документа и постепенно разветвляется на вложенные элементы.

В этой структуре можно увидеть:

* какой элемент является родительским;
* какие элементы находятся внутри него;
* какие элементы расположены рядом;
* где находится текст;
* какие атрибуты принадлежат элементу.

JavaScript может обращаться к объектам в этом дереве и изменять их.

Например, он может найти кнопку, заменить её текст или добавить ей CSS-класс.

## DOM и HTML — не одно и то же

HTML и DOM тесно связаны, но это разные понятия.

**HTML** — это текстовая разметка, которая находится в файле или приходит с сервера.

**DOM** — это текущее представление страницы, созданное браузером.

Когда JavaScript изменяет страницу, DOM обновляется, но исходный HTML-файл обычно не переписывается.

Например, в HTML находится такой заголовок:

```html
<h1>Запись на консультацию</h1>
```

После загрузки JavaScript может изменить его:

```javascript
heading.textContent = 'Запись подтверждена';
```

Пользователь увидит новый текст. В DOM заголовок тоже будет содержать новый текст, но в исходном HTML-файле по-прежнему может оставаться фраза «Запись на консультацию».

Поэтому полезно различать:

* исходную разметку;
* структуру, которую браузер создал после загрузки;
* текущее состояние страницы после работы JavaScript.

Это особенно важно при поиске ошибок. Элемент может отсутствовать в исходном HTML, но быть добавлен JavaScript позднее. Или, наоборот, элемент есть в файле, но после загрузки был удалён из DOM.

## Из чего состоит DOM

DOM состоит из узлов.

**Узел** — это отдельная часть структуры документа.

Узлами могут быть:

* сам документ;
* HTML-элементы;
* текст внутри элементов;
* комментарии;
* атрибуты и другие части страницы.

Для базовой работы чаще всего важны именно элементы.

Например:

```html
<section>
  <h2>Преимущества</h2>
  <p>Персональный план процедур.</p>
</section>
```

Здесь можно выделить:

* элемент `section`;
* вложенный элемент `h2`;
* текст внутри заголовка;
* вложенный элемент `p`;
* текст внутри абзаца.

Элементы связаны между собой отношениями.

### Родительский элемент

Элемент, внутри которого находится другой элемент.

В примере `section` является родителем для `h2` и `p`.

### Дочерний элемент

Элемент, вложенный внутрь другого.

`h2` и `p` являются дочерними элементами для `section`.

### Соседние элементы

Элементы, находящиеся на одном уровне внутри общего родителя.

`h2` и `p` являются соседними элементами.

Понимание этих связей помогает объяснять ИИ, где именно должно произойти изменение.

Например:

> Добавь сообщение об ошибке внутрь блока формы сразу после поля телефона.

Это точнее, чем:

> Добавь ошибку где-нибудь рядом с телефоном.

## Как браузер создаёт DOM

Когда пользователь открывает страницу, браузер получает HTML и начинает читать его сверху вниз.

Упрощённо процесс выглядит так:

1. Браузер получает HTML-документ.
2. Распознаёт теги и вложенность.
3. Создаёт объекты для элементов.
4. Связывает их в дерево.
5. Подключает CSS.
6. Рассчитывает расположение и внешний вид элементов.
7. Отображает результат на экране.
8. Выполняет JavaScript, который может изменить DOM.

Браузер старается исправлять некоторые ошибки HTML автоматически.

Например, если тег был закрыт неправильно, браузер может самостоятельно перестроить структуру. Из-за этого DOM иногда отличается от того, что ожидал разработчик.

Страница при этом может выглядеть нормально, но JavaScript или CSS начнут обращаться не к тем элементам.

Поэтому корректная HTML-структура важна не только для порядка в коде. От неё зависит то, какое DOM-дерево создаст браузер.

## Как DOM связан с JavaScript

JavaScript использует DOM как интерфейс для работы со страницей.

Он может:

* найти элемент;
* прочитать его текст;
* изменить содержимое;
* получить значение поля;
* добавить или удалить класс;
* изменить атрибут;
* создать новый элемент;
* переместить элемент;
* удалить элемент;
* обработать действие пользователя.

Например, на странице есть кнопка:

```html
<button id="open-form">Записаться</button>
```

JavaScript может найти её:

```javascript
const button = document.querySelector('#open-form');
```

После этого к кнопке можно добавить реакцию на нажатие:

```javascript
button.addEventListener('click', () => {
  console.log('Кнопка нажата');
});
```

Общий принцип выглядит так:

1. JavaScript находит объект в DOM.
2. Подключает к нему действие или изменяет его.
3. Браузер обновляет страницу.
4. Пользователь видит результат.

DOM является связующим звеном между JavaScript-кодом и видимым интерфейсом.

## Как JavaScript находит элементы

Чтобы изменить элемент, JavaScript сначала должен его найти.

Для поиска могут использоваться:

* название HTML-тега;
* CSS-класс;
* уникальный идентификатор;
* специальный атрибут;
* положение элемента внутри другого блока.

Например, можно найти:

* кнопку с идентификатором `open-form`;
* все карточки с классом `service-card`;
* поле с атрибутом `name="phone"`;
* сообщение внутри конкретной формы.

Для этого часто используется метод `querySelector`.

```javascript
const form = document.querySelector('.appointment-form');
```

Он возвращает первый элемент, который соответствует указанному селектору.

Для поиска нескольких элементов можно использовать `querySelectorAll`:

```javascript
const cards = document.querySelectorAll('.service-card');
```

При работе с ИИ важно сообщать, как в проекте обозначены нужные элементы.

Например:

> Найди форму по классу `appointment-form`, а кнопку отправки — по атрибуту `type="submit"`.

Если ИИ придумает селектор, которого нет в HTML, JavaScript не найдёт элемент.

Это одна из самых частых причин, почему код выглядит правильным, но ничего не происходит.

## Изменение содержимого страницы

Через DOM JavaScript может менять текст и другие части содержимого.

Например:

```javascript
title.textContent = 'Спасибо за заявку';
```

Такой код заменит текст выбранного элемента.

JavaScript также может изменить HTML-содержимое блока. Но с этой возможностью нужно обращаться осторожно, особенно если внутрь добавляются данные, введённые пользователем или полученные из ненадёжного источника.

Безопаснее использовать методы, которые добавляют текст как обычный текст, а не превращают его в HTML-разметку.

При работе с ИИ можно указать:

> Не вставляй пользовательские данные через `innerHTML`. Используй безопасное добавление текста и элементов DOM.

Это помогает снизить риск того, что посторонний текст будет воспринят браузером как код страницы.

## Изменение классов и атрибутов

Часто JavaScript не меняет оформление напрямую, а переключает CSS-класс.

Например:

```javascript
menu.classList.add('is-open');
```

JavaScript сообщает, что меню перешло в открытое состояние.

CSS уже определяет, как это состояние выглядит:

```css
.menu.is-open {
  display: block;
}
```

Классы можно:

* добавлять;
* удалять;
* переключать;
* проверять.

Это позволяет разделить ответственность:

* JavaScript управляет состоянием;
* CSS управляет внешним видом;
* HTML сохраняет структуру.

JavaScript также может изменять атрибуты.

Например:

* `disabled` — отключить кнопку;
* `hidden` — скрыть элемент;
* `aria-expanded` — сообщить о состоянии раскрывающегося блока;
* `src` — изменить адрес изображения;
* `href` — изменить адрес ссылки.

При этом важно менять не только визуальное состояние, но и смысловые атрибуты.

Например, если меню открылось, одной смены класса может быть недостаточно. Для доступности полезно также обновить атрибут, сообщающий программам экранного доступа, что меню раскрыто.

## Создание и удаление элементов

DOM позволяет создавать элементы после загрузки страницы.

Например, JavaScript может получить список услуг с сервера и сформировать для каждой услуги карточку.

Упрощённо процесс выглядит так:

1. Получить данные.
2. Создать элемент карточки.
3. Добавить заголовок.
4. Добавить описание и цену.
5. Добавить кнопку.
6. Поместить карточку в список на странице.

Элементы можно создавать с помощью JavaScript:

```javascript
const card = document.createElement('article');
```

Затем можно добавить содержимое и вставить элемент в нужное место DOM.

Таким способом создаются:

* карточки каталога;
* результаты поиска;
* сообщения;
* уведомления;
* пункты списка;
* строки таблицы;
* комментарии;
* элементы корзины.

Элементы также можно удалять:

```javascript
message.remove();
```

Важно понимать, откуда берутся данные и что должно происходить при их обновлении.

Например, перед повторным отображением списка может потребоваться удалить старые карточки. Иначе одинаковые элементы будут добавляться снова и снова.

## События в DOM

DOM позволяет JavaScript реагировать на события.

Событие — это то, что произошло на странице.

Например:

* пользователь нажал кнопку;
* ввёл текст;
* отправил форму;
* выбрал значение;
* прокрутил страницу;
* навёл курсор;
* нажал клавишу;
* загрузка страницы завершилась.

JavaScript может назначить обработчик события:

```javascript
button.addEventListener('click', openForm);
```

Это означает:

> Когда на этой кнопке произойдёт нажатие, выполни функцию `openForm`.

Событие обычно содержит информацию о том:

* на каком элементе оно произошло;
* какое действие выполнил пользователь;
* какая клавиша была нажата;
* какое значение изменилось;
* нужно ли отменить стандартное поведение браузера.

При постановке задачи ИИ полезно разделять:

* источник события;
* тип события;
* ожидаемое действие;
* условия выполнения;
* изменение DOM после события.

Например:

> Когда пользователь отправляет форму, останови стандартную перезагрузку страницы, проверь поля и покажи ошибки рядом с соответствующими полями.

## Всплытие событий

Некоторые события проходят от вложенного элемента к его родителям. Это называется **всплытием события**.

Например, пользователь нажал на кнопку внутри карточки:

```text
Список карточек
└── Карточка
    └── Кнопка
```

Событие сначала относится к кнопке, а затем может быть замечено карточкой и общим списком.

Это позволяет назначить один обработчик общему контейнеру вместо отдельного обработчика для каждой кнопки.

Такой подход называется **делегированием событий**.

Он полезен, когда:

* на странице много одинаковых элементов;
* элементы создаются динамически;
* список может обновляться;
* нет смысла подключать отдельный обработчик к каждой карточке.

Например, один обработчик списка может определить, на какую именно кнопку внутри карточки нажал пользователь.

Для начинающего необязательно сразу разбираться во всех деталях всплытия. Достаточно помнить: одно нажатие может быть замечено несколькими вложенными уровнями DOM.

Из-за этого иногда одно действие выполняется несколько раз или срабатывает не на том элементе.

В такой ситуации можно попросить ИИ:

> Проверь всплытие события и объясни, почему обработчик на родительском контейнере срабатывает при нажатии на вложенную кнопку.

## DOM и CSS

CSS применяет стили к элементам DOM.

Когда браузер строит страницу, он сопоставляет CSS-селекторы с текущими элементами.

Например, правило:

```css
.notification.is-visible {
  opacity: 1;
}
```

будет применено к элементу, который одновременно имеет классы `notification` и `is-visible`.

Если JavaScript добавит класс `is-visible`, браузер пересчитает подходящие стили и обновит отображение.

Поэтому DOM, CSS и JavaScript работают вместе:

1. HTML создаёт исходную структуру.
2. Браузер превращает её в DOM.
3. CSS оформляет элементы DOM.
4. JavaScript меняет DOM и его состояния.
5. Браузер обновляет вид страницы.

Если CSS-селектор не соответствует текущей DOM-структуре, оформление не применится.

Например, CSS ожидает кнопку внутри элемента `.card`, но JavaScript переместил кнопку за пределы карточки. После этого часть стилей может перестать работать.

## Текущее состояние страницы

DOM отражает текущее состояние интерфейса.

Например, в нём может быть видно:

* открыто или закрыто меню;
* какое сообщение показано;
* какие карточки загружены;
* какие поля заполнены;
* у какого элемента есть класс ошибки;
* какая вкладка активна;
* отключена ли кнопка;
* существует ли модальное окно.

Однако не всё состояние приложения обязательно хранится в DOM.

JavaScript может отдельно хранить:

* данные пользователя;
* список товаров;
* выбранные фильтры;
* статус загрузки;
* настройки интерфейса.

После изменения этих данных код обновляет DOM.

В небольшом проекте DOM иногда используется почти как основное хранилище состояния. В современных фреймворках состояние чаще хранится отдельно, а интерфейс строится на его основе.

Для базового понимания важно различать:

* данные приложения;
* текущее визуальное представление этих данных в DOM.

## DOM в современных фреймворках

React, Vue и другие frontend-инструменты тоже работают с DOM, но обычно скрывают часть прямых операций.

В обычном JavaScript разработчик может написать:

```javascript
title.textContent = 'Новый заголовок';
```

Во фреймворке разработчик чаще изменяет данные или состояние компонента, после чего инструмент сам обновляет нужную часть DOM.

Например, логика может звучать так:

> Измени состояние заголовка, а интерфейс обновится автоматически.

Некоторые инструменты используют промежуточные механизмы, например виртуальное представление DOM, чтобы определить, какие изменения нужно применить к реальной странице.

Но даже при использовании фреймворка браузер в итоге всё равно работает с настоящим DOM.

Поэтому понимание DOM остаётся полезным. Оно помогает разбираться:

* почему элемент появился;
* когда он был создан;
* почему обработчик не подключился;
* почему изменился фокус;
* откуда взялся определённый класс;
* почему интерфейс обновился не так, как ожидалось.

При работе с ИИ важно указывать технологию проекта.

Для обычного JavaScript запрос может звучать так:

> Найди элемент через DOM и переключай класс при нажатии.

Для React подход будет другим:

> Не изменяй DOM напрямую. Управляй состоянием компонента и формируй интерфейс на его основе.

## DOM и инструменты разработчика

DOM можно увидеть в инструментах разработчика браузера.

Обычно для этого используется вкладка **Elements** или **Элементы**.

В ней показывается не просто исходный HTML-файл, а текущее состояние DOM.

Там можно:

* раскрывать вложенные элементы;
* находить классы и атрибуты;
* смотреть применённые CSS-правила;
* временно менять текст;
* добавлять и удалять классы;
* проверять размеры элементов;
* находить нужный блок на странице.

Изменения, сделанные через инструменты разработчика, обычно временные. После обновления страницы они исчезают, потому что исходные файлы не изменились.

Это полезно для проверки гипотез.

Например, можно:

1. Найти скрытый блок.
2. Удалить у него класс, который скрывает элемент.
3. Посмотреть, появится ли он на странице.
4. Понять, связана ли проблема с CSS или JavaScript.
5. После этого попросить ИИ изменить нужный файл проекта.

## Почему элемент может не находиться

Одна из частых ошибок при работе с DOM возникает, когда JavaScript не может найти нужный элемент.

Причины могут быть разными.

### Неправильный селектор

В JavaScript указан класс или идентификатор, которого нет в HTML.

### Код запускается слишком рано

JavaScript пытается найти элемент до того, как браузер успел создать его в DOM.

### Элемент создаётся позднее

Нужный блок появляется только после запроса к серверу или действия пользователя.

### Элемент находится в другом компоненте

Код ищет его не в той части интерфейса.

### Идентификатор повторяется

Идентификатор должен быть уникальным, но по ошибке используется у нескольких элементов.

### HTML был перестроен браузером

Ошибка вложенности могла привести к неожиданной DOM-структуре.

Если `querySelector` не находит элемент, результатом часто становится значение `null`.

После этого следующая операция вызывает ошибку, потому что код пытается работать с несуществующим объектом.

При обращении к ИИ полезно передать:

* селектор;
* HTML нужного блока;
* место подключения скрипта;
* сообщение из консоли;
* момент, когда должен появиться элемент.

## DOM и производительность

Каждое заметное изменение DOM может заставить браузер пересчитать внешний вид или расположение элементов.

Для небольшой страницы это редко становится проблемой. Но если код постоянно добавляет, удаляет и перемещает сотни элементов, интерфейс может замедлиться.

Например, неэффективный код может:

* обновлять каждую строку таблицы по отдельности;
* несколько раз подряд перестраивать один блок;
* постоянно измерять размеры элементов;
* реагировать на каждое движение прокрутки без ограничений;
* полностью пересоздавать большой список при небольшом изменении.

На базовом уровне не нужно глубоко изучать оптимизацию. Достаточно помнить:

* не стоит изменять DOM без необходимости;
* крупные обновления лучше группировать;
* при большом количестве элементов нужно учитывать производительность;
* сначала важны правильность и понятность, а затем оптимизация.

ИИ можно попросить:

> Проверь, нет ли в этом коде лишних повторных изменений DOM. Сохрани простую структуру и предложи оптимизацию только там, где она действительно нужна.

## DOM и безопасность

Изменение DOM связано не только с интерфейсом, но и с безопасностью.

Особенно осторожно нужно вставлять на страницу данные, которые пришли:

* из формы;
* из комментариев;
* из адресной строки;
* из внешнего API;
* из базы данных;
* от другого пользователя.

Если такие данные без проверки вставить как HTML, браузер может воспринять их как часть разметки или исполняемый код.

Поэтому нельзя считать любые полученные строки безопасными.

Для обычного текста лучше использовать свойства и методы, которые добавляют именно текст.

Например, `textContent` обычно безопаснее, чем вставка строки через `innerHTML`.

При работе с ИИ можно сформулировать ограничение:

> Данные пользователя должны отображаться как текст. Не вставляй их в DOM через непроверенный HTML.

Важно также помнить: скрытие элемента в DOM не защищает данные. Если конфиденциальная информация уже передана браузеру, пользователь может найти её через инструменты разработчика или JavaScript.

Защита данных должна происходить на сервере.

## Доступность и DOM

DOM используется не только браузером и JavaScript.

Его структура помогает вспомогательным технологиям понимать страницу. Например, программы экранного доступа определяют:

* где находится заголовок;
* какой элемент является кнопкой;
* где начинается навигация;
* какое поле связано с подписью;
* открыто ли меню;
* появилось ли сообщение об ошибке.

Поэтому визуально работающий интерфейс может оставаться неудобным, если DOM построен неправильно.

Например, обычный `div` с обработчиком клика может выглядеть как кнопка, но не иметь её стандартных возможностей:

* выбора с клавиатуры;
* активации клавишей Enter или пробелом;
* понятной роли для программы экранного доступа;
* состояния отключения.

Лучше использовать подходящий HTML-элемент, а JavaScript подключать к нему дополнительное поведение.

При динамических изменениях также важно:

* управлять фокусом;
* сообщать о важных обновлениях;
* сохранять логичный порядок элементов;
* обновлять смысловые атрибуты;
* не удалять элемент, с которым пользователь сейчас взаимодействует, без необходимости.

## Почему это важно при работе с ИИ-ассистентом

Без понимания DOM сложно точно описывать задачи, связанные с поведением страницы.

Фраза:

> Сделай так, чтобы сообщение появлялось после отправки.

не объясняет:

* где должно появиться сообщение;
* существует ли оно заранее;
* нужно ли создать новый элемент;
* какой блок является его родителем;
* должно ли старое сообщение удаляться;
* как долго сообщение остаётся;
* куда перемещается фокус;
* что происходит при повторной отправке.

Более точный запрос:

> После успешной отправки формы создай сообщение внутри контейнера `.appointment-form`, размести его перед кнопкой отправки и добавь класс `success-message`. Перед созданием удали предыдущее сообщение, если оно уже существует. Не используй `innerHTML` для данных пользователя.

Понимание DOM помогает сообщить ИИ:

* какой элемент нужно найти;
* по какому селектору его искать;
* какое событие отслеживать;
* что нужно создать или удалить;
* куда вставить новый элемент;
* какие классы и атрибуты изменить;
* когда выполнить действие;
* какие части DOM нельзя перестраивать;
* используется ли обычный JavaScript или фреймворк.

## Как ставить задачу ИИ по работе с DOM

Хороший промт можно построить из нескольких частей.

### Контекст

Укажи, какой проект используется:

* обычные HTML, CSS и JavaScript;
* React;
* Vue;
* другой фреймворк;
* готовый сайт с существующей структурой.

### Исходный элемент

Сообщи, с чем нужно работать:

* кнопка;
* форма;
* список;
* карточка;
* модальное окно;
* сообщение.

### Способ поиска

Укажи существующий класс, идентификатор или атрибут.

### Событие

Объясни, что запускает действие:

* нажатие;
* отправка формы;
* ввод;
* выбор;
* загрузка страницы;
* получение ответа сервера.

### Изменение

Опиши, что должно произойти в DOM:

* добавить класс;
* заменить текст;
* создать элемент;
* удалить блок;
* изменить атрибут;
* переместить фокус.

### Дополнительные состояния

Укажи:

* загрузку;
* успех;
* ошибку;
* повторное действие;
* пустой результат.

Пример промта:

> В проекте используется обычный JavaScript. При нажатии на кнопку с классом `faq-question` нужно открыть связанный ответ, добавить вопросу класс `is-open` и обновить `aria-expanded`. Повторное нажатие должно закрывать ответ. Не меняй существующую HTML-структуру.

## Как проверять изменения DOM, созданные ИИ

### Проверь существование элементов

Убедись, что селекторы из JavaScript соответствуют реальным элементам HTML.

### Проверь момент запуска

Код должен выполняться после появления нужных элементов.

### Проверь первое действие

Работает ли сценарий после первой загрузки страницы?

### Проверь повторное действие

Не появляются ли дубликаты после нескольких нажатий?

### Проверь удаление элементов

Удаляется ли только нужный блок, а не весь контейнер?

### Проверь CSS-классы

Совпадают ли классы, которые добавляет JavaScript, с классами в CSS?

### Проверь атрибуты

Обновляются ли `disabled`, `hidden`, `aria-expanded` и другие состояния?

### Проверь динамические элементы

Работают ли обработчики у элементов, которые были добавлены после загрузки страницы?

### Проверь консоль

Нет ли ошибок, связанных с `null`, неправильным селектором или отсутствующим методом?

### Проверь доступность

Можно ли выполнить действие с клавиатуры? Сохраняется ли понятный порядок фокуса?

### Проверь безопасность

Не вставляются ли пользовательские данные как непроверенный HTML?

### Проверь другие части страницы

Изменение общего селектора может затронуть элементы, которые не относятся к задаче.

## Простая аналогия

HTML можно сравнить с инструкцией по сборке шкафа.

В инструкции описано:

* какие детали существуют;
* как они соединяются;
* какие полки находятся внутри;
* где расположены дверцы.

DOM — это уже собранный шкаф, который находится перед тобой.

С ним можно взаимодействовать:

* открыть дверцу;
* переставить полку;
* добавить ящик;
* убрать ненужную деталь;
* проверить, что находится внутри.

JavaScript в этой аналогии — человек или механизм, который выполняет изменения.

Исходная инструкция при этом может остаться прежней. Но текущая конструкция шкафа уже отличается от первоначальной схемы.

Так же браузер создаёт DOM на основе HTML, а JavaScript изменяет его во время работы страницы.

## Частые заблуждения новичков

### «DOM — это другое название HTML»

HTML — исходная разметка, а DOM — объектная структура, созданная браузером на её основе.

### «Если JavaScript изменил DOM, HTML-файл тоже изменился»

Изменение страницы в браузере обычно не переписывает файл проекта или документ на сервере.

### «В инструментах разработчика всегда показывается исходный HTML»

Вкладка Elements показывает текущее состояние DOM, которое уже могло измениться после выполнения JavaScript.

### «Любой элемент можно найти в любой момент»

Элемент может ещё не быть создан, уже быть удалён или появляться только после определённого действия.

### «Если элемент есть на экране, JavaScript обязательно найдёт его»

Код может использовать неправильный селектор или искать элемент в неподходящий момент.

### «Можно без ограничений вставлять строки через innerHTML»

Если строка содержит непроверенные внешние данные, это может привести к ошибкам и уязвимостям.

### «Чем больше прямых изменений DOM, тем лучше»

Лишние повторные изменения усложняют код и могут замедлять интерфейс.

### «Скрытый элемент исчезает из DOM»

Это зависит от способа скрытия. Элемент может оставаться в DOM, но не отображаться визуально.

### «Удаление кнопки защищает функцию»

Скрытие или удаление элемента интерфейса не заменяет проверку прав на сервере.

### «Во фреймворках DOM больше не используется»

Браузер всё равно отображает реальный DOM. Фреймворк лишь помогает управлять его обновлениями.

### «ИИ сам поймёт структуру страницы»

Без HTML, списка классов или описания компонентов ИИ может придумать селекторы и связи, которых нет в проекте.

### «Если изменение видно, значит оно реализовано правильно»

Интерфейс может выглядеть правильно, но содержать дубликаты, неправильные атрибуты, проблемы с доступностью или небезопасную вставку данных.

## Что важно запомнить

* DOM — это объектное представление страницы внутри браузера.
* Браузер создаёт DOM на основе HTML.
* HTML-файл и текущее DOM-дерево могут отличаться.
* DOM состоит из связанных узлов и элементов.
* Элементы имеют родителей, дочерние элементы и соседей.
* JavaScript использует DOM, чтобы находить и изменять части страницы.
* Через DOM можно менять текст, классы, атрибуты и структуру интерфейса.
* События связывают действия пользователя с изменениями DOM.
* CSS применяется к текущим элементам и классам DOM.
* Динамически созданные элементы тоже становятся частью DOM.
* Неправильный селектор или слишком ранний запуск кода часто приводят к ошибкам.
* Пользовательские данные нельзя без проверки вставлять как HTML.
* В современных фреймворках прямой работой с DOM часто управляет сам фреймворк.
* Инструменты разработчика позволяют увидеть текущее DOM-дерево и временно изменить его.
* При работе с ИИ нужно указывать структуру, селекторы, события и ожидаемые изменения.
* Человек определяет логику интерфейса и проверяет результат, а ИИ помогает реализовать работу с DOM.
