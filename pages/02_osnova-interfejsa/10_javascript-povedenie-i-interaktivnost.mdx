# JavaScript: поведение и интерактивность

## Коротко

JavaScript отвечает за поведение веб-страницы. Он помогает сайту реагировать на действия пользователя, изменять содержимое без полной перезагрузки, проверять формы, открывать меню и получать данные с сервера.

HTML создаёт структуру страницы, CSS оформляет её, а JavaScript добавляет действия и реакции.

Понимание JavaScript помогает тебе точнее объяснять ИИ-ассистенту, что должно происходить после нажатия кнопки, отправки формы, выбора значения или получения данных.

## Что такое JavaScript

JavaScript — это язык программирования, который широко используется в веб-разработке.

В браузере он может:

* реагировать на действия пользователя;
* изменять элементы страницы;
* показывать и скрывать блоки;
* проверять введённые данные;
* выполнять расчёты;
* отправлять запросы на сервер;
* получать и отображать новые данные;
* сохранять временную информацию;
* управлять состоянием интерфейса.

Например, пользователь нажимает кнопку «Показать цены», и на странице открывается дополнительный блок. Саму кнопку создаёт HTML, её внешний вид определяет CSS, а действие после нажатия выполняет JavaScript.

JavaScript используется не только в браузере. Он также может работать на сервере, в мобильных приложениях и других программах. Но в рамках веб-интерфейса его главная роль — управлять поведением страницы.

## Для чего используется JavaScript

JavaScript нужен в тех случаях, когда страница должна не просто показывать информацию, а взаимодействовать с пользователем.

С его помощью можно создать:

* выпадающее меню;
* модальное окно;
* переключатель вкладок;
* слайдер изображений;
* калькулятор стоимости;
* фильтрацию товаров или услуг;
* поиск по странице;
* интерактивную карту;
* форму с проверкой полей;
* уведомление об успешной отправке;
* загрузку новых данных без обновления страницы;
* личный кабинет с изменяемым содержимым.

Без JavaScript сайт всё ещё может содержать тексты, изображения, ссылки и формы. Но сложное интерактивное поведение будет ограничено.

При этом JavaScript не нужен для каждого элемента. Обычный заголовок, абзац или статичное изображение не требуют отдельной логики.

## Как это работает в общей картине

Когда браузер открывает страницу, он загружает HTML, CSS и JavaScript.

У каждой технологии своя роль:

* **HTML** описывает элементы страницы;
* **CSS** определяет их внешний вид;
* **JavaScript** управляет их поведением.

JavaScript получает доступ к структуре страницы через DOM — внутреннее представление HTML-документа в браузере.

Благодаря этому скрипт может:

1. Найти нужный элемент.
2. Узнать его текущее состояние.
3. Отреагировать на действие пользователя.
4. Изменить текст, стиль, содержимое или расположение элемента.
5. При необходимости отправить запрос на сервер.
6. Показать пользователю результат.

Например, при отправке формы записи JavaScript может проверить, заполнены ли обязательные поля. Затем он передаёт данные серверу и показывает сообщение об успешной отправке или ошибке.

## События и реакции

Большая часть интерактивности строится вокруг **событий**.

Событие — это действие или изменение, которое произошло на странице.

К событиям относятся:

* нажатие кнопки;
* ввод текста;
* выбор пункта;
* отправка формы;
* прокрутка страницы;
* наведение указателя;
* загрузка страницы;
* получение ответа от сервера.

JavaScript может отслеживать событие и запускать связанную с ним логику.

Упрощённо это выглядит так:

> Когда происходит определённое событие, выполни нужное действие.

Например:

> Когда пользователь нажимает кнопку мобильного меню, покажи список навигации.

Для работы с ИИ важно описывать не только элемент, но и событие, реакцию и ожидаемый результат.

## Условия и логика

JavaScript позволяет сайту вести себя по-разному в зависимости от ситуации.

Например:

* если поле пустое — показать ошибку;
* если пользователь вошёл в аккаунт — показать личный кабинет;
* если товар закончился — отключить кнопку заказа;
* если сервер вернул данные — показать их;
* если произошла ошибка — вывести уведомление;
* если выбран определённый город — изменить доступные услуги.

Такие правила называются логикой программы.

Когда ты ставишь задачу ИИ, полезно формулировать её в виде условий:

> Если пользователь не заполнил номер телефона, не отправляй форму и покажи понятную ошибку рядом с полем.

Такая формулировка точнее, чем просьба «сделай проверку формы».

## Переменные и данные

JavaScript работает с данными.

Данными могут быть:

* имя пользователя;
* выбранная услуга;
* стоимость заказа;
* содержимое корзины;
* список записей;
* результат поиска;
* статус загрузки;
* текст ошибки.

Для временного хранения данных используются **переменные**. Их можно представить как подписанные ячейки, в которых программа хранит значения во время работы.

Например, интерфейс может хранить информацию о том:

* открыто ли меню;
* какая вкладка выбрана;
* сколько товаров находится в корзине;
* загружаются ли данные;
* вошёл ли пользователь в аккаунт.

Когда значение меняется, JavaScript может обновить интерфейс.

## Функции

**Функция** — это отдельный набор действий, который выполняет конкретную задачу.

Например, одна функция может:

* открыть окно;
* проверить форму;
* рассчитать цену;
* загрузить список услуг;
* показать ошибку;
* отправить данные на сервер.

Функции помогают разделять логику на понятные части и повторно использовать действия.

При работе с ИИ можно просить не создавать всю логику одним большим блоком, а разделить её на небольшие функции с понятным назначением. Это упрощает проверку и дальнейшие изменения.

## Работа с сервером

JavaScript может обмениваться данными с сервером.

Например, когда пользователь:

* входит в аккаунт;
* отправляет форму;
* загружает список записей;
* выбирает доступное время;
* добавляет товар в корзину;
* обновляет профиль,

браузер может отправить запрос на сервер и получить ответ.

После этого JavaScript обновляет страницу без полной перезагрузки.

Важно понимать, что JavaScript в браузере обычно не должен напрямую работать с базой данных. Он обращается к серверу через API, а сервер уже проверяет запрос и взаимодействует с базой.

Эту границу важно учитывать при постановке задачи ИИ. Не стоит просить frontend-код напрямую сохранять конфиденциальные данные в базе.

## Синхронные и асинхронные действия

Некоторые операции выполняются сразу, а другие требуют времени.

Например, изменение текста на кнопке происходит почти мгновенно. Запрос к серверу может занять несколько секунд.

Операции, результат которых приходит позже, называют **асинхронными**.

Во время ожидания интерфейс должен сообщать пользователю, что происходит:

* показывать индикатор загрузки;
* временно отключать кнопку;
* предотвращать повторную отправку;
* отображать ошибку при неудаче;
* показывать полученные данные после завершения запроса.

При работе с ИИ полезно отдельно описывать состояния загрузки, успеха и ошибки. Иначе ассистент может создать только основной сценарий и не учесть, что сервер отвечает не мгновенно.

## JavaScript и состояние интерфейса

**Состояние** — это текущие данные, от которых зависит внешний вид и поведение интерфейса.

Например, форма может находиться в одном из состояний:

* ещё не заполнена;
* заполняется;
* проверяется;
* отправляется;
* успешно отправлена;
* содержит ошибку.

Меню может быть открыто или закрыто. Пользователь может быть авторизован или не авторизован. Данные могут загружаться или уже быть получены.

JavaScript отслеживает эти состояния и обновляет страницу.

В небольшом проекте состояние может управляться простыми средствами. В крупных приложениях для этого часто используются специальные возможности фреймворков и библиотек.

## JavaScript в современных проектах

В простом сайте JavaScript может находиться в одном или нескольких отдельных файлах.

В современных веб-приложениях логика часто распределяется между компонентами и модулями.

Например:

* отдельный модуль управляет формой;
* другой отвечает за меню;
* третий получает данные с сервера;
* четвёртый показывает уведомления.

Также могут использоваться библиотеки и фреймворки, например React, Vue или Angular. Они помогают организовать большие интерфейсы и управлять их состоянием.

Но независимо от инструмента основные принципы сохраняются: программа получает данные, реагирует на события, выполняет условия и обновляет интерфейс.

## Почему это важно при работе с ИИ-ассистентом

При создании JavaScript-логики недостаточно описать только внешний результат. Нужно объяснить сценарий поведения.

Хорошая постановка задачи обычно включает:

* какой элемент запускает действие;
* какое событие нужно отслеживать;
* что должно произойти;
* какие данные используются;
* какие условия нужно проверить;
* что показать во время загрузки;
* что делать при успехе;
* как обработать ошибку;
* какие части страницы можно изменять;
* какие существующие функции нельзя нарушить.

Например, вместо фразы:

> Сделай форму записи.

лучше написать:

> После отправки формы проверь обязательные поля. Если данные заполнены правильно, отключи кнопку, покажи состояние загрузки и отправь запрос на сервер. После успешного ответа очисти форму и покажи сообщение, а при ошибке восстанови кнопку и выведи понятное уведомление.

Такой промт описывает полный сценарий, а не только конечное действие.

При проверке JavaScript-кода стоит убедиться, что:

* логика запускается в нужный момент;
* один клик не вызывает действие несколько раз;
* ошибки не остаются незамеченными;
* пользователь получает понятную обратную связь;
* код не изменяет посторонние элементы;
* данные проверяются не только в браузере, но и на сервере;
* секретные ключи не находятся в frontend-коде;
* функции имеют понятное назначение;
* новое поведение не нарушает уже работающие части проекта.

## Простая аналогия

Представь веб-страницу как автомобиль.

HTML — это конструкция автомобиля: кузов, двери, руль и педали.

CSS — это его внешний вид: цвет, форма салона, материалы и оформление приборной панели.

JavaScript — это механизмы управления: нажатие педали, включение фар, работа поворотников и реакция приборов.

Кнопка может существовать и выглядеть правильно, но без логики нажатие на неё ничего не изменит.

## Частые заблуждения новичков

### JavaScript отвечает только за анимации

Анимации — лишь одна из возможностей. JavaScript также работает с формами, данными, серверными запросами, состоянием и логикой интерфейса.

### Любой элемент страницы требует JavaScript

Статичное содержимое часто можно создать с помощью HTML и CSS. Не стоит добавлять логику там, где она не нужна.

### Проверки в браузере достаточно

Пользователь может изменить или обойти frontend-код. Поэтому важные данные и права доступа должны дополнительно проверяться на сервере.

### Если код работает в основном сценарии, задача выполнена

Нужно также учитывать пустые данные, медленное соединение, повторные нажатия, ошибки сервера и неожиданные ответы.

### JavaScript может безопасно хранить секретные данные

Код, который работает в браузере, доступен пользователю. В нём нельзя хранить секретные ключи, пароли и другие закрытые данные.

### ИИ сам поймёт всю логику функции

ИИ может предположить поведение, но не знает бизнес-правила проекта. Условия, ограничения и сценарии нужно описывать явно.

### Чем больше JavaScript, тем современнее сайт

Избыточная логика может замедлить страницу и усложнить поддержку. JavaScript должен решать конкретные задачи.

## Что важно запомнить

* JavaScript управляет поведением и интерактивностью веб-страницы.
* Он реагирует на события и изменяет интерфейс.
* JavaScript работает с данными, условиями, функциями и состояниями.
* Через DOM он получает доступ к HTML-элементам.
* Через API он может обмениваться данными с сервером.
* Асинхронные действия требуют состояний загрузки, успеха и ошибки.
* Важные проверки должны выполняться не только в браузере, но и на сервере.
* Секретные данные нельзя хранить в frontend-коде.
* В промте для ИИ нужно описывать полный сценарий поведения.
* Человек определяет правила, ограничения и ожидаемый результат, а ИИ помогает реализовать логику.
