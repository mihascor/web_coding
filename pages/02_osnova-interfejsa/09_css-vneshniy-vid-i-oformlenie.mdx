# CSS: внешний вид и оформление

## Коротко

CSS отвечает за то, как выглядит веб-страница. С его помощью задают цвета, шрифты, размеры, отступы, расположение блоков, оформление кнопок и многие другие визуальные свойства интерфейса.

HTML определяет, какие элементы находятся на странице, а CSS объясняет браузеру, как эти элементы нужно показать пользователю.

Когда ты просишь ИИ-ассистента создать интерфейс, недостаточно сказать: «Сделай страницу красивой». Чтобы получить предсказуемый результат, полезно понимать, из каких решений складывается внешний вид сайта: цветовая палитра, типографика, размеры, сетка, отступы, состояния элементов и правила отображения на разных экранах.

## Что это такое

CSS расшифровывается как **Cascading Style Sheets** — каскадные таблицы стилей.

Это язык, с помощью которого описывают оформление HTML-элементов.

CSS может определять:

* цвет текста и фона;
* шрифт и размер текста;
* ширину и высоту элементов;
* внутренние и внешние отступы;
* границы и скругления;
* тени;
* расположение блоков;
* выравнивание содержимого;
* оформление при наведении;
* вид страницы на разных размерах экрана;
* простые переходы и анимации.

Обычное CSS-правило выглядит примерно так:

```css
.button {
  background-color: black;
  color: white;
  padding: 12px 24px;
}
```

Здесь браузеру сообщается, что элемент с классом `button` должен иметь чёрный фон, белый текст и определённые внутренние отступы.

Для общего понимания важно различать три части правила:

* **селектор** указывает, какой элемент нужно оформить;
* **свойство** обозначает, что именно нужно изменить;
* **значение** определяет, каким должно стать это свойство.

В примере `.button` — селектор, `color` — свойство, а `white` — его значение.

Тебе необязательно запоминать все CSS-свойства. Гораздо важнее понимать принцип: CSS находит нужные элементы страницы и применяет к ним правила оформления.

## Для чего это используется

Без CSS браузер всё равно сможет показать HTML-страницу, но она будет выглядеть как простой документ: заголовки, текст, ссылки, списки и изображения, расположенные почти без индивидуального оформления.

CSS превращает такую структуру в полноценный интерфейс.

С его помощью можно:

* оформить страницу в стиле бренда;
* расположить меню рядом с логотипом;
* собрать карточки в сетку;
* выделить основную кнопку;
* настроить читаемые размеры текста;
* создать расстояния между разделами;
* сделать форму понятной и аккуратной;
* показать пользователю, на какой элемент можно нажать;
* скрыть или показать отдельные части интерфейса;
* изменить расположение элементов на мобильном экране.

CSS нужен не только для красоты. Он также влияет на удобство использования сайта.

Например, кнопка может технически работать и без оформления. Но если она почти не отличается от обычного текста, пользователь может не понять, что на неё можно нажать.

Хорошее оформление помогает пользователю:

* быстрее находить нужную информацию;
* понимать структуру страницы;
* замечать важные действия;
* различать активные и неактивные элементы;
* читать текст без лишнего напряжения;
* пользоваться сайтом на разных устройствах.

## Как CSS связан с HTML

HTML создаёт элементы страницы, а CSS применяет к ним стили.

Например, в HTML может быть кнопка:

```html
<button class="appointment-button">Записаться</button>
```

CSS может использовать её класс:

```css
.appointment-button {
  border-radius: 8px;
  padding: 12px 20px;
}
```

Атрибут `class` помогает связать HTML-элемент с CSS-правилом.

Класс можно представить как понятную метку. Разработчик сообщает браузеру: «Этот элемент относится к типу кнопок записи». После этого CSS находит элементы с такой меткой и оформляет их.

Один класс можно применить к нескольким элементам. Например, все карточки услуг могут получить класс `service-card` и общее оформление.

У одного элемента также может быть несколько классов. Это позволяет сочетать базовые и дополнительные стили.

Например, у всех кнопок может быть общий класс, а у главной кнопки — дополнительный класс, который выделяет её цветом.

При работе с ИИ полезно просить использовать понятные названия классов:

* `header`;
* `main-navigation`;
* `service-card`;
* `appointment-form`;
* `primary-button`.

Названия вроде `block1`, `red-box` или `element123` хуже объясняют назначение элемента. Если позднее цвет или положение изменятся, такое название может стать нелогичным.

## Как CSS подключается к странице

CSS можно добавить на страницу несколькими способами.

### Отдельный CSS-файл

Чаще всего стили хранятся в отдельном файле, например:

```text
styles.css
```

HTML подключает этот файл, а браузер загружает содержащиеся в нём правила.

Такой подход помогает отделить структуру страницы от её оформления.

### Стили внутри HTML-документа

CSS можно разместить внутри специального блока `<style>` в HTML-файле.

Это может быть удобно для небольшой демонстрации или отдельной страницы, но в большом проекте стили обычно выносят в отдельные файлы.

### Стиль непосредственно у элемента

Оформление можно указать прямо внутри HTML-тега с помощью атрибута `style`.

Например:

```html
<p style="color: red;">Важное сообщение</p>
```

Такой способ называется **встроенным стилем**.

Он может быть полезен в отдельных ситуациях, но обычно усложняет поддержку проекта. Если встроенных стилей много, оформление оказывается разбросано по HTML, и его становится труднее изменять.

Поэтому в промте для ИИ можно сразу указать:

> Не добавляй стили непосредственно в HTML-элементы. Вынеси CSS в отдельный файл.

Это особенно важно, если ты планируешь развивать страницу, повторно использовать стили или передавать проект другому человеку.

## Как это работает в общей картине

Когда браузер открывает страницу, он получает HTML и строит структуру её элементов.

Затем браузер загружает CSS и определяет:

* какие правила относятся к каждому элементу;
* какие размеры нужно использовать;
* где расположить блоки;
* какие цвета и шрифты применить;
* какие правила имеют приоритет;
* как должен выглядеть итоговый интерфейс.

После этого браузер рассчитывает положение и размеры элементов и отображает страницу на экране.

В общей картине основные технологии интерфейса связаны так:

* **HTML** определяет структуру и смысл элементов;
* **CSS** отвечает за оформление и расположение;
* **JavaScript** добавляет поведение и интерактивность.

Например, на странице есть раскрывающееся меню.

HTML создаёт кнопку и список пунктов меню.

CSS определяет их размеры, цвет, положение и внешний вид.

JavaScript управляет открытием и закрытием меню.

При этом CSS тоже может реагировать на некоторые простые состояния без JavaScript. Например, изменять цвет кнопки, когда пользователь наводит на неё курсор.

Но CSS не предназначен для сложной бизнес-логики. Он не отправляет данные на сервер, не проверяет пароль и не создаёт запись в базе данных.

## Селекторы: как CSS находит элементы

Чтобы применить оформление, CSS должен понять, к каким элементам относится правило. Для этого используются селекторы.

Селектор может обращаться:

* ко всем элементам определённого типа;
* к элементу с конкретным классом;
* к элементу с уникальным идентификатором;
* к элементу внутри определённого блока;
* к элементу в конкретном состоянии.

Например, можно оформить:

* все заголовки второго уровня;
* все основные кнопки;
* только ссылки внутри меню;
* поле формы, в котором возникла ошибка;
* кнопку при наведении курсора.

Для начинающего особенно важно понимать классы. В большинстве случаев именно они используются как удобные и повторно применяемые метки для оформления элементов.

Слишком сложные селекторы могут сделать стили хрупкими. Если CSS сильно зависит от точной вложенности HTML, даже небольшое изменение структуры страницы может нарушить оформление.

Поэтому при работе с ИИ можно попросить:

> Используй простые CSS-классы и не создавай слишком длинные селекторы, зависящие от вложенности элементов.

## Что означает «каскадные» стили

В названии CSS есть слово «каскадные». Оно означает, что на один элемент могут одновременно влиять несколько правил.

Например:

* браузер применяет свои стандартные стили;
* общий файл задаёт оформление всех кнопок;
* отдельный класс изменяет цвет главной кнопки;
* правило для мобильного экрана уменьшает её размер.

Браузер должен решить, какие правила использовать в итоговом оформлении.

На результат влияют:

* место подключения правила;
* точность селектора;
* порядок правил;
* наследование некоторых свойств.

Это называется **каскадом**.

Из-за каскада иногда возникает ситуация: разработчик меняет цвет элемента, но на странице ничего не происходит. Причина может быть в том, что другое правило имеет более высокий приоритет или подключено позднее.

Для базовой работы необязательно сразу подробно изучать все правила приоритетов. Достаточно помнить: если стиль не применяется, проблема может быть не в самом свойстве, а в конкуренции нескольких CSS-правил.

При обращении к ИИ можно написать:

> Цвет кнопки не изменяется после редактирования CSS. Проверь каскад, специфичность селекторов и порядок подключения стилей.

**Специфичность** — это условная сила селектора. Чем точнее правило указывает на элемент, тем больше вероятность, что именно оно получит приоритет.

## Наследование стилей

Некоторые CSS-свойства могут передаваться от родительского элемента к вложенным элементам.

Например, если всему разделу задан определённый шрифт, текст внутри этого раздела обычно будет использовать тот же шрифт.

Это называется **наследованием**.

Наследование позволяет не повторять одинаковые правила для каждого абзаца или ссылки.

Однако наследуются не все свойства. Например, ширина, отступы и границы обычно не передаются дочерним элементам автоматически.

Если ИИ создаёт много одинаковых повторяющихся правил, можно попросить его проверить, какие стили лучше задать общему родительскому блоку.

## Блочная модель: из чего складывается размер элемента

Каждый элемент на странице можно представить как прямоугольную область.

У этой области есть несколько частей:

1. Содержимое.
2. Внутренний отступ.
3. Граница.
4. Внешний отступ.

Эта система называется **блочной моделью**, или **box model**.

### Содержимое

Это текст, изображение или другой материал внутри элемента.

### Внутренний отступ

Внутренний отступ, или `padding`, создаёт пространство между содержимым и границей элемента.

Например, благодаря внутреннему отступу текст кнопки не прижимается к её краям.

### Граница

Граница, или `border`, проходит вокруг содержимого и внутренних отступов.

Она может быть видимой, прозрачной или отсутствовать.

### Внешний отступ

Внешний отступ, или `margin`, создаёт расстояние между элементом и соседними блоками.

Новички часто путают внутренние и внешние отступы.

Простой ориентир:

* `padding` создаёт пространство внутри элемента;
* `margin` создаёт пространство снаружи элемента.

Понимание блочной модели помогает объяснить ИИ, почему блок выглядит слишком большим, текст прижат к краям или между карточками нет нужного расстояния.

## Расположение элементов на странице

CSS отвечает не только за цвет и шрифт, но и за общую композицию страницы.

Для расположения элементов часто используются две основные системы:

* **Flexbox**;
* **CSS Grid**.

### Flexbox

Flexbox удобен, когда нужно расположить элементы в одном основном направлении: в строку или столбец.

Например:

* логотип и меню в шапке;
* кнопки рядом друг с другом;
* элементы внутри карточки;
* выравнивание содержимого по центру.

### CSS Grid

Grid удобен для построения сеток из строк и колонок.

Например:

* каталог карточек;
* галерея;
* сложная структура страницы;
* блок с несколькими колонками.

На практике Flexbox и Grid часто используются вместе. Один отвечает за крупную сетку страницы, другой — за расположение элементов внутри отдельных компонентов.

Тебе необязательно знать все их свойства, чтобы работать с ИИ. Достаточно понимать, какой результат нужен.

Например:

> Расположи карточки услуг в сетку из трёх колонок на широком экране. Внутри каждой карточки используй Flexbox, чтобы кнопка оставалась внизу.

Такой промт намного точнее просьбы «красиво расставь карточки».

## Цвета, шрифты и визуальная система

В хорошем интерфейсе оформление обычно строится не из случайных значений, а из общей визуальной системы.

Она может включать:

* основные и дополнительные цвета;
* цвета текста и фона;
* шрифты;
* размеры заголовков;
* интервалы между элементами;
* радиусы скругления;
* стили кнопок;
* оформление форм;
* тени;
* ширину основных контейнеров.

Если каждый блок оформлять отдельно без общей системы, страница может выглядеть неоднородно.

Например, на одном экране могут случайно появиться:

* пять вариантов скругления;
* несколько почти одинаковых оттенков серого;
* разные размеры одинаковых заголовков;
* несогласованные отступы;
* кнопки с разным поведением.

Чтобы избежать этого, в проекте часто создают повторно используемые значения и общие правила.

В CSS для этого могут применяться **переменные**:

```css
:root {
  --primary-color: #222222;
  --border-radius: 8px;
}
```

Переменная позволяет один раз определить значение и использовать его в разных местах.

Если основной цвет бренда изменится, его можно будет заменить в одном месте, а не искать по всему проекту.

При работе с ИИ можно попросить:

> Создай небольшую систему CSS-переменных для основных цветов, отступов, скруглений и размеров текста. Не добавляй слишком много значений.

## Состояния элементов

Интерфейс должен показывать пользователю, что происходит с элементом.

У кнопки, ссылки или поля формы могут быть разные состояния:

* обычное;
* при наведении;
* при нажатии;
* в фокусе;
* отключённое;
* с ошибкой;
* во время загрузки.

Например, при наведении кнопка может немного менять цвет. Это помогает пользователю понять, что элемент интерактивный.

Состояние фокуса особенно важно для людей, которые перемещаются по странице с помощью клавиатуры. Если оформление фокуса полностью убрать, пользователь может потерять понимание, какой элемент сейчас выбран.

Поэтому просьба ИИ «убери все рамки вокруг кнопок и полей» может ухудшить доступность сайта.

Лучше сформулировать задачу так:

> Сделай состояние фокуса аккуратным и заметным, но не убирай его полностью.

## CSS и адаптивность

Страницу открывают на экранах разной ширины: смартфонах, планшетах, ноутбуках и больших мониторах.

CSS позволяет менять оформление в зависимости от размера экрана.

Например:

* уменьшать размеры заголовков;
* превращать три колонки в одну;
* переносить меню;
* изменять отступы;
* растягивать кнопку на всю ширину;
* скрывать второстепенные декоративные элементы.

Для этого используются специальные условия — **медиазапросы**.

Подробно адаптивная вёрстка рассматривается отдельно. На базовом уровне важно понимать: CSS должен описывать не только внешний вид страницы на одном экране, но и её поведение при изменении доступного пространства.

Если ИИ создал страницу только для широкого монитора, можно попросить:

> Проверь CSS на мобильной ширине. Устрани горизонтальную прокрутку, перестрой колонки и сделай текст и кнопки удобными для небольшого экрана.

## Почему это важно при работе с ИИ-ассистентом

ИИ может быстро сгенерировать CSS, но формулировка «сделай современно» оставляет слишком много неопределённости.

Для одного человека современный интерфейс — это крупные заголовки и яркие градиенты. Для другого — строгая типографика, много свободного пространства и спокойные цвета.

Поэтому полезно описывать оформление через конкретные параметры.

Например:

> Оформи страницу в спокойном минималистичном стиле. Используй светлый фон, тёмный текст, один акцентный цвет, умеренные скругления и достаточно свободного пространства. Не добавляй градиенты, яркие тени и декоративные анимации.

Можно также указать:

* основные цвета;
* желаемый стиль;
* плотность интерфейса;
* форму кнопок;
* размеры контейнера;
* количество колонок;
* правила для мобильных экранов;
* допустимость анимаций;
* наличие готового брендбука;
* необходимость сохранить существующий HTML.

Понимание CSS помогает разделять задачи.

Например, если нужно изменить только внешний вид, необязательно просить ИИ переписывать HTML или JavaScript.

Можно сформулировать так:

> Не меняй структуру HTML и поведение JavaScript. Обнови только CSS: увеличь отступы между разделами, сделай кнопки контрастнее и выровняй карточки по высоте.

Это снижает риск того, что ИИ случайно нарушит уже работающую часть проекта.

## Как проверять CSS, созданный ИИ

После получения результата полезно проверить несколько вещей.

### Сохранилась ли структура страницы

Иногда ИИ меняет HTML, хотя задача касалась только стилей. Это может повлиять на JavaScript или серверные шаблоны.

### Нет ли лишнего повторения

Если одинаковые свойства много раз повторяются, возможно, их стоит вынести в общий класс или переменную.

### Понятны ли названия классов

Класс должен объяснять назначение элемента, а не случайную деталь его текущего вида.

### Не используются ли слишком сложные селекторы

Длинные цепочки вложенных селекторов усложняют изменение страницы.

### Нет ли большого количества `!important`

Запись `!important` принудительно повышает приоритет правила.

Иногда она нужна, но большое количество таких правил часто говорит о проблемах с организацией CSS. Стили начинают конкурировать друг с другом, и дальнейшее изменение становится сложнее.

### Работает ли страница на разных экранах

Нужно проверить хотя бы широкий и мобильный варианты.

### Видны ли интерактивные состояния

Кнопки, ссылки и поля должны давать пользователю понятную обратную связь.

### Не ухудшилась ли читаемость

Стоит проверить:

* достаточно ли контрастны текст и фон;
* не слишком ли мелкий текст;
* не слишком ли длинные строки;
* достаточно ли расстояния между элементами.

### Не нарушено ли существующее оформление

Изменение общего класса может неожиданно затронуть несколько страниц или компонентов.

Поэтому полезно просить ИИ объяснять область влияния изменений.

## Простая аналогия

HTML можно сравнить с помещениями и конструкцией дома, а CSS — с его оформлением.

HTML определяет, где находятся стены, двери, окна и комнаты.

CSS определяет:

* цвет стен;
* напольное покрытие;
* освещение;
* расстояния между предметами;
* размеры мебели;
* общий стиль интерьера.

Один и тот же дом можно оформить совершенно по-разному, не меняя его основную конструкцию.

Но оформление не должно мешать назначению пространства. Очень красивый стул, на котором невозможно сидеть, остаётся плохим решением. Точно так же эффектная кнопка, которую трудно заметить или прочитать, ухудшает интерфейс.

## Частые заблуждения новичков

### «CSS нужен только для красоты»

CSS также отвечает за расположение элементов, читаемость, визуальную иерархию, адаптивность и понятные состояния интерфейса.

### «HTML и CSS можно не разделять»

Для небольшой демонстрации стили можно разместить прямо в HTML. Но в развивающемся проекте разделение структуры и оформления обычно упрощает поддержку.

### «Чем больше эффектов, тем современнее сайт»

Большое количество теней, градиентов, анимаций и сложных переходов не делает интерфейс автоматически лучше.

Оформление должно помогать пользователю, а не отвлекать его.

### «Все размеры лучше задавать в пикселях»

Пиксели используются часто, но CSS поддерживает разные единицы измерения. Некоторые из них лучше подходят для гибких интерфейсов и масштабирования текста.

Необязательно сразу изучать все единицы, но важно понимать, что фиксированные размеры подходят не для каждой задачи.

### «Можно исправить любой стиль с помощью `!important`»

`!important` может временно решить проблему, но большое количество таких правил часто усложняет проект и скрывает причину конфликта стилей.

### «Если элемент не виден, значит проблема в HTML»

Элемент может присутствовать в HTML, но быть скрыт, иметь нулевой размер, находиться за пределами экрана или сливаться с фоном из-за CSS.

### «Если страница хорошо выглядит на моём экране, задача решена»

Интерфейс может сломаться на другом размере экрана, при более длинном тексте, увеличенном масштабе браузера или другом системном шрифте.

### «ИИ сам поймёт, какой дизайн мне нужен»

ИИ может предложить оформление, но не знает фирменный стиль, аудиторию, ограничения и цель страницы, пока ты не передашь этот контекст.

### «CSS может реализовать любую логику интерфейса»

CSS умеет управлять оформлением и некоторыми простыми состояниями, но сложное поведение обычно требует JavaScript.

### «Сначала можно написать случайные стили, а потом легко привести их в порядок»

Чем больше в проекте случайных значений и повторяющихся правил, тем сложнее его поддерживать. Даже на небольшой странице полезно использовать понятную систему классов, цветов и отступов.

## Что важно запомнить

* CSS отвечает за внешний вид и расположение HTML-элементов.
* CSS-правило состоит из селектора, свойства и значения.
* Классы связывают HTML-элементы с повторно используемыми стилями.
* Каскад определяет, какое из конкурирующих правил получит приоритет.
* Блочная модель включает содержимое, внутренние отступы, границу и внешние отступы.
* Flexbox и Grid помогают управлять расположением элементов.
* CSS влияет не только на красоту, но и на удобство, читаемость и доступность.
* Оформление желательно строить как систему, а не как набор случайных значений.
* При работе с ИИ нужно конкретно описывать стиль, размеры, цвета, сетку и ограничения.
* Если требуется изменить только оформление, стоит прямо запретить ИИ менять HTML и JavaScript.
* Человек определяет цель и проверяет результат, а ИИ помогает создавать и улучшать стили.

## Промты для самостоятельного уточнения

* «Объясни мне простыми словами, как CSS используется в реальном веб-проекте и как он связан с HTML.»

* «Покажи короткий пример HTML-элемента и CSS-правила для него. Объясни селектор, свойство и значение без сложных деталей.»

* «Объясни на простом примере, чем margin отличается от padding.»

* «Расскажи, чем Flexbox отличается от CSS Grid и для каких задач подходит каждый вариант.»

* «Объясни простыми словами, что такое каскад, наследование и специфичность в CSS.»

* «Проверь мой CSS и найди повторяющиеся правила, слишком сложные селекторы и лишнее использование !important.»

* «Помоги составить CSS для страницы в спокойном минималистичном стиле. Сначала предложи небольшую систему цветов, размеров текста и отступов.»

* «Измени только оформление этой страницы. Не меняй HTML-структуру и JavaScript, а все изменения CSS объясни простыми словами.»
