# DOM: как браузер видит страницу

## Коротко

DOM — это внутреннее представление веб-страницы, которое браузер создаёт на основе HTML.

Для пользователя страница выглядит как текст, изображения, кнопки и формы. Браузер воспринимает её как связанную структуру элементов, с которой может работать JavaScript.

Понимание DOM помогает тебе объяснять ИИ-ассистенту, какой элемент нужно найти, изменить, добавить или удалить и после какого действия это должно произойти.

## Что такое DOM

DOM расшифровывается как **Document Object Model** — объектная модель документа.

Когда браузер получает HTML-файл, он разбирает его и создаёт из элементов страницы специальную структуру. Эта структура и называется DOM.

В DOM представлены:

* вся веб-страница;
* её основные разделы;
* заголовки и абзацы;
* ссылки и кнопки;
* изображения;
* формы и поля;
* атрибуты элементов;
* текстовое содержимое.

DOM можно воспринимать как рабочую версию HTML-страницы внутри браузера.

HTML — это исходное описание документа, а DOM — структура, которую браузер создал на его основе и с которой может взаимодействовать во время работы страницы.

## Для чего используется DOM

DOM позволяет браузеру и JavaScript работать с отдельными частями страницы.

Через DOM можно:

* найти нужный элемент;
* изменить текст;
* заменить изображение;
* добавить или удалить класс оформления;
* показать или скрыть блок;
* создать новый элемент;
* удалить существующий элемент;
* прочитать значение поля формы;
* отследить нажатие кнопки;
* обновить список без перезагрузки страницы.

Например, при нажатии кнопки «Показать подробнее» JavaScript находит через DOM нужный блок и делает его видимым.

Без DOM JavaScript не знал бы, с какой именно кнопкой, формой или частью страницы нужно работать.

## Как браузер создаёт DOM

Когда пользователь открывает страницу, браузер получает HTML-документ и начинает читать его сверху вниз.

Он распознаёт элементы и определяет связи между ними:

* какие элементы находятся внутри страницы;
* какие блоки вложены друг в друга;
* где расположены заголовки;
* какие ссылки принадлежат меню;
* какие поля находятся внутри формы.

После этого браузер создаёт DOM-дерево.

Оно называется деревом, потому что начинается с общего документа и постепенно разделяется на вложенные элементы.

Например, внутри страницы может находиться основная область, внутри неё — раздел с услугами, внутри раздела — карточки, а внутри каждой карточки — заголовок, описание и кнопка.

Такая структура помогает браузеру понимать не только список элементов, но и отношения между ними.

## Элементы, узлы и вложенность

Части DOM часто называют **узлами**.

Узлом может быть:

* HTML-элемент;
* текст внутри элемента;
* атрибут;
* сам документ;
* комментарий в разметке.

Для базового понимания достаточно воспринимать DOM как набор связанных элементов страницы.

У элементов есть отношения:

* **родительский элемент** — содержит другой элемент;
* **дочерний элемент** — находится внутри родительского;
* **соседние элементы** — расположены рядом внутри одного блока.

Например, карточка услуги может быть родительским элементом, а её название, описание и кнопка — дочерними.

Эти связи важны, когда JavaScript должен изменить не всю страницу, а конкретный элемент внутри определённого блока.

## Как DOM связан с HTML

HTML и DOM тесно связаны, но это не одно и то же.

HTML — это текстовая разметка, которую браузер получает от сервера или из файлов проекта.

DOM — это структура, созданная браузером после обработки этой разметки.

Если JavaScript изменил страницу, DOM может отличаться от первоначального HTML.

Например, исходный HTML содержит пустой список. После получения данных с сервера JavaScript добавляет в него карточки. Пользователь видит новые элементы, и они существуют в DOM, хотя их могло не быть в первоначальном HTML-файле.

Это важно при проверке работы сайта: содержимое, которое видно на экране, не всегда полностью записано в исходной разметке.

## Как DOM связан с JavaScript

JavaScript использует DOM как интерфейс для управления страницей.

Обычно логика выглядит так:

1. JavaScript находит нужный элемент.
2. Отслеживает действие пользователя или другое событие.
3. Выполняет заданную проверку.
4. Изменяет элемент или создаёт новый.
5. Браузер обновляет отображение страницы.

Например, пользователь выбирает услугу в форме. JavaScript получает выбранное значение, находит блок с ценой и обновляет его содержимое.

Через DOM JavaScript может работать как с отдельным элементом, так и с группой похожих элементов.

Поэтому при постановке задачи ИИ полезно точно описывать, какие элементы должны участвовать в сценарии.

## Как DOM связан с CSS

CSS оформляет элементы, которые находятся в DOM.

Когда JavaScript добавляет элементу новый класс, браузер применяет связанные с этим классом CSS-правила.

Благодаря этому можно:

* открыть мобильное меню;
* подсветить выбранную вкладку;
* показать сообщение об ошибке;
* изменить состояние кнопки;
* скрыть модальное окно;
* обозначить активный пункт навигации.

Например, JavaScript не обязательно должен напрямую задавать цвет кнопки. Он может добавить ей класс активного состояния, а нужный цвет будет определён в CSS.

Такой подход помогает разделять ответственность:

* HTML описывает элемент;
* CSS определяет его внешний вид;
* JavaScript меняет его состояние;
* DOM связывает эти части внутри браузера.

## Изменения DOM во время работы страницы

DOM не является неизменным.

После загрузки страницы он может обновляться много раз.

Изменения происходят, когда:

* пользователь открывает меню;
* появляется модальное окно;
* отправляется форма;
* загружаются данные с сервера;
* применяется фильтр;
* добавляется товар в корзину;
* переключается вкладка;
* показывается уведомление;
* удаляется запись из списка.

Такие изменения часто происходят без полной перезагрузки страницы.

Современные веб-приложения активно используют этот принцип: браузер сохраняет текущую страницу и обновляет только нужные части интерфейса.

## DOM и события

DOM помогает JavaScript узнавать о действиях пользователя.

Такие действия называются событиями.

К ним относятся:

* нажатие кнопки;
* ввод текста;
* выбор значения;
* отправка формы;
* наведение указателя;
* прокрутка страницы;
* изменение размера окна.

JavaScript может связать определённое событие с конкретным элементом DOM.

Например:

* при нажатии на кнопку открыть окно;
* при вводе текста обновить результаты поиска;
* при отправке формы проверить поля;
* при выборе категории показать подходящие карточки.

Для ИИ-ассистента такой сценарий лучше описывать последовательностью: элемент, событие, условие и результат.

## DOM в современных фреймворках

В обычном JavaScript разработчик может напрямую находить и изменять элементы DOM.

Во фреймворках, таких как React, Vue или Angular, работа часто организована иначе. Разработчик описывает состояние интерфейса, а фреймворк сам определяет, какие части DOM нужно обновить.

Например, вместо прямой команды «замени текст в этом элементе» приложение может изменить значение переменной, после чего фреймворк обновит связанный элемент.

Это не отменяет DOM. Браузер всё равно показывает страницу через DOM, но часть работы с ним берёт на себя фреймворк.

При обращении к ИИ важно сообщать, используется ли в проекте обычный JavaScript или определённый фреймворк. Подходы к изменению интерфейса могут отличаться.

## Почему это важно при работе с ИИ-ассистентом

Многие задачи для ИИ связаны с изменением элементов страницы.

Например:

* открыть меню после нажатия;
* изменить текст кнопки;
* добавить карточки из полученных данных;
* показать сообщение рядом с неправильным полем;
* удалить элемент из списка;
* переключить активную вкладку.

Чтобы ИИ правильно реализовал такую задачу, ему нужно понимать:

* какой элемент должен запускать действие;
* какой элемент нужно изменить;
* как этот элемент можно найти;
* когда должно происходить изменение;
* нужно ли добавить, удалить или заменить содержимое;
* должно ли изменение сохраняться после перезагрузки;
* используется ли обычный JavaScript или фреймворк.

Вместо запроса:

> Сделай, чтобы блок открывался.

лучше написать:

> При нажатии на кнопку «Подробнее» показывай связанный с ней блок описания. Повторное нажатие должно скрывать этот же блок, не затрагивая остальные карточки.

Такой запрос описывает конкретные элементы, событие и ожидаемое поведение.

При проверке результата стоит убедиться, что:

* изменяется нужный элемент;
* другие похожие элементы не затрагиваются;
* обработчик не запускается несколько раз;
* создаваемые элементы имеют правильную структуру;
* после изменения сохраняется корректное оформление;
* логика соответствует используемому фреймворку;
* элементы остаются доступными для клавиатуры и вспомогательных технологий.

## Простая аналогия

Представь страницу как организационную схему компании.

HTML — это документ, в котором перечислены отделы и сотрудники.

DOM — это живая схема внутри браузера, где видно, кто к какому отделу относится и какие элементы связаны между собой.

JavaScript может использовать эту схему, чтобы найти нужный отдел, добавить сотрудника, изменить информацию или удалить запись.

## Частые заблуждения новичков

### DOM — это то же самое, что HTML-файл

DOM создаётся на основе HTML, но может изменяться во время работы страницы. Поэтому текущее состояние DOM не всегда совпадает с исходным файлом.

### DOM — это отдельная технология для дизайна

DOM не отвечает за оформление. Стили задаются с помощью CSS, а DOM представляет структуру элементов.

### Любое изменение DOM сохраняется навсегда

Большинство изменений существует только в текущей вкладке браузера. После перезагрузки они исчезнут, если данные не были сохранены на сервере или в хранилище браузера.

### JavaScript изменяет сам HTML-файл проекта

JavaScript обычно изменяет DOM в браузере, а не исходный файл на сервере.

### Чем чаще обновляется DOM, тем лучше

Избыточные изменения могут усложнять логику и снижать производительность. Обновлять нужно только те части страницы, которые действительно изменились.

### ИИ всегда найдёт правильный элемент

Если на странице много одинаковых кнопок или блоков, ИИ может выбрать слишком общий способ поиска. Нужно уточнять, какой именно элемент связан с действием.

### При использовании React DOM больше не существует

DOM по-прежнему создаётся браузером. React и другие фреймворки лишь помогают управлять его обновлениями.

## Что важно запомнить

* DOM — это внутреннее представление веб-страницы в браузере.
* Он создаётся на основе HTML.
* DOM имеет древовидную структуру с вложенными элементами.
* Через DOM JavaScript находит и изменяет части страницы.
* CSS оформляет элементы, находящиеся в DOM.
* DOM может изменяться без перезагрузки страницы.
* Текущее состояние DOM может отличаться от исходного HTML.
* Изменения DOM обычно не сохраняются после перезагрузки автоматически.
* При работе с ИИ нужно точно описывать элемент, событие и ожидаемое изменение.
* Человек определяет логику интерфейса, а ИИ помогает реализовать работу с элементами страницы.

* «Расскажи, как связаны DOM, CSS-классы и изменение внешнего вида элемента».
