# npm и package.json

## Что такое npm

**npm** — это менеджер пакетов для JavaScript. Проще говоря, это инструмент, который помогает устанавливать, обновлять и удалять готовые библиотеки и инструменты для проекта. Изначально npm появился для Node.js, но сейчас активно используется и во фронтенд-разработке: React, Vue, Vite, TypeScript, сборщики, линтеры и многие другие инструменты обычно ставятся через npm. citeturn415874search3

Например, вместо того чтобы вручную скачивать библиотеку, ты пишешь в терминале:

```bash
npm install axios
```

И npm сам скачивает пакет, добавляет его в проект и записывает информацию о нём в настройки проекта.

---

## Что такое пакет

**Пакет** — это готовый кусок кода или инструмент, который можно подключить к своему проекту.

Примеры пакетов:

| Пакет | Для чего нужен |
|---|---|
| `react` | создание интерфейсов |
| `vite` | запуск и сборка фронтенд-проекта |
| `typescript` | добавление типизации в JavaScript |
| `axios` | выполнение HTTP-запросов |
| `eslint` | проверка качества кода |
| `prettier` | автоматическое форматирование кода |

Обычно пакеты берутся из **npm Registry** — большого онлайн-хранилища JavaScript-пакетов. Официальный сайт npm описывает npm Registry как центр обмена JavaScript-кодом с более чем двумя миллионами пакетов. citeturn415874search1

---

## Для чего нужен npm в веб-разработке

npm используется почти в каждом современном JavaScript-проекте.

С его помощью можно:

1. **Устанавливать библиотеки**

```bash
npm install react
```

2. **Устанавливать инструменты разработки**

```bash
npm install vite --save-dev
```

3. **Запускать команды проекта**

```bash
npm run dev
```

4. **Обновлять зависимости**

```bash
npm update
```

5. **Создавать новый проект**

Например, многие современные шаблоны проектов создаются через команды npm или похожие менеджеры пакетов.

# Что такое `package.json`

`package.json` — это главный конфигурационный файл JavaScript-проекта. Он лежит в корне проекта и описывает сам проект: его название, версию, зависимости, команды запуска и другие настройки.

Официальная документация npm описывает `package.json` как файл, в котором указываются данные пакета, зависимости и скрипты; при публикации пакета особенно важны поля `name` и `version`. citeturn415874search0turn415874search6

Пример простого `package.json`:

```json
{
  "name": "my-first-site",
  "version": "1.0.0",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "axios": "^1.7.0"
  },
  "devDependencies": {
    "vite": "^5.0.0"
  }
}
```

---

## Главные части `package.json`

### 1. `name`

Название проекта или пакета.

```json
"name": "my-first-site"
```

Если проект не публикуется в npm, это поле нужно в основном для порядка и идентификации проекта.

---

### 2. `version`

Версия проекта.

```json
"version": "1.0.0"
```

Обычно используется формат:

```text
major.minor.patch
```

Например:

```text
1.4.2
```

Это называется **семантическое версионирование**.

Условно:

| Часть | Значение |
|---|---|
| `1` | большая версия, могут быть серьёзные изменения |
| `4` | новая функциональность |
| `2` | исправления ошибок |

---

### 3. `scripts`

Это команды, которые можно запускать через npm.

Пример:

```json
"scripts": {
  "dev": "vite",
  "build": "vite build"
}
```

После этого в терминале можно писать:

```bash
npm run dev
```

или:

```bash
npm run build
```

Поле `scripts` в `package.json` хранит команды проекта в виде словаря: название команды и то, что должно выполниться. citeturn415874search0

Для начинающего веб-кодера это одна из самых важных частей `package.json`.

Обычно:

| Команда | Что делает |
|---|---|
| `npm run dev` | запускает проект в режиме разработки |
| `npm run build` | собирает проект для публикации |
| `npm run preview` | показывает собранную версию |
| `npm test` | запускает тесты, если они настроены |

---

### 4. `dependencies`

Это зависимости, которые нужны проекту **для работы**.

Например:

```json
"dependencies": {
  "axios": "^1.7.0",
  "react": "^18.2.0"
}
```

Если приложение использует React или Axios в реальном коде сайта, они должны быть в `dependencies`.

---

### 5. `devDependencies`

Это зависимости, которые нужны **только во время разработки**.

Например:

```json
"devDependencies": {
  "vite": "^5.0.0",
  "eslint": "^8.0.0",
  "prettier": "^3.0.0"
}
```

Сюда обычно попадают сборщики, линтеры, форматтеры, тестовые инструменты и другие вспомогательные программы.

Официальная документация npm объясняет, что зависимости проекта указываются в `dependencies` или `devDependencies`, а при запуске `npm install` npm скачивает зависимости, перечисленные в `package.json`. citeturn415874search4

# Что происходит при `npm install`

Когда ты пишешь:

```bash
npm install
```

npm смотрит в `package.json`, находит список зависимостей и скачивает их.

После этого обычно появляются:

```text
node_modules/
package-lock.json
```

---

## Что такое `node_modules`

`node_modules` — это папка, куда npm складывает установленные пакеты.

Например, если ты установила Vite, React или Axios, их код окажется внутри `node_modules`.

Эту папку обычно **не отправляют в GitHub**, потому что она может быть очень большой. Вместо неё в репозиторий отправляют `package.json` и `package-lock.json`, а другой разработчик потом просто запускает:

```bash
npm install
```

И npm заново скачивает всё нужное.

---

## Что такое `package-lock.json`

`package-lock.json` — это файл, который фиксирует точные версии установленных зависимостей.

Почему это важно?

В `package.json` может быть написано:

```json
"axios": "^1.7.0"
```

Знак `^` означает, что npm может установить совместимую более новую версию, например `1.7.5` или `1.8.0`.

А `package-lock.json` запоминает конкретно установленную версию, чтобы у всех участников проекта зависимости были одинаковыми.

# Как npm и `package.json` связаны между собой

Связь простая:

```text
npm — это инструмент
package.json — это инструкция для npm
```

npm читает `package.json` и понимает:

- как называется проект;
- какие пакеты нужно установить;
- какие команды можно запускать;
- какие инструменты нужны для разработки;
- как собирать или запускать проект.

---

## Пример из жизни

Допустим, ты скачала проект с GitHub.

Внутри есть:

```text
index.html
src/
package.json
package-lock.json
```

Но нет папки `node_modules`.

Чтобы проект заработал, ты открываешь терминал в папке проекта и пишешь:

```bash
npm install
```

npm скачивает все зависимости.

Потом запускаешь проект:

```bash
npm run dev
```

И dev server открывает сайт в браузере.

# Частые команды npm

| Команда | Что делает |
|---|---|
| `npm init` | создаёт новый `package.json` |
| `npm init -y` | создаёт `package.json` с настройками по умолчанию |
| `npm install` | устанавливает все зависимости из `package.json` |
| `npm install package-name` | устанавливает конкретный пакет |
| `npm install package-name --save-dev` | устанавливает пакет как инструмент разработки |
| `npm run dev` | запускает скрипт `dev` |
| `npm run build` | запускает скрипт `build` |
| `npm uninstall package-name` | удаляет пакет |

Создать новый `package.json` можно через `npm init`, а вариант с настройками по умолчанию часто используется как быстрый старт. citeturn415874search9

# Что важно запомнить начинающему

**npm** нужен, чтобы управлять пакетами и командами проекта.

**`package.json`** — это файл, который описывает проект и говорит npm, что нужно установить и какие команды доступны.

**`dependencies`** — то, что нужно сайту или приложению для работы.

**`devDependencies`** — то, что нужно разработчику во время создания проекта.

**`node_modules`** — папка с установленными пакетами.

**`package-lock.json`** — файл, который фиксирует точные версии зависимостей.

Главная практическая мысль: когда ты работаешь с современным веб-проектом, почти всегда сначала смотришь в `package.json`, затем запускаешь `npm install`, а потом одну из команд из `scripts`, например `npm run dev`.
