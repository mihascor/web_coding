# Адаптивная верстка

## Адаптивная верстка: что это

**Адаптивная верстка** — это подход к созданию сайтов, при котором страница удобно отображается на разных устройствах:

- смартфонах;
- планшетах;
- ноутбуках;
- больших мониторах.

Один и тот же сайт должен быть читаемым и удобным независимо от размера экрана.

Например, на компьютере блоки могут стоять в три колонки:

```text
[ Карточка 1 ] [ Карточка 2 ] [ Карточка 3 ]
```

А на телефоне те же блоки выстраиваются в одну колонку:

```text
[ Карточка 1 ]
[ Карточка 2 ]
[ Карточка 3 ]
```

Это и есть идея адаптивности.

---

## Зачем нужна адаптивная верстка

Пользователи заходят на сайты с очень разных экранов. Если сайт сделан только под компьютер, на телефоне могут появиться проблемы:

- текст слишком мелкий;
- элементы не помещаются на экран;
- кнопки трудно нажимать;
- приходится постоянно увеличивать страницу;
- горизонтальная прокрутка мешает пользоваться сайтом.

Адаптивная верстка нужна, чтобы сайт был **удобным, читаемым и аккуратным** на любом устройстве.

---

## Простая аналогия

Представь текстовый документ.

Если открыть его на большом экране, строки могут быть широкими.  
Если открыть на маленьком экране, текст должен переноситься и занимать меньше места по ширине.

Адаптивная верстка работает похожим образом: сайт «подстраивается» под доступное пространство.

---

## Из чего состоит адаптивная верстка

### 1. Гибкая структура

Вместо жестких размеров часто используют гибкие значения.

Например, плохо:

```css
.container {
  width: 1200px;
}
```

Такой блок всегда будет шириной 1200 пикселей. На телефоне он просто не поместится.

Лучше:

```css
.container {
  max-width: 1200px;
  width: 100%;
}
```

Теперь блок может быть максимум 1200 пикселей, но на маленьком экране он сожмется до доступной ширины.

---

## 2. Относительные единицы измерения

В адаптивной верстке часто используют не только пиксели, но и относительные единицы:

| Единица | Что означает |
|---|---|
| `%` | процент от родительского элемента |
| `em` | размер относительно шрифта элемента |
| `rem` | размер относительно базового размера шрифта страницы |
| `vw` | процент от ширины окна браузера |
| `vh` | процент от высоты окна браузера |

Например:

```css
.card {
  width: 50%;
}
```

Блок займет половину ширины родителя.

А так:

```css
.title {
  font-size: 2rem;
}
```

Размер заголовка будет зависеть от базового размера шрифта страницы.

---

## 3. Media queries

**Media queries** — это правила CSS, которые позволяют применять разные стили в зависимости от размера экрана.

Пример:

```css
.cards {
  display: flex;
  gap: 20px;
}

.card {
  width: 33.333%;
}

@media (max-width: 768px) {
  .cards {
    flex-direction: column;
  }

  .card {
    width: 100%;
  }
}
```

Что здесь происходит:

- на большом экране карточки стоят в ряд;
- если ширина экрана стала меньше 768px, карточки становятся в колонку;
- каждая карточка занимает всю доступную ширину.

---

## 4. Flexbox и Grid

Для адаптивной верстки часто используют современные CSS-инструменты:

### Flexbox

Удобен для расположения элементов в строку или колонку.

```css
.menu {
  display: flex;
  gap: 16px;
  flex-wrap: wrap;
}
```

`flex-wrap: wrap;` позволяет элементам переноситься на новую строку, если места мало.

---

### CSS Grid

Удобен для сеток, карточек, галерей и сложных раскладок.

```css
.gallery {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}

@media (max-width: 768px) {
  .gallery {
    grid-template-columns: 1fr;
  }
}
```

На большом экране будет 3 колонки, а на маленьком — 1 колонка.

---

## 5. Адаптивные изображения

Изображения тоже должны подстраиваться под экран.

Обычно используют такое правило:

```css
img {
  max-width: 100%;
  height: auto;
}
```

Это значит:

- изображение не будет выходить за пределы своего контейнера;
- высота изменится пропорционально ширине.

Без этого большая картинка может «сломать» верстку на маленьком экране.

---

## Mobile-first подход

**Mobile-first** — это подход, когда сначала верстают сайт для мобильного экрана, а потом добавляют стили для более широких экранов.

Например:

```css
.cards {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

@media (min-width: 768px) {
  .cards {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

Здесь сначала задана одна колонка для телефона.  
Потом, если экран шире 768px, появляются три колонки.

Такой подход часто удобен, потому что мобильная версия обычно проще, а затем ее можно постепенно усложнять.

---

## Breakpoints

**Breakpoint** — это ширина экрана, на которой меняется поведение верстки.

Например:

```css
@media (max-width: 768px) {
  /* стили для экранов до 768px */
}
```

Часто встречаются такие ориентиры:

| Размер | Пример устройства |
|---|---|
| до 480px | маленькие смартфоны |
| до 768px | смартфоны и небольшие планшеты |
| до 1024px | планшеты и маленькие ноутбуки |
| от 1200px | большие экраны |

Но важно понимать: эти значения не являются строгими правилами. Их выбирают в зависимости от конкретного макета.

---

## Адаптивная и отзывчивая верстка

Иногда встречаются два похожих понятия:

| Понятие | Смысл |
|---|---|
| Адаптивная верстка | Макет меняется под разные размеры экрана |
| Отзывчивая верстка | Макет плавно реагирует на изменение размера окна |

На практике эти термины часто используют почти как синонимы. Главное — сайт должен хорошо работать на разных устройствах.

## Простой пример адаптивной страницы

HTML:

```html
<section class="cards">
  <article class="card">
    <h2>Карточка 1</h2>
    <p>Описание первой карточки.</p>
  </article>

  <article class="card">
    <h2>Карточка 2</h2>
    <p>Описание второй карточки.</p>
  </article>

  <article class="card">
    <h2>Карточка 3</h2>
    <p>Описание третьей карточки.</p>
  </article>
</section>
```

CSS:

```css
.cards {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

.card {
  padding: 20px;
  border: 1px solid #ddd;
  border-radius: 12px;
}

@media (min-width: 768px) {
  .cards {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

Что получится:

- на телефоне карточки идут одна под другой;
- на экране шире 768px карточки становятся в три колонки.

## Частые ошибки новичков

### Жесткая ширина блоков

```css
.block {
  width: 1000px;
}
```

На маленьком экране такой блок не поместится.

Лучше:

```css
.block {
  width: 100%;
  max-width: 1000px;
}
```

---

### Отсутствие адаптивности у изображений

Плохо:

```css
img {
  width: 800px;
}
```

Лучше:

```css
img {
  max-width: 100%;
  height: auto;
}
```

---

### Слишком мелкий текст на телефоне

Если текст приходится увеличивать пальцами, это плохой знак.  
Обычно основной текст делают достаточно крупным, например около `16px`.

---

### Слишком маленькие кнопки

На телефоне кнопки нажимают пальцем, а не курсором. Поэтому интерактивные элементы должны быть достаточно большими и иметь отступы.

```css
.button {
  padding: 12px 20px;
}
```

---

## Что важно запомнить

**Адаптивная верстка** — это способ сделать сайт удобным на разных экранах.

Главные инструменты:

- гибкие размеры;
- относительные единицы;
- media queries;
- Flexbox;
- CSS Grid;
- адаптивные изображения;
- mobile-first подход.

Для начинающего веб-кодера важно понять не только отдельные CSS-свойства, но и саму идею: **страница не должна быть жесткой, она должна уметь подстраиваться под экран пользователя**.
