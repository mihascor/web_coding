# UI и UX: чем внешний вид отличается от удобства

## Коротко

UI и UX описывают разные стороны пользовательского интерфейса.

**UI** отвечает за внешний вид: цвета, шрифты, кнопки, поля, иконки, отступы и расположение элементов.

**UX** отвечает за удобство: легко ли человеку найти нужную функцию, понять следующий шаг, выполнить задачу и исправить ошибку.

Красивый сайт может быть неудобным, а удобный — выглядеть неаккуратно. Хороший интерфейс объединяет понятный внешний вид и продуманный пользовательский сценарий.

При работе с ИИ полезно разделять UI- и UX-задачи. Это помогает не смешивать просьбу изменить цвета с просьбой упростить форму или порядок действий.

## Что такое UI

UI расшифровывается как **User Interface** — пользовательский интерфейс.

К UI относятся все видимые элементы страницы:

* кнопки;
* поля формы;
* меню;
* карточки;
* таблицы;
* уведомления;
* изображения;
* заголовки;
* иконки.

UI определяет, как эти элементы выглядят:

* какой у них цвет;
* какой используется шрифт;
* насколько крупный текст;
* какие между блоками отступы;
* как выделяется основная кнопка;
* как показываются ошибки и активные состояния.

Хороший UI создаёт понятную визуальную систему. Одинаковые элементы выглядят одинаково, а главное действие заметно отличается от второстепенных.

Например, кнопка «Записаться» может быть выделена основным цветом, а ссылка «Подробнее» оформлена спокойнее.

## Что такое UX

UX расшифровывается как **User Experience** — пользовательский опыт.

UX рассматривает не только отдельный элемент, а весь путь человека к результату.

Например, пользователь хочет записаться на услугу. Он должен:

1. Найти кнопку записи.
2. Выбрать услугу.
3. Выбрать специалиста и время.
4. Ввести контактные данные.
5. Подтвердить запись.
6. Получить понятное сообщение об успехе.

UX отвечает за то, насколько этот процесс логичен и удобен.

Плохой пользовательский опыт может возникнуть, если:

* нужная кнопка спрятана;
* форма запрашивает лишние данные;
* введённая информация исчезает после ошибки;
* непонятно, завершилось ли действие;
* пользователь не может вернуться к предыдущему шагу;
* мобильная версия мешает заполнить форму.

Главный вопрос UX:

> Может ли пользователь без лишних препятствий выполнить свою задачу?

## В чём разница

UI отвечает на вопрос:

> Как интерфейс выглядит?

UX отвечает на вопрос:

> Как человек им пользуется?

Рассмотрим форму записи.

К UI относятся:

* цвет и размер полей;
* оформление кнопки;
* расстояние между элементами;
* вид сообщений об ошибке;
* стиль календаря.

К UX относятся:

* понятен ли порядок полей;
* нужны ли все запрашиваемые данные;
* удобно ли выбирать дату;
* объясняются ли ошибки;
* сохраняются ли введённые данные;
* понятно ли, что запись создана.

UI и UX связаны. Например, слишком бледный текст является проблемой внешнего вида, но одновременно ухудшает удобство чтения.

## Пользовательская цель и сценарий

UX начинается с цели пользователя.

Человек приходит на страницу, чтобы:

* узнать цену;
* найти адрес;
* записаться;
* купить товар;
* отправить заявку;
* изменить данные;
* проверить статус.

После определения цели выстраивается пользовательский сценарий — последовательность шагов до результата.

Например, цель страницы услуги может заключаться не только в том, чтобы рассказать о процедуре. Пользователь также хочет понять:

* подходит ли ему услуга;
* сколько она стоит;
* кто её проводит;
* как записаться.

Поэтому хороший интерфейс должен показывать информацию в понятном порядке и подводить человека к следующему действию.

При работе с ИИ полезно описывать не просто страницу, а пользовательский сценарий:

> Пользователь впервые открывает страницу услуги. Он должен узнать основную информацию, увидеть стоимость и записаться без регистрации.

## Визуальная иерархия

Визуальная иерархия показывает, какие элементы важнее.

Она создаётся с помощью:

* размера;
* цвета;
* контраста;
* расположения;
* отступов;
* группировки.

Пользователь должен быстро заметить:

1. Название страницы.
2. Основную информацию.
3. Главное действие.
4. Дополнительные сведения.

Если все элементы одинаково яркие и крупные, человеку сложнее понять, на что обратить внимание.

Особенно важно различать основное и второстепенное действие.

Например:

* «Записаться» — основная кнопка;
* «Узнать подробнее» — второстепенная ссылка;
* «Вернуться назад» — вспомогательное действие.

UI визуально показывает приоритеты, а UX определяет, какие приоритеты нужны пользователю.

## Состояния и обратная связь

Интерфейс должен сообщать пользователю, что происходит после его действия.

Например, форма может находиться в состояниях:

* ожидание ввода;
* ошибка заполнения;
* отправка;
* успешная отправка;
* ошибка сервера.

После нажатия кнопки пользователь должен понимать:

* началась ли загрузка;
* принято ли действие;
* возникла ли ошибка;
* что нужно сделать дальше.

Сообщение «Ошибка» почти не помогает. Лучше объяснить проблему:

> Не удалось отправить форму. Проверь подключение к интернету и попробуй ещё раз. Введённые данные сохранены.

UX определяет содержание сообщения, а UI — как оно будет оформлено и где появится.

## Тексты как часть интерфейса

Названия кнопок, подписи полей и сообщения об ошибках влияют на удобство не меньше, чем расположение элементов.

Кнопка должна описывать результат действия.

Вместо общего слова «Отправить» часто лучше использовать:

* «Записаться»;
* «Сохранить изменения»;
* «Отправить заявку»;
* «Подтвердить оплату».

Подпись должна помогать понять, что произойдёт после нажатия.

Сообщения об ошибках также должны быть конкретными:

* какое поле заполнено неправильно;
* какой формат требуется;
* как исправить ошибку.

ИИ можно отдельно попросить проверить интерфейсные тексты, не меняя структуру и оформление страницы.

## Связь с HTML, CSS и JavaScript

UI и UX реализуются через основные технологии интерфейса.

**HTML** создаёт структуру и смысл элементов. Он определяет, где находится кнопка, поле, заголовок или меню.

**CSS** отвечает за внешний вид: цвета, размеры, сетку, отступы и адаптивность.

**JavaScript** управляет поведением: открывает окна, проверяет формы, загружает данные и переключает состояния.

На UX также влияют backend и сервер. Даже хорошо оформленная форма будет неудобной, если сервер работает медленно, теряет данные или возвращает непонятные ошибки.

Поэтому пользовательский опыт создаётся всей системой, а не только дизайном страницы.

## Почему это важно при работе с ИИ

Запрос:

> Улучши дизайн страницы.

слишком общий. ИИ может изменить внешний вид, хотя проблема заключается в порядке действий или непонятной форме.

Лучше разделить задачу.

Для UX:

> Проверь сценарий записи. Найди лишние шаги, непонятные действия и места, где пользователь может потерять введённые данные. Пока не меняй визуальный стиль.

Для UI:

> Сохрани существующий сценарий, но улучши визуальную иерархию, отступы, типографику и состояния кнопок.

Для полноценной задачи полезно указать:

* кто пользователь;
* какую цель он хочет достичь;
* какое действие главное;
* какие состояния нужно предусмотреть;
* какой визуальный стиль используется;
* что нельзя менять.

## Как проверять результат ИИ

Не оценивай интерфейс только по первому экрану.

Нужно пройти весь пользовательский сценарий:

* найти основное действие;
* заполнить форму;
* допустить ошибку;
* исправить данные;
* дождаться результата;
* проверить мобильную версию.

Обрати внимание:

* понятно ли назначение страницы;
* заметно ли главное действие;
* нет ли лишних шагов;
* сохраняются ли данные после ошибки;
* понятны ли сообщения;
* удобно ли пользоваться интерфейсом с телефона;
* одинаково ли работают похожие элементы.

Если ИИ предлагает решение, полезно попросить его объяснить:

> Какую пользовательскую задачу решает каждый блок и почему элементы расположены именно в таком порядке?

## Простая аналогия

UI и UX можно сравнить с рестораном.

UI — это интерьер, меню, посуда, освещение и оформление столов.

UX — это весь опыт посетителя: легко ли найти вход, быстро ли получить меню, понятно ли описание блюд, удобно ли сделать заказ и оплатить счёт.

Красивый ресторан может быть неудобным, если посетитель долго ждёт и не понимает, к кому обратиться.

Хороший результат появляется, когда внешний вид помогает человеку получить нужный результат.

## Частые заблуждения

### «UI и UX — одно и то же»

UI связан с внешним видом, а UX — с общим опытом использования.

### «Красивый сайт всегда удобен»

Эффектное оформление не исправляет запутанную навигацию или длинную форму.

### «UX — это только расположение кнопок»

UX включает весь сценарий, тексты, состояния, ошибки, скорость и доступность.

### «UI — это только цвета»

UI также включает типографику, сетки, размеры, отступы и состояния элементов.

### «Чем меньше шагов, тем лучше»

Несколько понятных этапов могут быть удобнее одной перегруженной формы.

### «ИИ сам создаст хороший UX»

ИИ не знает цели аудитории и правила проекта без переданного контекста.

## Что важно запомнить

* UI отвечает за внешний вид интерфейса.
* UX отвечает за удобство и путь пользователя к результату.
* Хороший интерфейс объединяет UI и UX.
* UX начинается с цели пользователя и последовательности действий.
* UI помогает показать приоритеты и состояния.
* После каждого действия интерфейс должен давать понятную обратную связь.
* Тексты кнопок, полей и ошибок являются частью UX.
* HTML, CSS, JavaScript и серверная логика вместе формируют пользовательский опыт.
* При работе с ИИ UI- и UX-задачи лучше формулировать отдельно.
* Человек определяет цель и проверяет сценарий, а ИИ помогает предложить и реализовать решение.

## Промты для самостоятельного уточнения

* «Объясни разницу между UI и UX на примере страницы онлайн-записи.»

* «Проанализируй эту страницу отдельно с точки зрения UI и UX.»

* «Помоги определить основную цель пользователя и построить путь к её достижению.»

* «Найди лишние шаги, непонятные действия и возможные ошибки в этом сценарии.»

* «Улучши визуальную иерархию, не меняя логику страницы.»

* «Проверь подписи кнопок, полей и сообщения об ошибках.»

* «Перечисли состояния, которые нужно предусмотреть для этой формы.»

* «Помоги составить промт для улучшения удобства страницы без изменения фирменного стиля.»
