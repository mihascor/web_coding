# 11. DOM: как браузер видит страницу

Когда браузер получает HTML-файл, он не работает с ним как с обычным текстом. Браузер преобразует HTML-код в специальную структуру — **DOM**.

DOM расшифровывается как **Document Object Model**, то есть объектная модель документа.

DOM можно представить как дерево, в котором каждый элемент страницы становится отдельным объектом. Благодаря этой структуре JavaScript может находить элементы, изменять их, добавлять новые или удалять существующие.

## Как HTML превращается в DOM

Допустим, в HTML записано:

```html
<body>
  <h1>Запись в клинику</h1>
  <button>Записаться</button>
</body>
```

Браузер воспринимает эту страницу примерно как структуру:

```text
document
└── html
    └── body
        ├── h1
        └── button
```

Элементы находятся внутри друг друга и образуют иерархию.

В данном примере:

* `document` — весь открытый документ;
* `html` — основной элемент страницы;
* `body` — видимая часть страницы;
* `h1` — заголовок;
* `button` — кнопка.

Такая структура называется **DOM-деревом**.

## Что считается DOM-элементом

Практически каждый HTML-тег становится элементом DOM:

* заголовок;
* абзац;
* кнопка;
* изображение;
* поле формы;
* ссылка;
* список;
* секция страницы.

У каждого элемента могут быть:

* текст;
* атрибуты;
* CSS-классы;
* вложенные элементы;
* обработчики событий.

Например:

```html
<button class="booking-button" disabled>
  Записаться
</button>
```

В DOM кнопка содержит:

* тип элемента — `button`;
* класс — `booking-button`;
* атрибут — `disabled`;
* текст — `Записаться`.

JavaScript может изменить любую из этих частей.

## Как JavaScript находит элементы

Чтобы изменить элемент, JavaScript сначала должен найти его в DOM.

Один из распространённых способов — `querySelector`:

```javascript
const button = document.querySelector('.booking-button');
```

Этот код означает:

1. обратиться к документу;
2. найти элемент с классом `booking-button`;
3. сохранить найденный элемент в переменную `button`.

После этого JavaScript может работать с кнопкой.

Например, изменить текст:

```javascript
button.textContent = 'Выберите время';
```

Или сделать кнопку активной:

```javascript
button.disabled = false;
```

Важно понимать общую последовательность:

**найти элемент → сохранить ссылку на него → выполнить изменение**

## Как JavaScript изменяет DOM

JavaScript может изменять страницу уже после её загрузки.

Он способен:

* менять текст;
* заменять изображения;
* добавлять и удалять CSS-классы;
* скрывать и показывать блоки;
* создавать новые элементы;
* удалять элементы;
* изменять атрибуты;
* добавлять содержимое в списки.

Например, скрыть сообщение:

```javascript
const message = document.querySelector('.message');

message.hidden = true;
```

Или изменить класс элемента:

```javascript
message.classList.add('success');
```

После выполнения кода браузер сразу обновит отображение страницы.

## DOM и события

DOM позволяет JavaScript не только изменять элементы, но и отслеживать действия пользователя.

Например:

```javascript
const button = document.querySelector('.booking-button');

button.addEventListener('click', function () {
  button.textContent = 'Заявка отправлена';
});
```

Здесь JavaScript:

1. находит кнопку;
2. добавляет обработчик события `click`;
3. ожидает нажатия;
4. изменяет текст кнопки.

DOM связывает видимые элементы страницы с логикой JavaScript.

## Создание новых элементов

JavaScript может добавлять на страницу элементы, которых изначально не было в HTML.

Например:

```javascript
const item = document.createElement('li');

item.textContent = 'Консультация косметолога';
```

Этот код создаёт новый элемент списка, но пока не показывает его на странице.

Чтобы добавить его в DOM, нужно указать родительский элемент:

```javascript
const list = document.querySelector('.services');

list.append(item);
```

Теперь новый пункт появится внутри списка.

Общая схема:

**создать элемент → заполнить содержимое → добавить в DOM**

## DOM и CSS

DOM связан не только с JavaScript, но и с CSS.

JavaScript часто не меняет оформление напрямую, а добавляет или удаляет CSS-класс.

Например:

```javascript
menu.classList.add('open');
```

CSS может содержать правило:

```css
.menu.open {
  display: block;
}
```

В этом случае:

* JavaScript управляет состоянием элемента;
* CSS определяет его внешний вид.

Такой подход помогает разделять поведение и оформление.

## DOM и исходный HTML — не всегда одно и то же

Исходный HTML — это код, который сервер отправил браузеру.

DOM — это текущее состояние страницы после того, как браузер обработал HTML и выполнил JavaScript.

Например, исходный HTML может содержать пустой список:

```html
<ul class="appointments"></ul>
```

После загрузки данных JavaScript добавит в него записи. В DOM они уже будут присутствовать, хотя в исходном HTML их не было.

Поэтому страница может меняться без полной перезагрузки.

## Как посмотреть DOM в браузере

DOM можно увидеть в инструментах разработчика браузера.

Обычно для этого нужно:

1. открыть страницу;
2. нажать правой кнопкой мыши на элемент;
3. выбрать «Просмотреть код» или «Исследовать»;
4. открыть вкладку Elements или «Элементы».

Там отображается текущая структура DOM.

В инструментах разработчика можно:

* посмотреть вложенность элементов;
* найти классы и атрибуты;
* временно изменить текст;
* добавить или удалить CSS-класс;
* проверить, как устроен конкретный блок.

Изменения, сделанные через инструменты разработчика, обычно исчезают после перезагрузки страницы, потому что файлы проекта при этом не меняются.

## Почему DOM важен при работе с ИИ

Когда вы просите ИИ изменить поведение страницы, ему важно понимать, с какими элементами нужно работать.

Недостаточно написать:

> Сделай кнопку рабочей.

Полезнее указать:

> Найди кнопку с классом `booking-button`. После нажатия открой блок с классом `booking-form` и добавь ему класс `open`.

Такой запрос описывает:

* нужный DOM-элемент;
* событие;
* выполняемое действие;
* ожидаемое изменение.

Если вы не знаете точные классы, можно передать ИИ HTML-фрагмент и попросить определить нужные элементы.

## Как проверять работу с DOM

После изменений проверьте:

* правильно ли JavaScript находит элемент;
* совпадает ли класс или идентификатор с HTML;
* существует ли элемент в момент выполнения кода;
* меняется ли нужный элемент, а не похожий;
* работает ли действие повторно;
* не появляются ли дублирующиеся элементы;
* корректно ли отображается результат после обновления страницы.

Частая причина ошибки — JavaScript ищет класс, которого нет в HTML, или запускается раньше, чем браузер создал нужный элемент.

## Главное

DOM — это представление HTML-страницы внутри браузера.

Браузер превращает HTML в дерево элементов, а JavaScript использует это дерево, чтобы:

* находить элементы;
* реагировать на события;
* изменять содержимое;
* добавлять и удалять элементы;
* управлять классами и атрибутами.

Базовая модель работы выглядит так:

**HTML → DOM → JavaScript находит элемент → DOM изменяется → браузер обновляет страницу**

Для работы с ИИ важно уметь описывать не только желаемый внешний результат, но и конкретные элементы, события и изменения в структуре страницы.

### Промты для самостоятельного уточнения

> Покажи DOM-дерево для этого HTML-кода и объясни его простыми словами.

> Объясни разницу между HTML-файлом и текущим DOM страницы.

> Найди в этом коде элемент, который нужно изменить с помощью JavaScript.

> Объясни построчно, как этот JavaScript находит и изменяет DOM-элемент.

> Проверь, совпадают ли классы в HTML, CSS и JavaScript.

> Объясни, почему `querySelector` не находит элемент на странице.
