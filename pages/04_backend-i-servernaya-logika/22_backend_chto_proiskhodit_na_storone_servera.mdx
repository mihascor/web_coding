# Backend: что происходит на стороне сервера

## Коротко

Когда ты открываешь сайт, нажимаешь кнопку, отправляешь форму или входишь в личный кабинет, часть работы выполняется прямо в браузере. Но многие важные действия происходят в другой части системы — на сервере.

Эта часть веб-приложения называется **backend**, или серверная часть. Она принимает запросы, проверяет данные, применяет правила приложения, обращается к базе данных и возвращает результат.

Понимание backend помогает точнее объяснять ИИ-ассистенту, что должно происходить не только на экране пользователя, но и внутри всей системы.

## Что такое backend

**Backend** — это часть веб-приложения, которая работает на сервере и обычно недоступна пользователю напрямую.

Пользователь видит страницы, кнопки, формы и сообщения. Backend находится за этим интерфейсом и выполняет внутреннюю работу приложения.

Например, когда пользователь отправляет форму записи в клинику, серверная часть может:

* принять введённые данные;
* проверить их правильность;
* убедиться, что выбранное время свободно;
* сохранить запись в базе данных;
* отправить уведомление;
* вернуть браузеру сообщение об успешной записи.

Пользователь видит только конечный результат. Все промежуточные действия происходят на стороне сервера.

Backend обычно состоит не из одного файла или действия, а из нескольких связанных частей: маршрутов, правил обработки данных, работы с базой, системы авторизации и интеграций с внешними сервисами.

## Для чего используется backend

Backend нужен тогда, когда сайт должен не просто показывать заранее подготовленную информацию, а выполнять действия, хранить данные и учитывать правила приложения.

Серверная часть может отвечать за:

* регистрацию и вход пользователей;
* работу личного кабинета;
* сохранение заявок, заказов и записей;
* получение информации из базы данных;
* проверку прав доступа;
* расчёт стоимости;
* обработку оплаты;
* отправку писем и уведомлений;
* загрузку и хранение файлов;
* обмен данными с CRM, мессенджерами и другими сервисами.

Например, интернет-магазин должен проверить наличие товара, рассчитать итоговую сумму, создать заказ и сохранить его. Сделать всё это только с помощью интерфейса в браузере недостаточно. Нужна серверная логика.

## Что происходит после запроса пользователя

Обычно работа backend начинается с запроса от frontend.

Представим, что пользователь нажал кнопку «Записаться». Дальше система может пройти несколько этапов.

### 1. Браузер отправляет запрос

Frontend собирает нужную информацию и отправляет её на сервер. В запросе могут находиться имя пользователя, выбранная услуга, дата и время.

### 2. Сервер принимает запрос

Backend определяет, какое действие запрашивает пользователь и какая часть приложения должна его обработать.

### 3. Данные проверяются

Сервер проверяет, заполнены ли обязательные поля, допустимы ли полученные значения и имеет ли пользователь право выполнить это действие.

Даже если данные уже проверялись в браузере, сервер должен провести собственную проверку. Пользователь может изменить или подделать данные, отправляемые с frontend.

### 4. Выполняются правила приложения

Backend применяет бизнес-логику — правила, по которым работает конкретный проект.

Например:

* нельзя записаться на прошедшую дату;
* один временной интервал нельзя занять дважды;
* скидка действует только для определённой категории пользователей;
* менеджер видит заявки своего филиала, а администратор — все заявки.

### 5. Сервер обращается к базе данных

Backend может найти, создать, изменить или удалить нужную информацию.

Например, он проверяет расписание врача, создаёт новую запись и связывает её с аккаунтом пользователя.

### 6. При необходимости подключаются внешние сервисы

Сервер может отправить данные в CRM, платёжную систему, сервис рассылок или мессенджер.

### 7. Backend возвращает ответ

После выполнения всех действий сервер сообщает frontend результат.

Например:

* запись успешно создана;
* выбранное время уже занято;
* пользователь не авторизован;
* данные заполнены неправильно;
* на сервере произошла ошибка.

Frontend получает этот ответ и показывает пользователю соответствующее сообщение.

## Как backend связан с другими частями веб-приложения

Backend не работает отдельно от всего проекта. Он связывает несколько важных частей системы.

### Frontend

Frontend показывает интерфейс и принимает действия пользователя. Он отправляет запросы на backend и отображает полученные ответы.

Frontend отвечает за то, **как выглядит действие**, а backend — за то, **как оно обрабатывается внутри системы**.

### API

API определяет, каким способом frontend и backend обмениваются запросами и ответами.

Можно сказать, что API — это набор согласованных правил общения между частями приложения.

### База данных

База данных хранит пользователей, заказы, записи, товары и другую информацию. Backend управляет работой с этими данными.

Обычно frontend не должен обращаться к базе напрямую. Он отправляет запрос серверу, а сервер решает, какие данные можно получить или изменить.

### Внешние сервисы

Backend может подключать приложение к другим системам: платёжным сервисам, CRM, картам, службам доставки, email-рассылкам и мессенджерам.

Серверная часть передаёт этим сервисам данные и обрабатывает их ответы.

## Почему важные действия выполняются на сервере

Некоторые действия нельзя безопасно доверять браузеру пользователя.

Код frontend загружается на устройство пользователя и может быть просмотрен или изменён. Поэтому в нём нельзя надёжно хранить секретные ключи, принимать окончательные решения о правах доступа или доверять рассчитанной пользователем стоимости заказа.

Backend работает в контролируемой серверной среде. Это позволяет:

* скрывать секретные настройки и ключи;
* проверять права пользователя;
* защищать доступ к базе данных;
* централизованно применять правила приложения;
* контролировать полученные данные;
* фиксировать действия и ошибки в логах.

Это не означает, что наличие backend автоматически делает проект безопасным. Серверную часть тоже нужно правильно проектировать и защищать.

## Почему это важно при работе с ИИ-ассистентом

Без понимания backend легко описать ИИ только внешний результат.

Например, запрос «Сделай форму записи» может означать совершенно разные задачи:

* нарисовать форму на странице;
* добавить проверку полей в браузере;
* отправлять данные на сервер;
* сохранять запись в базе;
* проверять доступное время;
* отправлять уведомление администратору.

Если не уточнить ожидаемое поведение, ИИ может создать только интерфейс, который выглядит рабочим, но ничего не сохраняет.

При постановке задачи полезно сообщать ИИ:

* какие данные поступают от пользователя;
* что сервер должен с ними сделать;
* какие проверки необходимы;
* где должны храниться результаты;
* кто имеет право выполнять действие;
* какой ответ должен получить frontend;
* какие внешние сервисы нужно подключить;
* что должно происходить при ошибке.

Также можно попросить ИИ сначала описать серверный процесс обычными словами, а уже затем предлагать структуру проекта или писать код.

Так ты сможешь проверить логику до начала реализации.

## Простая аналогия

Frontend можно сравнить с залом ресторана, а backend — с кухней.

Посетитель видит меню, делает заказ и получает готовое блюдо. Но он не наблюдает, как сотрудники проверяют наличие продуктов, передают заказ повару, готовят блюдо и организуют выдачу.

Интерфейс сайта похож на меню и работу официанта. Серверная часть — на внутренние процессы кухни. Она получает запрос, выполняет необходимые действия и возвращает результат.

Красивое меню без работающей кухни не позволит ресторану обслужить посетителя. Так же и красивый интерфейс без необходимой серверной логики не сможет полноценно выполнять задачи веб-приложения.

## Частые заблуждения новичков

### Backend — это просто база данных

База данных только хранит информацию. Backend решает, какие данные получить, как их проверить, изменить и кому разрешить доступ.

### Сервер и backend — одно и то же

Сервер — это среда, в которой может работать приложение. Backend — программная часть, выполняющая серверную логику.

На одном сервере могут находиться несколько приложений и сервисов.

### Если форма выглядит рабочей, данные уже сохраняются

Форма может принимать данные и показывать сообщение, но без связи с backend информация может никуда не отправляться и не сохраняться.

### Проверки на frontend достаточно

Проверка в браузере улучшает удобство пользователя, но не защищает сервер. Backend должен самостоятельно проверять все полученные данные.

### Frontend может напрямую работать с базой данных

В большинстве обычных веб-приложений это небезопасно. Доступ к базе контролирует backend.

### Backend нужен каждому сайту

Простой сайт с неизменяемыми страницами может работать без собственной серверной логики. Backend становится необходимым, когда появляются аккаунты, формы с сохранением, личные кабинеты, заказы, платежи или другие динамические функции.

### ИИ сам определит все правила приложения

ИИ может предложить типовой вариант, но он не знает всех требований бизнеса. Человек должен определить правила, ограничения, роли пользователей и ожидаемые результаты.

## Что важно запомнить

* Backend — это часть приложения, работающая на стороне сервера.
* Он принимает запросы, проверяет данные и выполняет правила приложения.
* Backend управляет взаимодействием с базой данных и внешними сервисами.
* Frontend показывает результат, а backend выполняет внутреннюю обработку.
* Важные проверки и решения нельзя оставлять только на стороне браузера.
* Интерфейс может выглядеть готовым, даже если серверная часть ещё не реализована.
* При работе с ИИ нужно отдельно описывать серверные действия, данные, проверки и возможные ошибки.
* Человек определяет правила приложения, а ИИ помогает превратить их в техническое решение.

## Промты для самостоятельного уточнения

* «Объясни на простом примере, что происходит на backend после отправки формы пользователем.»

* «Покажи без сложного кода, как frontend, backend, API и база данных взаимодействуют в одном веб-приложении.»

* «Помоги определить, какие функции моего проекта требуют backend, а какие могут работать только на frontend.»

* «Составь пошаговую схему серверной обработки записи пользователя на услугу.»

* «Объясни, какие проверки должен выполнять backend и почему нельзя оставить их только на стороне браузера.»

* «Задай мне вопросы, которые помогут правильно описать серверную логику моего веб-приложения.»

* «Покажи пример хорошего промта для создания backend функции, которая принимает данные, проверяет их и сохраняет в базе.»

* «Проанализируй описание моей функции и раздели задачи между frontend, backend и базой данных.»
